# Vue 中的diff算法

## 一、是什么

`diff` 算法是一种通过同层的树节点进行比较的高效算法

其有两个特点：

* 比较只会在同层级进行, 不会跨层级比较
* 在diff比较的过程中，循环从两边向中间比较

`diff` 算法在很多场景下都有应用，在 `vue` 中，作用于虚拟 `dom` 渲染成真实 `dom` 的新旧 `VNode` 节点比较

## 二、比较方式

`diff`整体策略为：**深度优先，同层比较**

1. 比较只会在同层级进行, 不会跨层级比较

![91e9c9519a11caa0c5bf70714383f054.png](/assets/img/posts/Web前端/Vue/1772769088595-734986ef-fd0d-4df3-83f3-ed594bbe313b-876575.png)

2. 比较的过程中，循环从两边向中间收拢

![2dcd6ad5cf82c65b9cfc43a27ba1e4ec.png](/assets/img/posts/Web前端/Vue/1772769088399-51e4f4a9-c796-46f6-a483-2431310e2ecc-666181.png)

下面举个`vue`通过`diff`算法更新的例子：

新旧`VNode`节点如下图所示：

![80dc339f73b186479e6d1fc18bfbf66d.png](/assets/img/posts/Web前端/Vue/1772769088417-aadea233-f325-4938-9ff6-e002486f3dcf-530254.png)

第一次循环后，发现旧节点D与新节点D相同，直接复用旧节点D作为`diff`后的第一个真实节点，同时旧节点`endIndex`移动到C，新节点的 `startIndex` 移动到了 C

![76032c78c8ef74047efd42c070e48854.png](/assets/img/posts/Web前端/Vue/1772769088472-590efc9c-ca4c-4656-8296-9cf53c81030c-709737.png)

第二次循环后，同样是旧节点的末尾和新节点的开头(都是 C)相同，同理，`diff` 后创建了 C 的真实节点插入到第一次创建的 D 节点后面。同时旧节点的 `endIndex` 移动到了 B，新节点的 `startIndex` 移动到了 E

![1c76e7489660188d35f0a38ea8c8ecd7.png](/assets/img/posts/Web前端/Vue/1772769088529-a92b953d-5af5-4a7a-a3fe-e7894d383a66-562633.png)

第三次循环中，发现E没有找到，这时候只能直接创建新的真实节点 E，插入到第二次创建的 C 节点之后。同时新节点的 `startIndex` 移动到了 A。旧节点的 `startIndex` 和 `endIndex` 都保持不动

![4b622c0d61673ec5474465d82305d308.png](/assets/img/posts/Web前端/Vue/1772769088525-1b1a341a-76e3-4ecc-ae31-f65853f6afb4-845788.png)

第四次循环中，发现了新旧节点的开头(都是 A)相同，于是 `diff` 后创建了 A 的真实节点，插入到前一次创建的 E 节点后面。同时旧节点的 `startIndex` 移动到了 B，新节点的`startIndex` 移动到了 B

![5982417c3e0b2fa9ae940354a0e67ab4.png](/assets/img/posts/Web前端/Vue/1772769088385-946f5927-be59-4b9c-a8d5-6c0a3cbc8cad-585516.png)

第五次循环中，情形同第四次循环一样，因此 `diff` 后创建了 B 真实节点 插入到前一次创建的 A 节点后面。同时旧节点的 `startIndex`移动到了 C，新节点的 startIndex 移动到了 F

![16cf0ef90f6e19d26c0ddffeca067e86.png](/assets/img/posts/Web前端/Vue/1772769088540-1ab73a90-807d-4a6e-aa91-6cccfdc46f31-250956.png)

新节点的 `startIndex` 已经大于 `endIndex` 了，需要创建 `newStartIdx` 和 `newEndIdx` 之间的所有节点，也就是节点F，直接创建 F 节点对应的真实节点放到 B 节点后面

![dc215b45682cf6c9cc4700a5425673ad.png](/assets/img/posts/Web前端/Vue/1772769089404-0f03f1ce-093c-4920-bebc-644ad7d5852b-625832.png)
