# 受控组件 vs 非受控组件

## <font style="color:rgb(15, 17, 21);">概念理解</font>

### <font style="color:rgb(15, 17, 21);">🎯</font><font style="color:rgb(15, 17, 21);"> 受控组件 (Controlled Component)</font>

* **<font style="color:rgb(15, 17, 21);">数据由父组件控制</font>**<font style="color:rgb(15, 17, 21);">：组件的值通过</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">value</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">属性从外部传入</font>
* **<font style="color:rgb(15, 17, 21);">单向数据流</font>**<font style="color:rgb(15, 17, 21);">：数据从父组件流向子组件</font>
* **<font style="color:rgb(15, 17, 21);">状态提升</font>**<font style="color:rgb(15, 17, 21);">：状态保存在父组件中，子组件只是"展示"数据</font>

### <font style="color:rgb(15, 17, 21);">🎮</font><font style="color:rgb(15, 17, 21);"> 非受控组件 (Uncontrolled Component)</font>

* **<font style="color:rgb(15, 17, 21);">数据由组件自己管理</font>**<font style="color:rgb(15, 17, 21);">：组件内部维护自己的状态</font>
* **<font style="color:rgb(15, 17, 21);">双向数据流</font>**<font style="color:rgb(15, 17, 21);">：组件内部可以修改自己的状态</font>
* **<font style="color:rgb(15, 17, 21);">状态内聚</font>**<font style="color:rgb(15, 17, 21);">：状态保存在组件内部</font>

***

## <font style="color:rgb(15, 17, 21);">📊</font><font style="color:rgb(15, 17, 21);"> 对比表格</font>

| <font style="color:rgb(15, 17, 21);">特性</font> | **<font style="color:rgb(15, 17, 21);">受控组件</font>** | **<font style="color:rgb(15, 17, 21);">非受控组件</font>** |
| --- | --- | --- |
| <font style="color:rgb(15, 17, 21);">数据来源</font> | <font style="color:rgb(15, 17, 21);">通过</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">value</font></code><br/><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">从外部传入</font> | <font style="color:rgb(15, 17, 21);">组件内部管理</font> |
| <font style="color:rgb(15, 17, 21);">状态位置</font> | <font style="color:rgb(15, 17, 21);">父组件</font> | <font style="color:rgb(15, 17, 21);">组件自身</font> |
| <font style="color:rgb(15, 17, 21);">数据流</font> | <font style="color:rgb(15, 17, 21);">单向</font> | <font style="color:rgb(15, 17, 21);">双向</font> |
| <font style="color:rgb(15, 17, 21);">可预测性</font> | <font style="color:rgb(15, 17, 21);">高</font> | <font style="color:rgb(15, 17, 21);">低</font> |
| <font style="color:rgb(15, 17, 21);">调试难度</font> | <font style="color:rgb(15, 17, 21);">低</font> | <font style="color:rgb(15, 17, 21);">高</font> |
| <font style="color:rgb(15, 17, 21);">适用场景</font> | <font style="color:rgb(15, 17, 21);">表单、复杂交互</font> | <font style="color:rgb(15, 17, 21);">简单独立组件</font> |

***

## <font style="color:rgb(15, 17, 21);">💡</font><font style="color:rgb(15, 17, 21);"> 核心原则</font>

**<font style="color:rgb(15, 17, 21);">在 Flutter 中，推荐使用受控组件</font>**<font style="color:rgb(15, 17, 21);">，因为：</font>

* <font style="color:rgb(15, 17, 21);">数据流更清晰</font>
* <font style="color:rgb(15, 17, 21);">更容易调试</font>
* <font style="color:rgb(15, 17, 21);">符合 Flutter 的声明式UI理念</font>
* <font style="color:rgb(15, 17, 21);">便于状态管理</font>

***

## <font style="color:rgb(15, 17, 21);">🛠️</font><font style="color:rgb(15, 17, 21);"> 代码示例</font>

### <font style="color:rgb(15, 17, 21);">1. 受控组件示例</font>

```dart
/// 受控输入框组件
class ControlledTextField extends StatelessWidget {
  final String value;
  final ValueChanged<String> onChanged;

  const ControlledTextField({
    super.key,
    required this.value,
    required this.onChanged,
  });

  @override
  Widget build(BuildContext context) {
    return TextField(
      controller: TextEditingController(text: value),
      onChanged: onChanged,
    );
  }
}

/// 使用受控组件
class ParentWidget extends StatefulWidget {
  @override
  State<ParentWidget> createState() => _ParentWidgetState();
}

class _ParentWidgetState extends State<ParentWidget> {
  String _inputValue = ''; // 状态在父组件

  @override
  Widget build(BuildContext context) {
    return Column(
      children: [
        // 数据流向：父组件 → 子组件
        ControlledTextField(
          value: _inputValue, // 数据从父组件传入
          onChanged: (newValue) {
            setState(() {
              _inputValue = newValue; // 回调通知父组件更新状态
            });
          },
        ),
        Text('当前值: $_inputValue'), // 实时显示，数据一致
      ],
    );
  }
}
```

### <font style="color:rgb(15, 17, 21);">2. 非受控组件示例</font>

```dart
/// 非受控输入框组件
class UncontrolledTextField extends StatefulWidget {
  final String initialValue;
  final ValueChanged<String>? onChanged;

  const UncontrolledTextField({
    super.key,
    this.initialValue = '',
    this.onChanged,
  });

  @override
  State<UncontrolledTextField> createState() => _UncontrolledTextFieldState();
}

class _UncontrolledTextFieldState extends State<UncontrolledTextField> {
  late TextEditingController _controller; // 状态在组件内部

  @override
  void initState() {
    super.initState();
    _controller = TextEditingController(text: widget.initialValue);
  }

  @override
  Widget build(BuildContext context) {
    return TextField(
      controller: _controller,
      onChanged: (value) {
        widget.onChanged?.call(value); // 可选的回调
      },
    );
  }

  @override
  void dispose() {
    _controller.dispose();
    super.dispose();
  }
}

/// 使用非受控组件
class ParentWidget extends StatefulWidget {
  @override
  State<ParentWidget> createState() => _ParentWidgetState();
}

class _ParentWidgetState extends State<ParentWidget> {
  String _displayValue = '';

  @override
  Widget build(BuildContext context) {
    return Column(
      children: [
        UncontrolledTextField(
          initialValue: '初始值',
          onChanged: (value) {
            setState(() {
              _displayValue = value; // 只能通过回调获取值
            });
          },
        ),
        Text('显示值: $_displayValue'), // 可能不同步！

        ElevatedButton(
          onPressed: () {
            // 问题：无法直接获取输入框的当前值
            // _displayValue 可能不是最新的
          },
          child: Text('提交'),
        ),
      ],
    );
  }
}
```

## <font style="color:rgb(15, 17, 21);">🔄</font><font style="color:rgb(15, 17, 21);"> 数据流对比</font>

### <font style="color:rgb(15, 17, 21);">受控组件数据流</font>

```dart
父组件状态 → value属性 → 子组件显示 → 用户操作 → onChanged回调 → 更新父组件状态
  ↓
  数据始终同步
```

### <font style="color:rgb(15, 17, 21);">非受控组件数据流</font>

```plain
初始值 → 子组件内部状态 → 用户操作 → 内部状态变化 → 可选回调通知父组件
     ↓
数据可能不同步
```

## <font style="color:rgb(15, 17, 21);">🎯</font><font style="color:rgb(15, 17, 21);"> 实际场景示例</font>

### <font style="color:rgb(15, 17, 21);">场景：表单提交</font>

#### <font style="color:rgb(15, 17, 21);">受控组件方式（推荐）</font>

```dart
class UserForm extends StatefulWidget {
  @override
  State<UserForm> createState() => _UserFormState();
}

class _UserFormState extends State<UserForm> {
  // 所有状态在父组件统一管理
  String _name = '';
  String _email = '';
  bool _subscribe = false;

  void _submitForm() {
    // 直接使用状态，数据保证是最新的
    final userData = {
      'name': _name,
      'email': _email,
      'subscribe': _subscribe,
    };
    print('提交数据: $userData');
  }

  @override
  Widget build(BuildContext context) {
    return Column(
      children: [
        TextFieldFormItem(
          label: '姓名',
          value: _name,
          onChanged: (value) => setState(() => _name = value),
        ),
        TextFieldFormItem(
          label: '邮箱', 
          value: _email,
          onChanged: (value) => setState(() => _email = value),
        ),
        SwitchFormItem(
          label: '订阅通知',
          value: _subscribe,
          onChanged: (value) => setState(() => _subscribe = value),
        ),
        ElevatedButton(
          onPressed: _submitForm,
          child: Text('提交'),
        ),
      ],
    );
  }
}
```

#### <font style="color:rgb(15, 17, 21);">非受控组件方式（不推荐）</font>

```dart
class UserForm extends StatefulWidget {
  @override
  State<UserForm> createState() => _UserFormState();
}

class _UserFormState extends State<UserForm> {
  // 需要维护两套状态
  String _name = '';
  String _email = '';
  bool _subscribe = false;

  // 还需要引用子组件来获取值
  final _nameFieldKey = GlobalKey<_UncontrolledTextFieldState>();
  final _emailFieldKey = GlobalKey<_UncontrolledTextFieldState>();
  final _subscribeSwitchKey = GlobalKey<_UncontrolledSwitchState>();

  void _submitForm() {
    // 需要通过各种方式获取子组件的状态
    final name = _nameFieldKey.currentState?._controller.text ?? '';
    final email = _emailFieldKey.currentState?._controller.text ?? '';
    final subscribe = _subscribeSwitchKey.currentState?._value ?? false;

    final userData = {
      'name': name,
      'email': email, 
      'subscribe': subscribe,
    };
    print('提交数据: $userData');
  }

  @override
  Widget build(BuildContext context) {
    return Column(
      children: [
        UncontrolledTextField(
          key: _nameFieldKey,
          initialValue: _name,
          onChanged: (value) => setState(() => _name = value),
        ),
        // ... 其他类似
        ElevatedButton(
          onPressed: _submitForm,
          child: Text('提交'),
        ),
      ],
    );
  }
}
```

**<font style="color:rgb(15, 17, 21);">建议</font>**<font style="color:rgb(15, 17, 21);">：在 Flutter 开发中，除非有特殊需求，否则</font>**<font style="color:rgb(15, 17, 21);">始终使用受控组件</font>**<font style="color:rgb(15, 17, 21);">。这样可以让你的应用数据流更清晰、更易于维护和调试。</font>

***

## <font style="color:rgb(15, 17, 21);">🚀</font><font style="color:rgb(15, 17, 21);"> 最佳实践</font>

1. **<font style="color:rgb(15, 17, 21);">状态提升</font>**<font style="color:rgb(15, 17, 21);">：将状态提升到最近的共同祖先</font>
2. **<font style="color:rgb(15, 17, 21);">单一数据源</font>**<font style="color:rgb(15, 17, 21);">：每个状态只有一个"真相来源"</font>
3. **<font style="color:rgb(15, 17, 21);">不可变数据</font>**<font style="color:rgb(15, 17, 21);">：使用</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">setState</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">创建新状态而不是修改旧状态</font>
4. **<font style="color:rgb(15, 17, 21);">明确的数据流</font>**<font style="color:rgb(15, 17, 21);">：数据向下流动，事件向上传递</font>

<font style="color:rgb(15, 17, 21);">记住这个口诀：</font>**<u><font style="color:rgb(15, 17, 21);">"状态上提，数据下传，事件回调"</font></u>**
