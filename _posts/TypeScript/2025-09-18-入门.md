### 1. TypeScript静态类型

#### 1.1 如何定义静态类型

```typescript
let age: number = 18;
let name: string = '周杰伦';
```

#### 1.2 自定义静态类型

```typescript
interface singer {
  name: string;
  age: number;
}
const jieLun: singer = {
  name: '周杰伦',
  age: 18
}
console.log(jieLun); //{ name: '周杰伦', age: 18 }
```

### 2. TypeScript基础静态类型和对象类型

#### 2.1 基础静态类型

```typescript
let age: number = 18;
let name: string = '周杰伦';
```

> <font style="color:#FF502C;background-color:#FFF5F5;">null</font><font style="color:#777777;">、</font><font style="color:#FF502C;background-color:#FFF5F5;">undefinde</font><font style="color:#777777;">、</font><font style="color:#FF502C;background-color:#FFF5F5;">symbol</font><font style="color:#777777;">、</font><font style="color:#FF502C;background-color:#FFF5F5;">boolean</font><font style="color:#777777;">、</font><font style="color:#FF502C;background-color:#FFF5F5;">void  </font><font style="color:#777777;">这些都是最常用的基础数据类型</font>

```typescript
//对象类型
```

### 3.Typescript中的类型注解和类型判断

#### 3.1 type annotation 类型注解

```typescript
let count: number;
count = 123;
```

<font style="color:#777777;">这段代码就是类型注解，意思是显示的告诉代码，我们的</font><font style="color:#FF502C;background-color:#FFF5F5;">count</font><font style="color:#777777;">变量就是一个数字类型，这就叫做</font><font style="color:#FF502C;background-color:#FFF5F5;">类型注解</font><font style="color:#777777;">。</font>

#### 3.2 type inferrence 类型判断

```typescript
let countInference = 123;
```

<font style="color:#777777;">这时候我并没有显示的告诉你变量</font><font style="color:#FF502C;background-color:#FFF5F5;">countInference</font><font style="color:#777777;">是一个数字类型，但是如果你把鼠标放到变量上时，你会发现 TypeScript 自动把变量注释为了</font><font style="color:#FF502C;background-color:#FFF5F5;">number</font><font style="color:#777777;">（数字）类型，也就是说它是有某种推断能力的，通过你的代码 TS 会自动的去尝试分析变量的类型。</font>

#### 3.3 工作中使用

* <font style="color:#777777;">如果</font><font style="color:#FF502C;background-color:#FFF5F5;">TS</font><font style="color:#777777;">能够自动分析变量类型， 我们就什么也不需要做了</font>
* <font style="color:#777777;">如果</font><font style="color:#FF502C;background-color:#FFF5F5;">TS</font><font style="color:#777777;">无法分析变量类型的话， 我们就需要使用类型注解</font>

<font style="color:#777777;">先来看一个不用写类型注解的例子：</font>

```typescript
const one = 1;
const two = 2;
const three = one + two;
```

<font style="color:#777777;">再来看一个用写类型注解的例子：</font>

```typescript
function getTotal(one, two) {
  return one + two;
}

const total = getTotal(1, 2);
```

<font style="color:#777777;">这种形式，就需要用到类型注释了，因为这里的 </font><font style="color:#FF502C;background-color:#FFF5F5;">one </font><font style="color:#777777;">和 </font><font style="color:#FF502C;background-color:#FFF5F5;">two </font><font style="color:#777777;">会显示为 </font><code><font style="color:#FF502C;background-color:#FFF5F5;">any</font></code><font style="color:#FF502C;background-color:#FFF5F5;"> </font><font style="color:#777777;">类型。这时候如果你传字符串，你的业务逻辑就是错误的，所以你必须加一个</font><font style="color:#FF502C;background-color:#FFF5F5;">类型注解</font><font style="color:#777777;">，把上面的代码写成下面的样子。</font>

```typescript
function getTotal(one: number, two: number) {
  return one + two;
}

const total = getTotal(1, 2);
```

<font style="color:#777777;">这里有的一个问题是，为什么 </font><font style="color:#FF502C;background-color:#FFF5F5;">total </font><font style="color:#777777;">这个变量不需要加类型注解，因为当 </font><font style="color:#FF502C;background-color:#FFF5F5;">one </font><font style="color:#777777;">和 </font><font style="color:#FF502C;background-color:#FFF5F5;">two </font><font style="color:#777777;">两个变量加上注解后，</font><code><font style="color:#777777;">TypeScript</font></code><font style="color:#777777;"> 就可以自动通过类型推断，分析出变量的类型。</font>

<font style="color:#777777;">当然 </font><code><font style="color:#777777;">TypeScript</font></code><font style="color:#777777;"> 也可以推断出对象中属性的类型：</font>

```typescript
const singers = {
  name: "周杰伦",
  age: 18,
};
```

<font style="color:#777777;">写完后你把鼠标放在 </font><font style="color:#F5222D;">singers </font><font style="color:#777777;">对象上面，就会提示出他里边的属性，这表明 </font><code><font style="color:#777777;">TypeScript</font></code><font style="color:#777777;"> 也分析出了对象的属性的类型。</font>

> <font style="color:#777777;">在写 TypeScript 代码的一个重要宗旨就是每个变量，每个对象的属性类型都应该是固定的，如果你推断就让它推断，推断不出来的时候你要进行注释。</font>

### 4. Typescript函数参数和返回类型定义

#### 4.1 简单的类型定义

定义返回类型为 `number`

```typescript
function getTotal(one: number, two: number): number {
  return one + two;
}

const total = getTotal(1, 2);
```

#### 4.2 函数无返回值时的定义

<font style="color:#777777;">有时候函数是没有返回值的，比如现在定义一个 </font><font style="color:#FF502C;background-color:#FFF5F5;">sayHello </font><font style="color:#777777;">的函数，这个函数只是简单的 </font><font style="color:#FF502C;background-color:#FFF5F5;">terminal </font><font style="color:#777777;">打印，并没有返回值。我们就可以给他一个类型注解 </font><code><font style="color:#FF502C;background-color:#FFF5F5;">void</font></code><font style="color:#FF502C;background-color:#FFF5F5;"> </font><font style="color:#777777;">，代表没有任何返回值。</font>

```typescript
function sayHello(): void {
  console.log("hello world");
}
```

#### 4.3 never返回值类型

<font style="color:#777777;">如果一个函数是永远也执行不完的，就可以定义返回值为 </font><code><font style="color:#FF502C;background-color:#FFF5F5;">never</font></code><font style="color:#777777;">。</font>

<font style="color:#777777;">比如执行的时候，抛出了异常，这时候就无法执行完了：</font>

```typescript
function errorFuntion(): never {
  throw new Error();
  console.log("Hello World");
}
```

<font style="color:#777777;">还有一种是</font><font style="color:#F5222D;"> 死循环</font><font style="color:#777777;">，这样也运行不完，比如下面的代码：</font>

```typescript
function forNever(): never {
  while (true) {}
  console.log("Hello JSPang");
}
```

#### 4.4 函数参数为对象（解构）时

<font style="color:#777777;">当一个函数的参数是对象时，我们如何定义参数对象的属性类型。</font>

```typescript
function add({ one, two }: { one: number, two: number }): number {
  return one + two;
}

const three = add({ one: 1, two: 2 });
```

<font style="color:#777777;">如果参数是对象，并且里边只有一个属性时：</font>

```typescript
function getNumber({ one }: { one: number }): number {
  return one;
}

const one = getNumber({ one: 1 });
```

### 5. Typescript中数组类型的定义

#### 5.1 一般数组类型的定义

<font style="color:#777777;">定义一个最简单的数组类型，比如数字类型：</font>

```typescript
const numberArr = [1, 2, 3];
```

<font style="color:#777777;">这时候你把鼠标放在 </font><font style="color:#FF502C;background-color:#FFF5F5;">numberArr </font><font style="color:#777777;">上面可以看出，这个数组的类型就是 </font><code><font style="color:#777777;">number</font></code><font style="color:#777777;"> 类型。这是 </font><code><font style="color:#777777;">TypeScript</font></code><font style="color:#777777;"> 通过</font><font style="color:#FF502C;background-color:#FFF5F5;">类型推断</font><font style="color:#777777;">自己推断出来的。 如果你要显示的注解，也非常简单，可以写成下面的形式。</font>

```typescript
const numberArr: number[] = [1, 2, 3];
```

<font style="color:#777777;">数组各项是字符串：</font>

```typescript
const stringArr: string[] = ["a", "b", "c"];
```

<font style="color:#777777;">定义任意类型的数组，比如是</font><font style="color:#FF502C;background-color:#FFF5F5;">undefined</font><font style="color:#777777;">：</font>

```typescript
const undefinedArr: undefined[] = [undefined, undefined];
```

<font style="color:#777777;">数组中有多种类型，比如既有数字类型，又有字符串的时候，只要加个 </font><font style="color:#FF502C;background-color:#FFF5F5;">()</font><font style="color:#777777;">，然后在里边加上 </font><font style="color:#FF502C;background-color:#FFF5F5;">| </font><font style="color:#777777;">就可以了：</font>

```typescript
const arr: (number | string)[] = [1, "string", 2];
```

#### 5.2 数组中对象类型的定义

定义一个歌手的数组：

```typescript
const singer: { name: string, age: Number }[] = [
  { name: "周杰伦", age: 18 },
  { name: "林俊杰", age: 28 },
];
```

<font style="color:#777777;">这种形式看起来比较麻烦，而且如果有同样类型的数组，写代码也比较麻烦，TypeScript 为我们准备了一个概念，叫做 </font><font style="color:#FF502C;background-color:#FFF5F5;">类型别名</font><font style="color:#777777;">(type alias)。</font>

<font style="color:#777777;">比如刚才的代码，就可以定义一个</font><font style="color:#FF502C;background-color:#FFF5F5;">类型别名</font><font style="color:#777777;">，定义别名的时候要以</font><code><font style="color:#FF502C;background-color:#FFF5F5;">type</font></code><font style="color:#777777;">关键字开始。现在定义一个</font><font style="color:#FF502C;background-color:#FFF5F5;">Men</font><font style="color:#777777;">的别名。</font>

```typescript
type Men = { name: string, age: Number };
```

<font style="color:#777777;">有了这样的类型别名以后哦，就可以把上面的代码改为下面的形式了。</font>

```typescript
type Men = { name: string, age: Number };

const singer: Men[] = [
  { name: "周杰伦", age: 18 },
  { name: "林俊杰", age: 28 },
];
```

<font style="color:#777777;">这样定义是完全起作用的，比如我们下面在对象里再加入一个属性，这时候编译器就会直接给我们报错了。</font>

<font style="color:#777777;">用类进行定义也是可以的，比如我们定义一个</font><font style="color:#FF502C;background-color:#FFF5F5;">Gentle</font><font style="color:#777777;">的类，然后用这个类来限制数组的类型也是可以的：</font>

```typescript
class Gentle {
  name: string;
  age: number;
}

const singer: Gentle[] = [
  { name: "周杰伦", age: 18 },
  { name: "林俊杰", age: 28 },
];
```

### 6. Typescript中<font style="color:#F5222D;">元组</font>的使用和类型约束

#### 6.1 元组的基本使用

<font style="color:#777777;">我们先来看一个数组和这个数组注解的缺点：</font>

```typescript
const singer = ["周杰伦", "林俊杰", 28];
```

<font style="color:#777777;">这时候把鼠标放到 </font><font style="color:#F5222D;background-color:#FFE8E6;">singer</font><font style="background-color:#FFE8E6;"> </font><font style="color:#777777;">变量上面，可以看出推断出来的类型。我们就用类型注解的形式给他作一个注解，代码如下：</font>

```typescript
const singer: (string | number)[] = ["周杰伦", "林俊杰", 28];
```

<font style="color:#777777;">这时候你已经增加了代码注解，但是这并不能很好的限制，比如我们把代码改成下面的样子，</font><font style="color:#FF502C;background-color:#FFF5F5;">TypeScript</font><font style="color:#777777;">依然不会报错。</font>

```typescript
const singer: (string | number)[] = ["周杰伦", 28, "林俊杰"];
```

<font style="color:#777777;">我们只是简单的把数组中的位置调换了一下，但是</font><font style="color:#FF502C;background-color:#FFF5F5;">TypeScript</font><font style="color:#777777;">并不能发现问题，这时候我们需要一个更强大的类型，来解决这个问题，这就是</font>**<font style="color:#F5222D;">元组</font>**<font style="color:#777777;">。</font>

<font style="color:#777777;">元组和数组类似，但是类型注解时会不一样。</font>

```typescript
const singer: [string, string, number] = ["周杰伦", "林俊杰", 28];
```

<font style="color:#777777;">这时候我们就把数组中的每个元素类型的位置给固定住了，这就叫做</font>**<font style="color:#F5222D;">元组</font>**<font style="color:#777777;">。</font>

#### 6.2 元组的使用

<font style="color:#777777;">目前我的工作中不经常使用元组，因为如果要使用元组，完全可以使用对象的形式来代替，但是如果你维护老系统，你会发现有一种数据源时</font><font style="color:#FF502C;background-color:#FFF5F5;">CSV</font><font style="color:#777777;">,这种文件提供的就是用逗号隔开的，如果要严谨的编程就需要用到元组了。例如我们有这样一组由</font><font style="color:#FF502C;background-color:#FFF5F5;">CSV</font><font style="color:#777777;">提供的（注意这里只是模拟数据）。</font>

```typescript
"zhoujie", "teacher", 28;
"xusong", "teacher", 18;
"zhaolei", "teacher", 25;
```

<font style="color:#777777;">如果数据源得到的数据时这样的，你就可以使用元组了：</font>

```typescript
const singer: [string, string, number][] = [
  ["zhoujie", "teacher", 28],
  ["xusong", "teacher", 18],
  ["zhaolei", "teacher", 25],
];
```

### 7. Typescript中的 interface 接口

#### 7.1 interface 接口初步了解

<font style="color:#777777;">现在我们要作一个简历的自动筛选程序，年龄小于 25 岁，胸围大于 90 公分的：</font>

```typescript
const screenResume = (name: string, age: number, bust: number) => {
  age < 24 && bust >= 90 && console.log(name + "进入面试");
  age > 24 || (bust < 90 && console.log(name + "你被淘汰"));
};

screenResume("杰伦", 18, 94);
```

<font style="color:#777777;">这时候又增加了需求，必须能看到这些人的简历信息，于是又写了下面这个方法：</font>

```typescript
const getResume = (name: string, age: number, bust: number) => {
  console.log(name + "年龄是：" + age);
  console.log(name + "胸围是：" + bust);
};

getResume("杰伦", 18, 94);
```

<font style="color:#777777;">这时候问题来了，程序开发中一直强调“代码重用”，两个方法用的类型注解一样，需要作个统一的约束。上面的 </font><font style="color:#FF502C;background-color:#FFF5F5;">类型别名 </font><font style="color:#777777;">可以解决代码重复的问题，现在看一个更常用的语法 </font><font style="color:#FF502C;background-color:#FFF5F5;">接口</font><font style="color:#777777;">（Interface）.</font>

<font style="color:#777777;">我们可以把这两个重复的类型注解，定义成统一的接口。代码如下：</font>

```typescript
interface Men {
  name: string;
  age: number;
  bust: number;
}

const screenResume = (men: Men) => {
  men.age < 24 && men.bust >= 90 && console.log(men.name + "进入面试");
  men.age > 24 || (men.bust < 90 && console.log(men.name + "你被淘汰"));
};

const getResume = (men: Men) => {
  console.log(men.name + "年龄是：" + men.age);
  console.log(men.name + "胸围是：" + men.bust);
};

const men = {
  name: "杰伦",
  age: 18,
  bust: 94,
};

screenResume(men);
getResume(men);
```

#### 7.2 接口和类型别名的区别

> 类型别名可以直接给类型，比如 <font style="color:#F5222D;">string</font>，而<font style="color:#F5222D;">接口必须代表对象</font>。

<font style="color:#777777;">比如我们的 </font><font style="color:#FF502C;background-color:#FFF5F5;">类型别名 </font><font style="color:#777777;">可以写出下面的代码：</font>

```typescript
type Men = stirng;
```

<font style="background-color:#FFFFFF;">但是接口就不能这样写，它必须代表的是一个对象，也就是说，你初始化 </font><font style="color:#F5222D;background-color:#FFFFFF;">men</font><font style="background-color:#FFFFFF;"> 的时候，必须写出下面的形式：</font>

```typescript
const men = {
  name: "杰伦",
  age: 18,
  bust: 94,
};
```

#### 7.3 接口非必选值的定义

<font style="color:#777777;">比如这时候又有了新的要求，要求尽量能看到腰围，但是不作强制要求，就是可选值。在 </font>**<font style="color:#FF502C;background-color:#FFF5F5;">:</font>**<font style="color:#FF502C;background-color:#FFF5F5;"> </font><font style="color:#777777;">号前加一个 </font>**<font style="color:#FF502C;background-color:#FFF5F5;">?</font>**

<font style="color:#777777;">比如把 </font><font style="color:#FF502C;background-color:#FFF5F5;">Men </font><font style="color:#777777;">的接口写成这样：</font>

```typescript
interface Men {
  name: string;
  age: number;
  bust: number;
  waistline?: number;
}
```

<font style="color:#777777;">然后我们再修改一下</font><font style="color:#FF502C;background-color:#FFF5F5;">getResume</font><font style="color:#777777;">方法，写成这样：</font>

```typescript
const getResume = (men: Men) => {
  console.log(men.name + "年龄是：" + men.age);
  console.log(men.name + "胸围是：" + men.bust);
  men.waistline && console.log(men.name + "腰围是：" + men.waistline);
};
```

<font style="color:#777777;background-color:#FFFFFF;">这时候在定义</font><font style="color:#FF502C;background-color:#FFFFFF;"> men </font><font style="color:#777777;">对象的时候，就可以写 </font><font style="color:#FF502C;background-color:#FFF5F5;">saistline</font><font style="color:#777777;">（腰围），也可以不写了。</font>

#### 7.4 允许加入任意值

```typescript
interface Men {
  name: string;
  age: number;
  bust: number;
  waistline?: number;
  [propname: string]: any;
}
```

<font style="color:#777777;">这个的意思是，属性的名字是字符串类型，属性的值可以是任何类型。</font>

<font style="color:#777777;">这时候我们在对象里给一个性别,代码如下：</font>

```typescript
const men = {
  name: "杰伦",
  age: 18,
  bust: 94,
  waistline: 21,
  sex: "男",
};

const getResume = (men: Men) => {
  console.log(men.name + "年龄是：" + men.age);
  console.log(men.name + "胸围是：" + men.bust);
  men.waistline && console.log(men.name + "腰围是：" + men.waistline);
  men.sex && console.log(men.name + "性别是：" + men.sex);
};
```

#### 7.5 接口里的方法

<font style="color:#777777;">接口里不仅可以存属性，还可以存方法，比如这时候有个</font><font style="color:#FF502C;background-color:#FFF5F5;">say()</font><font style="color:#777777;">方法，返回值是</font><font style="color:#FF502C;background-color:#FFF5F5;">string</font><font style="color:#777777;">类型：</font>

```typescript
interface Men {
  name: string;
  age: number;
  bust: number;
  waistline?: number;
  [propname: string]: any;
  say(): string;
}

const men = {
  name: "杰伦",
  age: 18,
  bust: 94,
  waistline: 21,
  sex: "男",
  say() {
    return "天青色等烟雨，而我在等你！";
  },
};
```

#### 7.6 接口和类的约束

<font style="color:#777777;">类可以和接口很好的结合，我们先来看一个例子：</font>

```typescript
class singer implements Men {}
```

<font style="color:#595959;">先写一个简单的类：</font>

<font style="color:#8C8C8C;">这就相当于下面的这段代码：</font>


> <code><font style="color:#F5222D;background-color:rgb(255, 245, 245);">public</font></code><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);"> </font><font style="color:rgb(102, 102, 102);background-color:rgb(248, 248, 248);">从英文字面的解释就是</font><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">公共的</font><font style="color:rgb(102, 102, 102);background-color:rgb(248, 248, 248);">，在程序里的意思就是</font><font style="color:#F5222D;background-color:rgb(248, 248, 248);">允许在类的</font>**<u><font style="color:#F5222D;background-color:rgb(248, 248, 248);">内部和外部</font></u>**<font style="color:#F5222D;background-color:rgb(248, 248, 248);">被调用</font>

<font style="color:rgb(119, 119, 119);">比如我们在类内调用，我们在写一个 </font><code><font style="color:#8C8C8C;background-color:rgb(255, 245, 245);">sayHello</font></code><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);"> </font><font style="color:rgb(119, 119, 119);">的方法：</font>

<font style="color:rgb(119, 119, 119);">这是的 </font><code><font style="color:#595959;background-color:rgb(255, 245, 245);">this.name</font></code><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);"> </font><font style="color:rgb(119, 119, 119);">就是类的</font><font style="color:#F5222D;">内部调用</font><font style="color:rgb(119, 119, 119);">；下面从注释横线下，全部是类的外部：</font>


### 12 抽象类

<font style="color:rgb(119, 119, 119);">抽象类的关键词是 </font><code><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">abstract</font></code><font style="color:rgb(119, 119, 119);">，里边的抽象方法也是 </font><code><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">abstract</font></code><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);"> </font><font style="color:rgb(119, 119, 119);">开头的，现在我们就写一个</font><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);"> Person </font><font style="color:rgb(119, 119, 119);">的抽象类。</font>

### 16 联合类型和类型保护

<font style="color:rgb(119, 119, 119);">所谓联合类型，可以认为一个变量可能有两种或两种以上的类型。需要注意的是，</font><font style="color:#F5222D;">只有联合类型存在的情况下，才需要类型保护</font><font style="color:rgb(119, 119, 119);">。普通的类型注解，并不需要我们这种特殊操作。</font>

#### 16.1 简单的联合类型

```javascript
let myFavoriteNumber: string | number;
myFavoriteNumber = 'seven';
myFavoriteNumber = 7;
```

<font style="color:#F5222D;">联合类型使用 | 分隔每个类型。</font>

<font style="color:rgb(51, 51, 51);">这里的 let myFavoriteNumber: string | number 的含义是，允许 myFavoriteNumber 的类型是 string 或者 number，但是不能是其他类型。</font>

#### 16.2 联合类型展示

例：<font style="color:rgb(119, 119, 119);">声明两个接口</font><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);"> qianDuan</font><font style="color:rgb(119, 119, 119);">(前端工程师接口和 </font><font style="color:#F5222D;">houDuan</font><font style="color:rgb(119, 119, 119);">(后端工程师）接口，然后在写一个 </font><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">judgeWho</font><font style="color:rgb(119, 119, 119);">(判断是谁)的方法，里边传入一个 </font><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">animal</font><font style="color:rgb(119, 119, 119);">(任意值)，这时候可以能是</font><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">qianDuan</font><font style="color:rgb(119, 119, 119);">，也可能是</font><font style="color:#F5222D;">houDuan</font><font style="color:rgb(119, 119, 119);">。所以我们使用了联合类型，关键符号是</font><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">|</font><font style="color:rgb(119, 119, 119);">(竖线)。</font>

```typescript
interface qianDuan {
  ps: boolean;
  goJs: () => {};
}

interface houDuan {
  ps: boolean;
  goPhp: () => {};
}

function judgeWho(animal: qianDuan | houDuan) {}
```

<font style="color:rgb(119, 119, 119);">但这时候问题来了，如果我直接写一个这样的方法，就会报错，因为</font><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">judgeWho</font><font style="color:rgb(119, 119, 119);">不能准确的判断联合类型具体的实例是什么。</font>

```typescript
function judgeWho(animal: Waiter | Teacher) {
  animal.goJs(); //报错
}
```

<font style="color:rgb(119, 119, 119);">这时候就需要再引出一个概念叫做</font><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">类型保护</font><font style="background-color:rgb(255, 245, 245);"></font>

#### 16.3 类型保护 - 类型断言

<font style="color:rgb(119, 119, 119);">类型断言就是通过断言的方式确定传递过来的准确值，比如上面的程序，如果会 </font><font style="color:#F5222D;">ps</font><font style="color:rgb(119, 119, 119);">，说明他就是前端工程师，这时候就可以通过断言</font><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">animal as qianDuan</font><font style="color:rgb(119, 119, 119);">，然后直接调用</font><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);"> goJs() </font><font style="color:rgb(119, 119, 119);">方法，程序就不再报错了。同样如果不会</font><font style="color:#F5222D;">ps</font><font style="color:rgb(119, 119, 119);">，说明就是后端工程师，这时候调用 </font><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">goPhp()</font><font style="color:rgb(119, 119, 119);">方法，就不会报错了。这就是通过断言的方式进行类型保护。也是最常见的一种类型保护形式。</font>

```typescript
interface qianDuan {
  ps: boolean;
  goJs: () => {};
}

interface houDuan {
  ps: boolean;
  goPhp: () => {};
}

function judgeWho(animal: qianDuan | houDuan) {
  if (animal.ps) {
    (animal as qianDuan).goJs();
  }else{
    (animal as houDuan).goPhp();
  }
}
```

#### 16.4 类型保护 - in 方法

<font style="color:rgb(119, 119, 119);">我们还经常使用 </font><code><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">in</font></code><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);"> </font><font style="color:rgb(119, 119, 119);">语法来作类型保护，比如用 </font><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">if </font><font style="color:rgb(119, 119, 119);">来判断 </font><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">animal </font><font style="color:rgb(119, 119, 119);">里有没有 </font><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">goJs() </font><font style="color:rgb(119, 119, 119);">方法：</font>

```typescript
function judgeWhoTwo(animal: qianDuan | houDuan) {
  if ("goJs" in animal) {
    animal.goJs();
  } else {
    animal.goPhp();
  }
}
```

<font style="color:rgb(119, 119, 119);">这里的</font><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">else</font><font style="color:rgb(119, 119, 119);">部分能够自动判断，得益于</font><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">TypeScript</font><font style="color:rgb(119, 119, 119);">的自动判断。</font>

#### 16.5 类型保护 - typeof 方法

<font style="color:rgb(119, 119, 119);">写一个新的 </font><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">add </font><font style="color:rgb(119, 119, 119);">方法，方法接收两个参数，这两个参数可以是数字</font><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">number</font><font style="color:rgb(119, 119, 119);">也可以是字符串</font><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">string</font><font style="color:rgb(119, 119, 119);">，如果我们不做任何的类型保护，只是相加，这时候就会报错。代码如下：</font>

```typescript
function add(first: string | number, second: string | number) {
  return first + second;
}
```

<font style="color:rgb(119, 119, 119);">解决这个问题，就可以直接使用</font><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">typeof</font><font style="color:rgb(119, 119, 119);">来进行解决：</font>

```typescript
function add(first: string | number, second: string | number) {
  if (typeof first === "string" || typeof second === "string") {
    return `${first}${second}`;
  }
  return first + second;
}
```

#### 16.6 类型保护 - instanceof 方法

<font style="color:rgb(119, 119, 119);">现在要作类型保护的是一个对象，这时候就可以使用</font><code><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">instanceof</font></code><font style="color:rgb(119, 119, 119);">语法来作。现在先写一个</font><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">NumberObj</font><font style="color:rgb(119, 119, 119);">的类：</font>

```typescript
class NumberObj {
  count: number;
}
```

<font style="color:rgb(119, 119, 119);">然后我们再写一个</font><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">addObj</font><font style="color:rgb(119, 119, 119);">的方法，这时候传递过来的参数，可以是任意的</font><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">object</font><font style="color:rgb(119, 119, 119);">,也可以是</font><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">NumberObj</font><font style="color:rgb(119, 119, 119);">的实例，然后我们返回相加值，当然不进行类型保护，这段代码一定是错误的：</font>

```typescript
function addObj(first: object | NumberObj, second: object | NumberObj) {
  return first.count + second.count;
}
```

<font style="color:rgb(119, 119, 119);">报错不要紧，直接使用 </font><code><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">instanceof</font></code><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);"> </font><font style="color:rgb(119, 119, 119);">语法进行判断一下，就可以解决问题：</font>

```typescript
function addObj(first: object | NumberObj, second: object | NumberObj) {
  if (first instanceof NumberObj && second instanceof NumberObj) {
    return first.count + second.count;
  }
  return 0;
}
```

> <font style="color:#F5222D;">instanceof 只能用在类上</font>

### 17 Enum 枚举类型

#### 17.1 案例

比如我现在写个程序随机选一个景点去旅游：

<font style="color:#595959;">初级程序员写法：</font>

```typescript
function getTravel(status: number) {
  if (status === 0) {
    return "北京";
  } else if (status === 1) {
    return "上海";
  } else if (status === 2) {
    return "杭州";
  }
}
const result = getTravel(0);
console.log(`我要去${result}`); //我要去北京
```

<font style="color:rgb(119, 119, 119);">中级程序员写法：</font>

```typescript
const Status = {
  BEIJING: 0,
  SHANGHAI: 1,
  HANGZHOU: 2,
};

function getTravel(status: any) {
  if (status === Status.BEIJING) {
    return "北京";
  } else if (status === Status.SHANGHAI) {
    return "上海";
  } else if (status === Status.HANGZHOU) {
    return "杭州";
  }
}

const result = getTravel(Status.BEIJING);

console.log(`我要去${result}`); //我要去北京
```

<font style="color:rgb(119, 119, 119);">高级程序员写法：</font>

```typescript
enum Status {
  BEIJING,
  SHANGHAI,
  HANGZHOU,
}

function getServe(status: any) {
  if (status === Status.BEIJING) {
    return "北京";
  } else if (status === Status.SHANGHAI) {
    return "上海";
  } else if (status === Status.HANGZHOU) {
    return "杭州";
  }
}

const result = getServe(Status.BEIJING);

console.log(`我要去${result}`); //我要去北京
```

#### 17.2 枚举类型的对应值

上面的你调用时传一个 0，也会输出 `我要去北京`。

```typescript
const result = getServe(0);
console.log(`我要去${result}`); //我要去北京
```

<font style="color:rgb(119, 119, 119);">这是因为枚举类型是有对应的数字值的，默认是从 0 开始的。我们直接用</font><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">console.log()</font><font style="color:rgb(119, 119, 119);">就可以看出来了：</font>

```typescript
console.log(Status.BEIJING); // 0
console.log(Status.SHANGHAI); // 1
console.log(Status.HANGZHOU); // 2
```

<font style="color:rgb(119, 119, 119);">那这时候不想默认从 0 开始，而是想从 1 开始。可以这样写：</font>

```typescript
enum Status {
  BEIJING = 1,
  SHANGHAI,
  HANGZHOU,
}
```

#### 17.3 枚举通过下标反查

<font style="color:rgb(119, 119, 119);">我们这里能打印出枚举的值(也有叫下标的)，那如果我们知道下标后，也可以通过反差的方法，得到枚举的值：</font>

```typescript
console.log(Status.BEIJING, Status[0]);
```
