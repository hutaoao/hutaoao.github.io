# Class的基本语法

### 简介

<font style="color:rgb(13, 20, 30);">JavaScript 语言中，生成实例对象的传统方法是通过构造函数：</font>

```javascript
function Point(x, y) {
  this.x = x;
  this.y = y;
}

Point.prototype.toString = function () {
  return '(' + this.x + ', ' + this.y + ')';
};

var p = new Point(1, 2);
```

等同于：

```javascript
class Point {
  constructor(x, y) {
    this.x = x;
    this.y = y;
  }

  toString() {
    return '(' + this.x + ', ' + this.y + ')';
  }
}
```

> <font style="color:rgb(13, 20, 30);">可以看到里面有一个 </font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">constructor()</font> <font style="color:rgb(13, 20, 30);">方法，这就是</font>**<font style="color:#F5222D;">构造方法</font>**<font style="color:rgb(13, 20, 30);">，而 </font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">this</font> <font style="color:rgb(13, 20, 30);">关键字则代表实例对象。</font>

<font style="color:rgb(13, 20, 30);"></font>

```javascript
class Point {
  // ...
}

typeof Point // "function"
Point === Point.prototype.constructor // true
```

> <font style="color:rgb(13, 20, 30);">上面代码表明，</font>**<font style="color:#F5222D;">类的数据类型就是函数，类本身就指向构造函数。</font>**

**<font style="color:#F5222D;"></font>**

**使用**：`new` 命令

```javascript
class Bar {
  doStuff() {
    console.log('stuff');
  }
}

const b = new Bar();
b.doStuff() // "stuff"
```

<font style="color:rgb(13, 20, 30);">构造函数的</font>**<font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">prototype</font>**<font style="color:rgb(13, 20, 30);">属性，在 ES6 的“类”上面继续存在。事实上，</font>**<font style="color:rgb(13, 20, 30);">类的所有方法都定义在类的</font>****<font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">prototype</font>****<font style="color:rgb(13, 20, 30);">属性上面。</font>**

```javascript
class Point {
  constructor() {
    // ...
  }

  toString() {
    // ...
  }

  toValue() {
    // ...
  }
}

// 等同于

Point.prototype = {
  constructor() {},
  toString() {},
  toValue() {},
};
```

<font style="color:rgb(13, 20, 30);"></font>

<font style="color:rgb(13, 20, 30);">因此，</font>**<font style="color:rgb(13, 20, 30);">在类的实例上面调用方法，其实就是调用原型上的方法</font>**<font style="color:rgb(13, 20, 30);">。</font>

```javascript
class B {}
const b = new B();

b.constructor === B.prototype.constructor // true
```

**<font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">prototype</font>****<font style="color:rgb(13, 20, 30);">对象的</font>****<font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">constructor()</font>\*\*\*\*<font style="color:rgb(13, 20, 30);">属性，直接指向“类”的本身</font>**<font style="color:rgb(13, 20, 30);">，这与 ES5 的行为是一致的。</font>

```javascript
Point.prototype.constructor === Point // true
```

**<font style="color:#F5222D;">类的内部所有定义的方法，都是不可枚举的（non-enumerable）。</font>**

```javascript
class Point {
  constructor(x, y) {
    // ...
  }

  toString() {
    // ...
  }
}

Object.keys(Point.prototype)
// []
Object.getOwnPropertyNames(Point.prototype)
// ["constructor","toString"]
```

> 上面代码中 <font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">toString()</font><font style="color:rgb(13, 20, 30);">方法是</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">Point</font><font style="color:rgb(13, 20, 30);">类内部定义的方法，它是不可枚举的</font>

```javascript
var Point = function (x, y) {
  // ...
};

Point.prototype.toString = function () {
  // ...
};

Object.keys(Point.prototype)
// ["toString"]
Object.getOwnPropertyNames(Point.prototype)
// ["constructor","toString"]
```

> <font style="color:rgb(13, 20, 30);">上面 ES5 的写法，</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">toString()</font><font style="color:rgb(13, 20, 30);">方法就是可枚举的。</font>

<font style="color:rgb(13, 20, 30);"></font>

### constructor 方法

**<font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">constructor()</font>****<font style="color:rgb(13, 20, 30);">方法是类的默认方法，通过</font>****<font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">new</font>\*\*\*\*<font style="color:rgb(13, 20, 30);">命令生成对象实例时，自动调用该方法。</font>**

**<font style="color:rgb(13, 20, 30);">一个类必须有</font>****<font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">constructor()</font>****<font style="color:rgb(13, 20, 30);">方法，如果没有显式定义，一个空的</font>****<font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">constructor()</font>****<font style="color:rgb(13, 20, 30);">方法会被默认添加。</font>**

```javascript
class Point {
}

// 等同于
class Point {
  constructor() {}
}
```

<font style="color:rgb(13, 20, 30);"></font>

> **<font style="color:rgb(13, 20, 30);">类必须使用</font>****<font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">new</font>****<font style="color:rgb(13, 20, 30);">调用</font>**<font style="color:rgb(13, 20, 30);">，否则会报错。这是它跟普通构造函数的一个主要区别，后者不用</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">new</font><font style="color:rgb(13, 20, 30);">也可以执行。</font>

<font style="color:rgb(13, 20, 30);"></font>

### 类的实例

**<font style="color:rgb(13, 20, 30);">与 ES5 一样，实例的属性除非显式定义在其本身（即定义在</font>****<font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">this</font>****<font style="color:rgb(13, 20, 30);">对象上），否则都是定义在原型上（即定义在</font>****<font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">class</font>****<font style="color:rgb(13, 20, 30);">上）。</font>**

```javascript
//定义类
class Point {

  constructor(x, y) {
    this.x = x;
    this.y = y;
  }

  toString() {
    return '(' + this.x + ', ' + this.y + ')';
  }

}

var point = new Point(2, 3);

point.toString() // (2, 3)

point.hasOwnProperty('x') // true
point.hasOwnProperty('y') // true
point.hasOwnProperty('toString') // false
point.__proto__.hasOwnProperty('toString') // true
```

上面代码中<font style="color:rgb(13, 20, 30);">，</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">x</font><font style="color:rgb(13, 20, 30);">和</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">y</font><font style="color:rgb(13, 20, 30);">都是实例对象</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">point</font><font style="color:rgb(13, 20, 30);">自身的属性（因为定义在</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">this</font><font style="color:rgb(13, 20, 30);">对象上），所以</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">hasOwnProperty()</font><font style="color:rgb(13, 20, 30);">方法返回</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">true</font><font style="color:rgb(13, 20, 30);">，而</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">toString()</font><font style="color:rgb(13, 20, 30);">是原型对象的属性（因为定义在</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">Point</font><font style="color:rgb(13, 20, 30);">类上），所以</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">hasOwnProperty()</font><font style="color:rgb(13, 20, 30);">方法返回</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">false</font><font style="color:rgb(13, 20, 30);">。这些都与 ES5 的行为保持一致。</font>

> Object 的 [**hasOwnProperty()**](https://www.yuque.com/hutaoao/blog/fzgqsa) 方法返回一个布尔值，判断对象是否包含特定的自身（非继承）属性。

**<font style="color:#F5222D;">与 ES5 一样，类的所有实例共享一个原型对象。</font>**

```javascript
var p1 = new Point(2,3);
var p2 = new Point(3,2);

p1.__proto__ === p2.__proto__		//true
```
