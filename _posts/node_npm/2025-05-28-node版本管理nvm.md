# node 版本管理 nvm

### <font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">nvm</font><font style="color:rgb(51, 51, 51);"> 与 </font><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">n</font><font style="color:rgb(51, 51, 51);"> 的不同</font>

<font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">nvm</font><font style="color:rgb(51, 51, 51);"> </font><font style="color:rgb(51, 51, 51);">是一个命令，来管理 node.js 版本，对于多个 node.js 版本，它是创建了多个目录来存放 node.js 的代码，实现了完全隔离，即沙箱这种方式。切换版本也是通过修改 node 路径所对应的文件夹目录。</font>

<font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">n</font><font style="color:rgb(51, 51, 51);"> </font><font style="color:rgb(51, 51, 51);">呢，是一个 npm 全局包命令，切换版本是将安装的 node 版本复制到 /usr/local/bin 目录下。多版本之间没有很强的隔离。甚至会出现全局包错误问题。</font>

<font style="color:rgb(51, 51, 51);">因此，我更推荐使用</font><font style="color:rgb(51, 51, 51);"> </font><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">nvm</font><font style="color:rgb(51, 51, 51);"> </font><font style="color:rgb(51, 51, 51);">来做 node.js 的版本管理器。</font>

### <font style="color:rgb(51, 51, 51);">nvm 安装</font>

<font style="color:rgb(119, 119, 119);">之前令人难以选择的是，Windows 用户无法使用 nvm，对于版本管理，只好选择</font><font style="color:rgb(119, 119, 119);"> </font><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">n</font><font style="color:rgb(119, 119, 119);"> </font><font style="color:rgb(119, 119, 119);">了，但现在而言，</font><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">nvm</font><font style="color:rgb(119, 119, 119);"> </font><font style="color:rgb(119, 119, 119);">也是有 Windows 版本的。</font>[传送门](https://github.com/coreybutler/nvm-windows)

<font style="color:rgb(51, 51, 51);">这里简单复制部分，链接：</font>[官方文档安装方式](https://github.com/nvm-sh/nvm#installing-and-updating)

`curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.1/install.sh | bash `

<font style="color:rgb(51, 51, 51);">需要注意的是在执行完安装脚本后，也要检查确认终端文件中是否有这一段代码。(~/.bash\_profile, ~/.zshrc, ~/.profile, or ~/.bashrc)，这个看你的终端使用的是什么了，比如我的是 zsh，那么文件就是 .zshrc 了。</font>

<code>export NVM_DIR="$HOME/.nvm" [ -s "$NVM_DIR/nvm.sh" ] && \. "$NVM_DIR/nvm.sh"  # This loads nvm [ -s "$NVM_DIR/bash_completion" ] && \. "$NVM_DIR/bash_completion"  # This loads nvm bash_completion </code>

<font style="color:rgb(51, 51, 51);">这段代码的意思是：</font>

1. <font style="color:rgb(51, 51, 51);">声明 nvm 的目录所在路径，</font>
2. <font style="color:rgb(51, 51, 51);">加载 nvm，</font>
3. <font style="color:rgb(51, 51, 51);">加载 nvm bash\_completion。</font>

<font style="color:rgb(51, 51, 51);">这样，在终端内就可以使用 nvm 命令了</font>

### <font style="color:rgb(51, 51, 51);">nvm 使用</font>

<font style="color:rgb(51, 51, 51);">常用的几个 nvm 命令</font>

* <font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">nvm ls</font><font style="color:rgb(51, 51, 51);"> </font><font style="color:rgb(51, 51, 51);">查看本地安装的 node.js 版本列表</font>
* <font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">nvm install \*\*\*</font><font style="color:rgb(51, 51, 51);"> </font><font style="color:rgb(51, 51, 51);">安装 \*\*\* 版本</font>
* <font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">nvm use \*\*\*</font><font style="color:rgb(51, 51, 51);"> </font><font style="color:rgb(51, 51, 51);">在当前终端窗口使用某个版本</font>
* <font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">nvm alias default \*\*\*</font><font style="color:rgb(51, 51, 51);"> </font><font style="color:rgb(51, 51, 51);">设置某个版本为默认（新开终端窗口 or 重新打开终端生效）</font>

<font style="color:rgb(51, 51, 51);">至于其他的命令，可以用</font><font style="color:rgb(51, 51, 51);"> </font><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">nvm help</font><font style="color:rgb(51, 51, 51);"> </font><font style="color:rgb(51, 51, 51);">查看。</font>

### <font style="color:rgb(51, 51, 51);">nvm 需要注意的地方</font>

<font style="color:rgb(51, 51, 51);">由于切换版本是沙箱式，隔离 100%，故 npm 全局依赖在切换版本后，都需要重新安装一番了。这就是享受安全、互不影响的另一端了。事物都有双面性不是～</font>

<font style="color:rgb(51, 51, 51);">因此可以用</font><font style="color:rgb(51, 51, 51);"> </font><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">npm list -g --depth=0</font><font style="color:rgb(51, 51, 51);"> </font><font style="color:rgb(51, 51, 51);">快速查看全局安装了哪些依赖，嘿嘿，再安装一遍呗。</font>

### <font style="color:rgb(51, 51, 51);">补充，nrm</font>

<font style="color:rgb(51, 51, 51);">再补充一个小工具：一个管理 npm 源的 nrm。</font>

<font style="color:rgb(51, 51, 51);">咋一看两者还挺像的，nvm 是 node.js 的版本管理，nrm 是 npm 源的管理。谁让前端是个大杂烩呢。</font>

<font style="color:rgb(51, 51, 51);">(nvm: node version manager, nrm: npm registry manager)</font>

<font style="color:rgb(51, 51, 51);">什么时候会用到 nrm，如：国内 npm 源访问太慢，公司内部有自己的 npm 仓库，这时候切换源，用 nrm 就很方便。</font>

#### <font style="color:rgb(51, 51, 51);">安装与使用</font>

`npm i nrm -g nrm ls nrm use ${name}`
