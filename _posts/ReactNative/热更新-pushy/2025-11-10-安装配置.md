# 安装配置

## <font style="color:rgb(13, 26, 38);">安装</font>
<font style="color:rgb(49, 70, 89);">在你的项目根目录下运行以下命令：</font>

```bash
# 全局安装命令行工具，每台电脑只用装一次
npm i -g react-native-update-cli

# 在项目中安装热更新模块
npm i react-native-update
```

> 如果下载极慢或者显示网络失败，请设置使用淘宝镜像npx nrm use taobao
>

<font style="color:rgb(49, 70, 89);">如果你的RN版本 >= 0.60，请在iOS目录下执行:</font>

```bash
pod install
```

<font style="color:rgb(49, 70, 89);">如果你的RN版本 < 0.60，那么还需要</font>[手动link](https://pushy.reactnative.cn/docs/getting-started.html#%E6%89%8B%E5%8A%A8link)

> 注意：如果是混编RN项目，由于目前官方集成文档并不完善，参照官方文档可能无法实现自动link功能。此时即便RN版本 >= 0.60，你可能也需要执行手动link操作。
>

<font style="color:rgb(49, 70, 89);">如果你的RN版本比较老（< 0.46），请注意</font>

<font style="color:rgb(49, 70, 89);">如果你的RN版本比较老，请按下面表格尝试老一些的版本（但这些版本我们已不再维护，不能保证可以使用）：</font>

| <font style="color:rgb(49, 70, 89);">React Native版本</font> | <font style="color:rgb(49, 70, 89);">react-native-update版本</font> |
| --- | --- |
| <font style="color:rgb(49, 70, 89);">0.26及以下</font> | <font style="color:rgb(49, 70, 89);">1.0.x</font> |
| <font style="color:rgb(49, 70, 89);">0.27 - 0.28</font> | <font style="color:rgb(49, 70, 89);">2.x</font> |
| <font style="color:rgb(49, 70, 89);">0.29 - 0.33</font> | <font style="color:rgb(49, 70, 89);">3.x</font> |
| <font style="color:rgb(49, 70, 89);">0.34 - 0.45</font> | <font style="color:rgb(49, 70, 89);">4.x</font> |


<font style="color:rgb(49, 70, 89);">安装命令示例：</font>

```bash
npm i react-native-update@4.x
```

<font style="color:rgb(49, 70, 89);">如果RN的版本是0.45及以下，你还必须安装</font>[Android NDK](https://androiddevtools.cn/)<font style="color:rgb(49, 70, 89);">，版本最好选用r10e，并设置环境变量</font><font style="color:rgb(49, 70, 89);background-color:rgb(242, 244, 245);">ANDROID_NDK_HOME</font><font style="color:rgb(49, 70, 89);">，指向你的NDK根目录(例如</font><font style="color:rgb(49, 70, 89);background-color:rgb(242, 244, 245);">/Users/tdzl2003/Downloads/android-ndk-r10e</font><font style="color:rgb(49, 70, 89);">)。</font>

> 请记得，任意在ios和android目录下的修改，一定要重新编译（npx react-native run-ios或run-android命令编译，或在Xcode/Android Studio中重新编译）才能生效。
>

## <font style="color:rgb(13, 26, 38);">手动link</font>
<font style="color:rgb(49, 70, 89);">如果RN版本 >= 0.60则不需要此手动link步骤。</font>

> 注意：如果是混编RN项目，由于目前官方集成文档并不完善，参照官方文档可能无法实现自动link功能。此时即便RN版本 >= 0.60，你可能也需要执行手动link操作。
>

### <font style="color:rgb(13, 26, 38);">iOS</font>
<font style="color:rgb(49, 70, 89);">RN < 0.60且使用CocoaPods（推荐）</font>

1. <font style="color:rgb(49, 70, 89);">在ios/Podfile中添加</font>

```bash
pod 'react-native-update', path: '../node_modules/react-native-update'
```

2. <font style="color:rgb(49, 70, 89);">在项目的ios目录下运行</font><font style="color:rgb(49, 70, 89);background-color:rgb(242, 244, 245);">pod install</font>
3. <font style="color:rgb(49, 70, 89);">重新编译</font>

<font style="color:rgb(49, 70, 89);">RN < 0.60且不使用CocoaPods</font>

1. <font style="color:rgb(49, 70, 89);">在XCode中的Project Navigator里,右键点击</font><font style="color:rgb(49, 70, 89);background-color:rgb(242, 244, 245);">Libraries</font><font style="color:rgb(49, 70, 89);">➜</font><font style="color:rgb(49, 70, 89);background-color:rgb(242, 244, 245);">Add Files to [你的工程名]</font>
2. <font style="color:rgb(49, 70, 89);">进入</font><font style="color:rgb(49, 70, 89);background-color:rgb(242, 244, 245);">node_modules</font><font style="color:rgb(49, 70, 89);">➜</font><font style="color:rgb(49, 70, 89);background-color:rgb(242, 244, 245);">react-native-update</font><font style="color:rgb(49, 70, 89);">➜</font><font style="color:rgb(49, 70, 89);background-color:rgb(242, 244, 245);">ios 并选中</font><font style="color:rgb(49, 70, 89);">RCTPushy.xcodeproj`</font>
3. <font style="color:rgb(49, 70, 89);">在XCode中的project navigator里,选中你的工程,在</font><font style="color:rgb(49, 70, 89);background-color:rgb(242, 244, 245);">Build Phases</font><font style="color:rgb(49, 70, 89);">➜</font><font style="color:rgb(49, 70, 89);background-color:rgb(242, 244, 245);">Link Binary With Libraries</font><font style="color:rgb(49, 70, 89);">中添加</font><font style="color:rgb(49, 70, 89);background-color:rgb(242, 244, 245);">libRCTPushy.a</font><font style="color:rgb(49, 70, 89);">、</font><font style="color:rgb(49, 70, 89);background-color:rgb(242, 244, 245);">libz.tbd</font><font style="color:rgb(49, 70, 89);">、</font><font style="color:rgb(49, 70, 89);background-color:rgb(242, 244, 245);">libbz2.1.0.tbd</font>
4. <font style="color:rgb(49, 70, 89);">继续在</font><font style="color:rgb(49, 70, 89);background-color:rgb(242, 244, 245);">Build Settings</font><font style="color:rgb(49, 70, 89);">里搜索</font><font style="color:rgb(49, 70, 89);background-color:rgb(242, 244, 245);">Header Search Path</font><font style="color:rgb(49, 70, 89);">，添加</font><font style="color:rgb(49, 70, 89);background-color:rgb(242, 244, 245);">$(SRCROOT)/../node_modules/react-native-update/ios</font><font style="color:rgb(49, 70, 89);">，勾选</font><font style="color:rgb(49, 70, 89);background-color:rgb(242, 244, 245);">recursive</font><font style="color:rgb(49, 70, 89);">。</font>
5. <font style="color:rgb(49, 70, 89);">在</font><font style="color:rgb(49, 70, 89);background-color:rgb(242, 244, 245);">Build Phases</font><font style="color:rgb(49, 70, 89);">添加一个</font><font style="color:rgb(49, 70, 89);background-color:rgb(242, 244, 245);">New Run Script Phase</font><font style="color:rgb(49, 70, 89);">运行脚本，内容如下</font>

```bash
#!/bin/bash
set -x
DEST="../node_modules/react-native-update/ios/"
date +%s > "$DEST/pushy_build_time.txt"
```

6. <font style="color:rgb(49, 70, 89);">尝试编译一下，顺利的话就会在</font><font style="color:rgb(49, 70, 89);background-color:rgb(242, 244, 245);">../node_modules/react-native-update/ios/</font><font style="color:rgb(49, 70, 89);">文件夹下面生成一个</font><font style="color:rgb(49, 70, 89);background-color:rgb(242, 244, 245);">pushy_build_time.txt</font><font style="color:rgb(49, 70, 89);">文件。 然后在</font><font style="color:rgb(49, 70, 89);background-color:rgb(242, 244, 245);">Copy Bundle Resources</font><font style="color:rgb(49, 70, 89);">里把生成的</font><font style="color:rgb(49, 70, 89);background-color:rgb(242, 244, 245);">pushy_build_time.txt</font><font style="color:rgb(49, 70, 89);">文件添加进去。</font>

### <font style="color:rgb(13, 26, 38);">Android</font>
<font style="color:rgb(49, 70, 89);">RN < 0.60</font>

1. <font style="color:rgb(49, 70, 89);">在</font><font style="color:rgb(49, 70, 89);background-color:rgb(242, 244, 245);">android/settings.gradle</font><font style="color:rgb(49, 70, 89);">中添加如下代码:</font>

include ':react-native-update'

```
```

```java

```

3. <font style="color:rgb(49, 70, 89);">打开</font><font style="color:rgb(49, 70, 89);background-color:rgb(242, 244, 245);">android/app/src/main/java/[...]/MainApplication.java</font><font style="color:rgb(49, 70, 89);">,</font>

## <font style="color:rgb(13, 26, 38);">配置Bundle URL</font>
> <font style="color:rgb(49, 70, 89);">注意此步骤无论任何版本，目前都需要手动配置。</font>
>

### <font style="color:rgb(13, 26, 38);">iOS</font>
<font style="color:rgb(49, 70, 89);">在你的AppDelegate.m文件中增加如下代码：</font>

```c
// ... 其它代码
#import "AppDelegate.h"

#import "RCTPushy.h"  // <-- import头文件，注意要放到if条件外面

// 可能项目里有一些条件编译语句，例如像较新版本RN自带的flipper
// #if DEBUG
// 注意**不要**在这里面引入"RCTPushy.h"
// #import <FlipperKit/FlipperClient.h>
// ...
// #endif


// 如果RN版本 >= 0.59，修改sourceURLForBridge
- (NSURL *)sourceURLForBridge:(RCTBridge *)bridge
{
#if DEBUG
  return [[RCTBundleURLProvider sharedSettings] jsBundleURLForBundleRoot:@"index" fallbackResource:nil];
#else
  // 非DEBUG情况下替换为热更新bundle
  return [RCTPushy bundleURL];
#endif
}

// 如果RN版本 < 0.59，修改didFinishLaunchingWithOptions
- (BOOL)application:(UIApplication *)application didFinishLaunchingWithOptions:(NSDictionary *)launchOptions
{
#if DEBUG
  // 原来的jsCodeLocation保留在这里
  jsCodeLocation = ..........
#else
  // 非DEBUG情况下替换为热更新bundle
  jsCodeLocation = [RCTPushy bundleURL];
#endif
  // ... 其它代码
}
```

### <font style="color:rgb(13, 26, 38);">Android</font>
<font style="color:rgb(49, 70, 89);">在MainApplication中增加如下代码（如果是混编原生的项目或其他原因没有使用ReactApplication，请</font>[使用此api集成](https://pushy.reactnative.cn/docs/api.html#updatecontextsetcustominstancemanagerreactinstancemanager-instancemanager)<font style="color:rgb(49, 70, 89);">)：</font>

```java
// ... 其它代码

// 请注意不要少了这句import
import cn.reactnative.modules.update.UpdateContext;
public class MainApplication extends Application implements ReactApplication {

  private final ReactNativeHost mReactNativeHost = new ReactNativeHost(this) {
    // 注意这一段在 ReactNativeHost 内部！
    @Override
    protected String getJSBundleFile() {
        return UpdateContext.getBundleUrl(MainApplication.this);
    }
    // ... 其它代码
  }
}
```

> 请记得，任意在ios和android目录下的修改，一定要重新编译（npx react-native run-ios或run-android命令编译，或在Xcode/Android Studio中重新编译）才能生效。
>

## <font style="color:rgb(13, 26, 38);">禁用android的crunch优化</font>
<font style="color:rgb(49, 70, 89);">android会在生成apk时自动对png图片进行压缩，此操作既耗时又影响增量补丁的生成。为了保证补丁能正常生成，您需要在</font><font style="color:rgb(49, 70, 89);background-color:rgb(242, 244, 245);">android/app/build.gradle</font><font style="color:rgb(49, 70, 89);">中关闭此操作：</font>

```java
...
android {
    ...
    signingConfigs { ... }
    buildTypes {
        release {
            ...
            // 添加下面这行以禁用crunch
            crunchPngs false
        }
    }
}
...
```

## <font style="color:rgb(13, 26, 38);">登录与创建应用</font>
<font style="color:rgb(49, 70, 89);">首先请在</font>[https://update.reactnative.cn](https://update.reactnative.cn/)<font style="color:rgb(49, 70, 89);">注册帐号，然后在你的项目根目录下运行以下命令：</font>

```bash
pushy login

email: <输入你的注册邮箱>
password: <输入你的密码>
```

<font style="color:rgb(49, 70, 89);">这会在项目文件夹下创建一个</font><font style="color:rgb(49, 70, 89);background-color:rgb(242, 244, 245);">.update</font><font style="color:rgb(49, 70, 89);">文件，注意不要把这个文件上传到Git等CVS系统上。你可以在</font><font style="color:rgb(49, 70, 89);background-color:rgb(242, 244, 245);">.gitignore</font><font style="color:rgb(49, 70, 89);">末尾增加一行</font><font style="color:rgb(49, 70, 89);background-color:rgb(242, 244, 245);">.update</font><font style="color:rgb(49, 70, 89);">来忽略这个文件。</font>

<font style="color:rgb(49, 70, 89);">登录之后可以创建应用。注意iOS平台和安卓平台需要分别创建：</font>

```bash
$ pushy createApp --platform ios
App Name: <输入应用名字>
$ pushy createApp --platform android
App Name: <输入应用名字>
```

> 两次输入的名字可以相同，这没有关系。
>

<font style="color:rgb(49, 70, 89);">如果你已经在网页端或者其它地方创建过应用，也可以直接选择应用：</font>

```bash
$ pushy selectApp --platform ios
1) 鱼多多(ios)
2) 招财旺(ios)

Total 2 ios apps
Enter appId: <输入应用前面的编号>
```

<font style="color:rgb(49, 70, 89);">选择或者创建过应用后，你将可以在文件夹下看到</font><font style="color:rgb(49, 70, 89);background-color:rgb(242, 244, 245);">update.json</font><font style="color:rgb(49, 70, 89);">文件，其内容类似如下形式：</font>

```json
{
    "ios": {
        "appId": 1,
        "appKey": "<一串随机字符串>"
    },
    "android": {
        "appId": 2,
        "appKey": "<一串随机字符串>"
    }
}
```

<font style="color:rgb(49, 70, 89);">你可以安全的把</font><font style="color:rgb(49, 70, 89);background-color:rgb(242, 244, 245);">update.json</font><font style="color:rgb(49, 70, 89);">上传到Git等CVS系统上，与你的团队共享这个文件，它不包含任何敏感信息。当然，他们在使用任何功能之前，都必须首先输入</font><font style="color:rgb(49, 70, 89);background-color:rgb(242, 244, 245);">pushy login</font><font style="color:rgb(49, 70, 89);">进行登录。</font>
