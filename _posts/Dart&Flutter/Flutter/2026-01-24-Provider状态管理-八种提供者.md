# Provider状态管理-八种提供者

## 前言
<font style="color:rgb(52, 52, 60);">在我们上一篇文章中对</font>Provider<font style="color:rgb(52, 52, 60);">进行了介绍以及类结构的说明，最后还写了一个简单的示例，通过上一章节我们对</font>Provider<font style="color:rgb(52, 52, 60);">有了一个基本的了解，这一章节我们来说说</font>Provider<font style="color:rgb(52, 52, 60);">的8种提供者以及他们的使用区别。</font>

## Provider
Provider<font style="color:rgb(52, 52, 60);">是最基本的</font>**<font style="color:rgb(52, 52, 60);">Provider</font>**<font style="color:rgb(52, 52, 60);">组件，可以使用它为组件树中的任何位置提供值，但是当该值更改的时候，它并不会更新UI，下面我们给出一个示例</font>

### 第一步：创建模型
```dart
class UserModel {

  String name = "Jimi";

  void changeName() {
    name = "hello";
  }
}
```

### 第二步：应用程序入口设置
```dart
return Provider<UserModel>(
  create: (_) => UserModel(),
  child: MaterialApp(
    debugShowCheckedModeBanner: false,
    home: ProviderExample(),
  ),
);
```

### 第三步：使用共享数据
<font style="color:rgb(52, 52, 60);">关于</font>Consumer<font style="color:rgb(52, 52, 60);">后面将消费者在提及，我们这里只需要知道有两个消费者，第一个用于展示模型的数据，第二个用于改变模型的数据。</font>

+ <font style="color:rgb(52, 52, 60);">第一个</font>Comsumer<font style="color:rgb(52, 52, 60);">是用于读取模型的数据</font>name
+ <font style="color:rgb(52, 52, 60);">第二个</font>Consumer<font style="color:rgb(52, 52, 60);">用于改变模型的数据</font>name

```dart
import 'package:flutter/material.dart';
import 'package:flutter_provider_example/provider_example/user_model.dart';
import 'package:provider/provider.dart';

class ProviderExample extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text("ProviderExample"),
      ),
      body: Center(
        child: Column(
          mainAxisAlignment: MainAxisAlignment.center,
          children: [
            Consumer<UserModel>(
              builder: (_, userModel, child) {
                return Text(userModel.name,
                            style: TextStyle(
                              color: Colors.red,
                              fontSize: 30
                            )
                           );
              },
            ),
            Consumer<UserModel>(
              builder: (_, userModel, child) {
                return Padding(
                  padding: EdgeInsets.all(20),
                  child: ElevatedButton(
                    onPressed: (){
                      userModel.changeName();
                    },
                    child: Text("改变值"),
                  ),
                );
              },
            ),
          ],
        ),
      ),
    );
  }
}
```

### 运行结果
<font style="color:rgb(52, 52, 60);">我们点击按钮的会导致模型数据改变，但是模型数据改变之后UI并没有变化也没有重建，那是因为</font>Provider<font style="color:rgb(52, 52, 60);">提供者组件不会监听它提供的值的变化。</font>

## ChangeNotifierProvider
<font style="color:rgb(52, 52, 60);">它跟</font>Provider<font style="color:rgb(52, 52, 60);">组件不同，</font>ChangeNotifierProvider<font style="color:rgb(52, 52, 60);">会监听模型对象的变化，而且当数据改变时，它也会重建</font>Consumer<font style="color:rgb(52, 52, 60);">（消费者），下面我们给出一个示例</font>

### 第一步：创建模型
<font style="color:rgb(52, 52, 60);">细心点我们可以发现这里定义的模型有两处变化，如下：</font>

+ <font style="color:rgb(52, 52, 60);">混入了</font>ChangeNotifier
+ <font style="color:rgb(52, 52, 60);">调用了</font>notifyListeners()

<font style="color:rgb(52, 52, 60);">因为模型类使用了</font>ChangeNotifier<font style="color:rgb(52, 52, 60);">，那么我们就可以访问</font>notifyListeners()<font style="color:rgb(52, 52, 60);">并且在调用它的任何时候，</font>ChangeNotifierProvider<font style="color:rgb(52, 52, 60);">都会收到通知并且消费者将重建UI。</font>

```dart
import 'package:flutter/material.dart';

class UserModel1 with ChangeNotifier {

  String name = "Jimi";

  void changeName() {
    name = "hello";
    notifyListeners();
  }
}
```

### 第二步：应用程序入口设置
```dart
return ChangeNotifierProvider<UserModel1>(
  create: (_) => UserModel1(),
  child: MaterialApp(
    debugShowCheckedModeBanner: false,
    home: ChangeNotifierProviderExample(),
  ),
);
```

### 第三步：使用共享数据

## FutureProvider
<font style="color:rgb(52, 52, 60);">简单来说，</font>FutureProvider<font style="color:rgb(52, 52, 60);">用于提供在组件树中准备好使用其值时可能尚未准备好的值，主要是确保空值不会传递给任何子组件，而且</font>FutureProvider<font style="color:rgb(52, 52, 60);">有一个初始值，子组件可以使用该</font>Future<font style="color:rgb(52, 52, 60);">值并告诉子组件使用新的值来进行重建。</font>

**<font style="color:rgb(52, 52, 60);">注意：</font>**

+ FutureProvider<font style="color:rgb(52, 52, 60);">只会重建一次</font>
+ <font style="color:rgb(52, 52, 60);">默认显示初始值</font>
+ <font style="color:rgb(52, 52, 60);">然后显示</font>Future<font style="color:rgb(52, 52, 60);">值</font>
+ <font style="color:rgb(52, 52, 60);">最后不会再次重建</font>

### 第一步：创建模型
<font style="color:rgb(52, 52, 60);">这里和</font>Provider<font style="color:rgb(52, 52, 60);">不同的是增加了构造函数，以及</font>changeName<font style="color:rgb(52, 52, 60);">变成了</font>Future<font style="color:rgb(52, 52, 60);">，我们模拟网络请求延迟两秒后改变其值。</font>

```dart
class UserModel2{

  UserModel2({this.name});

  String? name = "Jimi";

  Future<void> changeName() async {
    await Future.delayed(Duration(milliseconds: 2000));
    name = "hello";
  }
}
```

### 第二步：提供Future
<font style="color:rgb(52, 52, 60);">我们有一个方法，就是异步获取</font>userModel2<font style="color:rgb(52, 52, 60);">，模拟网络请求延迟两秒执行，最后修改了</font>name<font style="color:rgb(52, 52, 60);">并返回</font>UserModel2

```dart
import 'package:flutter_provider_example/future_provider_example/user_model2.dart';

class UserFuture {

  Future<UserModel2> asyncGetUserModel2() async {
    await Future.delayed(Duration(milliseconds: 2000));
    return UserModel2(name: "获取新的数据");
  }

}
```

### 第三步：应用程序入口设置
initialData<font style="color:rgb(52, 52, 60);">是默认值，</font>create<font style="color:rgb(52, 52, 60);">参数我们传了一个</font>Future<UserModel2><font style="color:rgb(52, 52, 60);">，因为它接收的模型</font>Create<Future<T>?>

```dart
return FutureProvider<UserModel2>(
  initialData: UserModel2(name: "hello"),
  create: (_) => UserFuture().asyncGetUserModel2(),
  child: MaterialApp(
    debugShowCheckedModeBanner: false,
    home: FutureProviderExample(),
  ),
);
```

### 第四步：使用共享数据

## StreamProvider
StreamProvider<font style="color:rgb(52, 52, 60);">提供流值，是围绕</font>StreamBuilder<font style="color:rgb(52, 52, 60);">，所提供的值会在传入的时候替换掉新值。和</font>FutureProvider<font style="color:rgb(52, 52, 60);">一样，主要的区别在于值会根据多次触发重新构建UI。</font>

<font style="color:rgb(52, 52, 60);">如果你对</font>StreamBuilder<font style="color:rgb(52, 52, 60);">不太了解的话，那么你就很难理解</font>StreamProvider<font style="color:rgb(52, 52, 60);">，</font>[StreamProvider文档地址](https://book.flutterchina.club/chapter7/futurebuilder_and_streambuilder.html#_7-6-2-streambuilder)

### 第一步：创建模型
```dart
class UserModel3{

  UserModel3({this.name});

  String? name = "Jimi";

  void changeName() {
    name = "hello";
  }
}
```

### 第二步：提供Stream
<font style="color:rgb(52, 52, 60);">下面这段代码类似计时器，每隔一秒钟生成一个数字</font>

```dart
import 'package:flutter_provider_example/stream_provider_example/user_model3.dart';

class UserStream {

  Stream<UserModel3> getStreamUserModel() {
    return Stream<UserModel3>.periodic(Duration(milliseconds: 1000),
                                       (value) => UserModel3(name: "$value")
                                      ).take(10);
  }
}
```



### 第三步：应用程序入口设置
<font style="color:rgb(52, 52, 60);">这里也有</font>initialData<font style="color:rgb(52, 52, 60);">初始值，和</font>FutureProvider<font style="color:rgb(52, 52, 60);">类似，只是</font>create<font style="color:rgb(52, 52, 60);">属性是获取一个</font>Stream<font style="color:rgb(52, 52, 60);">流。</font>

```dart
return StreamProvider<UserModel3>(

    debugShowCheckedModeBanner: false,
```

## MultiProvider
<font style="color:rgb(52, 52, 60);">在上面的例子中我们都只是返回了一个提供者，在实际开发过程中肯定会有多个提供者，我们虽然可以采用嵌套的方式来解决，但是这样无疑是混乱的，可读性级差。这个时候强大的</font>MultiProvder<font style="color:rgb(52, 52, 60);">就产生了，我们来看下示例:</font>

### 第一步：创建两个模型
```dart
import 'package:flutter/material.dart';

class UserModel1 with ChangeNotifier {

  String name = "Jimi";

  void changeName() {
    name = "hello";
    notifyListeners();
  }
}

class UserModel4 with ChangeNotifier {

  String name = "Jimi";
  int age = 18;

  void changeName() {
    name = "hello";
    age = 20;
    notifyListeners();
  }
}
```



### 第二步：应用程序入口设置
<font style="color:rgb(52, 52, 60);">相对于方式一这种嵌套方式设置，方式二就显得尤为简单。</font>

#### 方式一：嵌套设置
```dart
return ChangeNotifierProvider<UserModel1>(
  create: (_) => UserModel1(),
  child: ChangeNotifierProvider<UserModel4>(
    create: (_) => UserModel4(),
    child: MaterialApp(
      debugShowCheckedModeBanner: false,
      home: MultiProviderExample(),
    ),
  ),
);
```



#### 方式二：使用MultiProvider
```dart
return MultiProvider(
  providers: [
    ChangeNotifierProvider<UserModel1>(create: (_) => UserModel1()),
    ChangeNotifierProvider<UserModel4>(create: (_) => UserModel4()),
    /// 添加更多
  ],
  child: MaterialApp(
    debugShowCheckedModeBanner: false,
    home: MultiProviderExample(),
  ),
);
```



### 第三步：使用共享数据

## ProxyProvider
<font style="color:rgb(52, 52, 60);">当我们有多个模型的时候，会有模型依赖另一个模型的情况，在这种情况下，我们可以使用</font>ProxyProvider<font style="color:rgb(52, 52, 60);">从另一个提供者获取值，然后将其注入到另一个提供者中。我们来看下代码演示</font>

### 第一步：创建两个模型
<font style="color:rgb(52, 52, 60);">下面我们创建了两个模型</font>UserModel5<font style="color:rgb(52, 52, 60);">和</font>WalletModel<font style="color:rgb(52, 52, 60);">，而</font>WalletModel<font style="color:rgb(52, 52, 60);">依赖与</font>UserModel5<font style="color:rgb(52, 52, 60);">，当调用</font>WalletModel<font style="color:rgb(52, 52, 60);">的</font>changeName<font style="color:rgb(52, 52, 60);">方法时会改变</font>UserModel5<font style="color:rgb(52, 52, 60);">里面的name，当然我们在实际开发的过程中并不是这么简单，这里只是演示模型依赖时如果使用</font>ProxyProvider

```dart
import 'package:flutter/material.dart';

class UserModel5 with ChangeNotifier {

  String name = "Jimi";

  void changeName({required String newName}) {
    name = newName;
    notifyListeners();
  }
}


class WalletModel {

  UserModel5? userModel5;

  WalletModel({this.userModel5});

  void changeName() {
    userModel5?.changeName(newName: "JIMI");
  }
}
```



### 第二步：应用程序入口设置
```dart
return MultiProvider(
  providers: [
    ChangeNotifierProvider<UserModel5>(create: (_) => UserModel5()),
    ProxyProvider<UserModel5, WalletModel>(
      update: (_, userModel5, walletModel) => WalletModel(userModel5: userModel5),
    )
  ],
  child: MaterialApp(
    debugShowCheckedModeBanner: false,
    home: ProxyProviderExample(),
  ),
);
```

## ChangeNotifierProxyProvider
<font style="color:rgb(52, 52, 60);">和</font>ProxyProvider<font style="color:rgb(52, 52, 60);">原理一样，唯一的区别在于它构建和同步</font>ChangeNotifier<font style="color:rgb(52, 52, 60);">的</font>ChangeNotifierProvider<font style="color:rgb(52, 52, 60);">，当提供者数据变化时，将会重构UI。</font>

<font style="color:rgb(52, 52, 60);">下面我们给出一个例子：</font>

+ <font style="color:rgb(52, 52, 60);">获取书籍列表</font>
+ <font style="color:rgb(52, 52, 60);">获取收藏书籍列表</font>
+ <font style="color:rgb(52, 52, 60);">点击书籍可加入或者取消收藏</font>
+ <font style="color:rgb(52, 52, 60);">通过代理实时重构UI</font>

### 第一步：创建两个模型
#### 1、BookModel
BookModel<font style="color:rgb(52, 52, 60);">用户存储模型数据，将书籍转换成模型。</font>

```dart

    Book(6, "深空彼岸"),
  ];

  // 获取书籍长度
  int get length => _books.length;

  // 根据ID获取书籍
  Book getById(int id) => _books[id -1];

  // 根据索引获取数据
  Book getByPosition(int position) => _books[position];

  // 更多....
}

class Book {
```

#### 2、BookManagerModel
BookManagerModel<font style="color:rgb(52, 52, 60);">主要用于管理书籍、收藏书籍、取消收藏等操作</font>

```dart
import 'package:flutter/material.dart';
import 'package:flutter_provider_example/change_notifier_proxy_provider_example/models/book_model.dart';

class BookManagerModel with ChangeNotifier {

  // 依赖bookModel
  final BookModel _bookModel;

  // 获取数据所有的ID
  List<int>? _bookIds;

  // 构造函数
  BookManagerModel(this._bookModel, {BookManagerModel? bookManagerModel})
    : _bookIds = bookManagerModel?._bookIds ?? [];

  // 获取所有的书
  List<Book> get books => _bookIds!.map((id) => _bookModel.getById(id)).toList();

  // 根据索引获取数据
  Book getByPosition(int position) => books[position];

  // 获取书籍的长度
  int get length => _bookIds?.length ?? 0;

  // 添加书籍
  void addFaves(Book book) {
    _bookIds!.add(book.bookId);
    notifyListeners();
  }

  // 删除书籍
  void removeFaves(Book book) {
    _bookIds!.remove(book.bookId);
    notifyListeners();
  }
}
```
