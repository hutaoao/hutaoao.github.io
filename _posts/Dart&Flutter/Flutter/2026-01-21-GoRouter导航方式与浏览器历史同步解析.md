# GoRouter导航方式与浏览器历史同步解析

<https://pub.dev/documentation/go_router/latest/topics/Navigation-topic.html>

这个链接的下面这段话说表达的深层含义是什么

> Imperative navigation
> GoRouter can push a screen onto the Navigator's history stack using context.push(), and can pop the current screen via context.pop(). However, imperative navigation is known to cause issues with the browser history.

<font style="color:rgb(15, 17, 21);">这段话的核心是在解释</font><font style="color:rgb(15, 17, 21);"> </font>**<font style="color:rgb(15, 17, 21);">GoRouter 提供的两种导航方式（声明式和命令式）以及为什么更推荐使用声明式</font>**<font style="color:rgb(15, 17, 21);">。</font>

### <font style="color:rgb(15, 17, 21);">表层含义直译</font>

**<font style="color:rgb(15, 17, 21);">命令式导航</font>**<font style="color:rgb(15, 17, 21);">\ </font><font style="color:rgb(15, 17, 21);">GoRouter 可以使用</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">context.push()</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">将一个页面压入 Navigator 的历史堆栈，也可以使用</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">context.pop()</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">弹出当前页面。然而，众所周知，命令式导航会导致浏览器历史记录出现问题。</font>

### <font style="color:rgb(15, 17, 21);">深层含义解析</font>

<font style="color:rgb(15, 17, 21);">这段话的“深层含义”可以从以下几个层面来理解：</font>

#### <font style="color:rgb(15, 17, 21);">1. 区分两种编程范式</font>

* **<font style="color:rgb(15, 17, 21);">命令式导航 (Imperative Navigation)</font>**<font style="color:rgb(15, 17, 21);">：你通过直接发出命令来告诉程序</font>**<font style="color:rgb(15, 17, 21);">如何</font>**<font style="color:rgb(15, 17, 21);">做。就像说：“现在，立刻，把页面B推到当前页面A之上。”</font>
  * <font style="color:rgb(15, 17, 21);">对应的方法：</font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">context.push('/details')</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">和</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">context.pop()</font></code><font style="color:rgb(15, 17, 21);">。</font>
  * **<font style="color:rgb(15, 17, 21);">特点</font>**<font style="color:rgb(15, 17, 21);">：你精确地控制着每一步操作，类似于在原生 iOS/Android 开发中操作</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">UINavigationController</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">或</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">FragmentManager</font></code><font style="color:rgb(15, 17, 21);">。</font>
* **<font style="color:rgb(15, 17, 21);">声明式导航 (Declarative Navigation)</font>**<font style="color:rgb(15, 17, 21);">：你只需要描述</font>**<font style="color:rgb(15, 17, 21);">当前的状态</font>**<font style="color:rgb(15, 17, 21);">（即“现在应该显示哪个页面”），而由框架（如 Flutter）负责</font>**<font style="color:rgb(15, 17, 21);">如何</font>**<font style="color:rgb(15, 17, 21);">从上一个状态过渡到当前状态。</font>
  * <font style="color:rgb(15, 17, 21);">对应的方法：在构建</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">GoRouter</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">时定义所有的路由规则 (</font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">routes</font></code><font style="color:rgb(15, 17, 21);">)，然后通过</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">go()</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">或</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">push()</font></code>*<font style="color:rgb(15, 17, 21);">（注意：这里特指 GoRouter 的</font>\_\_<font style="color:rgb(15, 17, 21);"> </font>*<code>_<font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">go</font>_</code>*<font style="color:rgb(15, 17, 21);"> </font>\_\_<font style="color:rgb(15, 17, 21);">方法）</font>*<font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">来改变代表当前位置的状态（通常是</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">GoRouterState</font></code><font style="color:rgb(15, 17, 21);">）。</font>
  * **<font style="color:rgb(15, 17, 21);">特点</font>**<font style="color:rgb(15, 17, 21);">：你关心的是“是什么”（What），而不是“怎么做”（How）。这是 Flutter 框架的核心思想，应用于状态管理、UI 构建等各个方面。</font>

#### <font style="color:rgb(15, 17, 21);">2. 核心问题：Web 浏览器历史的同步</font>

<font style="color:rgb(15, 17, 21);">这是这段话最关键的“深层含义”。命令式导航（尤其是传统的</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">push</font></code><font style="color:rgb(15, 17, 21);">/</font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">pop</font></code><font style="color:rgb(15, 17, 21);">）在</font><font style="color:rgb(15, 17, 21);"> </font>**<font style="color:rgb(15, 17, 21);">Web 平台</font>**<font style="color:rgb(15, 17, 21);">上会遇到一个根本性的挑战：</font>

* **<font style="color:rgb(15, 17, 21);">浏览器的期望</font>**<font style="color:rgb(15, 17, 21);">：浏览器自身维护着一个历史记录堆栈。用户点击“前进/后退”按钮时期望的是在整个</font>**<font style="color:rgb(15, 17, 21);">网站</font>**<font style="color:rgb(15, 17, 21);">的导航历史中跳转，而不仅仅是在你当前 Flutter 应用的某个流程里跳转。</font>
* **<font style="color:rgb(15, 17, 21);">传统命令式的缺陷</font>**<font style="color:rgb(15, 17, 21);">：如果你在 Flutter Web 应用中连续调用多次</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">context.push()</font></code><font style="color:rgb(15, 17, 21);">，你只是在 Flutter 的 Navigator 堆栈里添加了记录，</font>**<font style="color:rgb(15, 17, 21);">浏览器对此一无所知</font>**<font style="color:rgb(15, 17, 21);">。这会导致：</font>
  * **<font style="color:rgb(15, 17, 21);">无法使用浏览器后退按钮</font>**<font style="color:rgb(15, 17, 21);">：用户点击浏览器后退按钮，期望是返回上一个“站点”，但你的 Flutter 应用可能只是弹出了一个对话框或内部页面，体验非常割裂，甚至直接退出了整个 Web 应用。</font>
  * **<font style="color:rgb(15, 17, 21);">历史记录错乱</font>**<font style="color:rgb(15, 17, 21);">：浏览器的地址栏 URL 没有随着你的内部导航而更新，导致无法直接分享链接、无法刷新页面（刷新后会回到初始状态，而不是当前看到的页面）。</font>

#### <font style="color:rgb(15, 17, 21);">3. GoRouter 的解决方案与优势</font>

<font style="color:rgb(15, 17, 21);">GoRouter 被设计为一个</font>**<font style="color:rgb(15, 17, 21);">声明式优先</font>**<font style="color:rgb(15, 17, 21);">的路由库，它完美地解决了上述问题：</font>

* **<font style="color:rgb(15, 17, 21);">状态即真理</font>**<font style="color:rgb(15, 17, 21);">：GoRouter 的核心是一个状态（当前 URL）。无论你是通过</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">go()</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">方法导航，还是用户点击浏览器前进/后退按钮，最终都是</font>**<font style="color:rgb(15, 17, 21);">改变这个状态</font>**<font style="color:rgb(15, 17, 21);">。</font>
* **<font style="color:rgb(15, 17, 21);">自动同步</font>**<font style="color:rgb(15, 17, 21);">：每当状态（URL）改变时，GoRouter 会做两件事：</font>
  1. **<font style="color:rgb(15, 17, 21);">更新 Flutter 导航堆栈</font>**<font style="color:rgb(15, 17, 21);">：根据新的 URL，解析并显示出对应的页面。</font>
  2. **<font style="color:rgb(15, 17, 21);">更新浏览器历史</font>**<font style="color:rgb(15, 17, 21);">：将新的 URL 告知浏览器，将其加入浏览器的历史记录堆栈。</font>
* **<font style="color:rgb(15, 17, 21);">双向同步</font>**<font style="color:rgb(15, 17, 21);">：这样一来，就实现了</font><font style="color:rgb(15, 17, 21);"> </font>**<font style="color:rgb(15, 17, 21);">Flutter 导航堆栈</font>**<font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">和</font><font style="color:rgb(15, 17, 21);"> </font>**<font style="color:rgb(15, 17, 21);">浏览器历史记录</font>**<font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">的</font>**<font style="color:rgb(15, 17, 21);">双向同步</font>**<font style="color:rgb(15, 17, 21);">。</font>
  * <font style="color:rgb(15, 17, 21);">在 App 内导航 -> 更新 URL -> 浏览器记录更新。</font>
  * <font style="color:rgb(15, 17, 21);">用户操作浏览器按钮 -> URL 改变 -> GoRouter 接收到变化 -> 更新 Flutter 页面。</font>

### <font style="color:rgb(15, 17, 21);">结论与建议</font>

<font style="color:rgb(15, 17, 21);">这段话的深层含义其实是</font><font style="color:rgb(15, 17, 21);"> </font>**<font style="color:rgb(15, 17, 21);">GoRouter 作者的一种强烈建议和设计哲学阐述</font>**<font style="color:rgb(15, 17, 21);">：</font>

<font style="color:rgb(15, 17, 21);">“虽然我们提供了熟悉的命令式方法</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">push</font></code><font style="color:rgb(15, 17, 21);">/</font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">pop</font></code><font style="color:rgb(15, 17, 21);">（为了兼容性和一些简单场景），但你要知道它们在有 Web 需求的场景下是有缺陷的。</font>**<font style="color:rgb(15, 17, 21);">我们强烈推荐你使用声明式的</font>\*\*\*\*<font style="color:rgb(15, 17, 21);"> </font>**<code>**<font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">go()</font>**</code>**<font style="color:rgb(15, 17, 21);"> </font>\*\*\*\*<font style="color:rgb(15, 17, 21);">方法来进行导航，因为这是我们库的核心设计，它能确保你的应用在所有平台（尤其是 Web）上拥有一致且正确的导航行为。</font>**<font style="color:rgb(15, 17, 21);">”</font>

**<font style="color:rgb(15, 17, 21);">总结一下关键点：</font>**

| <font style="color:rgb(15, 17, 21);">特性</font> | <font style="color:rgb(15, 17, 21);">命令式 (</font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">push</font></code><br/><font style="color:rgb(15, 17, 21);">/</font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">pop</font></code><br/><font style="color:rgb(15, 17, 21);">)</font> | <font style="color:rgb(15, 17, 21);">声明式 (</font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">go()</font></code><br/><font style="color:rgb(15, 17, 21);">)</font> |
| --- | --- | --- |
| **<font style="color:rgb(15, 17, 21);">哲学</font>** | <font style="color:rgb(15, 17, 21);">告诉程序“如何”做</font> | <font style="color:rgb(15, 17, 21);">描述“现在是什么”状态</font> |
| **<font style="color:rgb(15, 17, 21);">浏览器支持</font>** | **<font style="color:rgb(15, 17, 21);">差</font>**<font style="color:rgb(15, 17, 21);">，易导致历史记录不同步</font> | **<font style="color:rgb(15, 17, 21);">优秀</font>**<font style="color:rgb(15, 17, 21);">，自动与浏览器历史同步</font> |
| **<font style="color:rgb(15, 17, 21);">深度链接</font>** | <font style="color:rgb(15, 17, 21);">难以支持</font> | **<font style="color:rgb(15, 17, 21);">天然支持</font>**<font style="color:rgb(15, 17, 21);">，URL 即状态</font> |
| **<font style="color:rgb(15, 17, 21);">推荐度</font>** | <font style="color:rgb(15, 17, 21);">仅用于简单或无 Web 需求的场景</font> | **<font style="color:rgb(15, 17, 21);">首选和推荐</font>**<font style="color:rgb(15, 17, 21);">的方式</font> |

<font style="color:rgb(15, 17, 21);">因此，在大多数情况下，你应该优先使用 </font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">context.go(</font></code><font style="color:rgb(15, 17, 21);">/</font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">path</font></code><font style="color:rgb(15, 17, 21);">/</font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">)</font></code><font style="color:rgb(15, 17, 21);"> 而不是 </font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">context.push(</font></code><font style="color:rgb(15, 17, 21);">/</font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">path</font></code><font style="color:rgb(15, 17, 21);">/</font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">)</font></code><font style="color:rgb(15, 17, 21);">，以确保应用的最佳实践和跨平台兼容性。</font>

***

> <font style="color:rgb(15, 17, 21);">嵌套路由是实现清晰、可维护且与浏览器历史完美同步的 Web 应用结构的最佳实践。</font>

<font style="color:rgb(15, 17, 21);">下面我们来详细解释为什么：</font>

### <font style="color:rgb(15, 17, 21);">1. 核心原则：URL 作为唯一真相源</font>

<font style="color:rgb(15, 17, 21);">GoRouter 的声明式导航核心是</font><font style="color:rgb(15, 17, 21);"> </font>**<font style="color:rgb(15, 17, 21);">URL 路径（Path）</font>**<font style="color:rgb(15, 17, 21);">。你的应用在任何时刻显示哪个页面，完全由当前的 URL 决定。</font>

* <code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">context.go('/path')</font></code><font style="color:rgb(15, 17, 21);">：这个操作的本质是</font>**<font style="color:rgb(15, 17, 21);">改变这个唯一真相源</font>**<font style="color:rgb(15, 17, 21);">。</font>
* <font style="color:rgb(15, 17, 21);">浏览器前进/后退：本质也是</font>**<font style="color:rgb(15, 17, 21);">改变这个唯一真相源</font>**<font style="color:rgb(15, 17, 21);">。</font>

<font style="color:rgb(15, 17, 21);">为了让</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">go()</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">能精确地控制整个页面栈，你的路由配置必须能够通过 URL</font><font style="color:rgb(15, 17, 21);"> </font>**<font style="color:rgb(15, 17, 21);">唯一地、明确地</font>**<font style="color:rgb(15, 17, 21);">推断出整个页面层级结构（例如：哪个是父页面，哪个是子页面，哪个是模态对话框）。而</font><font style="color:rgb(15, 17, 21);"> </font>**<font style="color:rgb(15, 17, 21);">嵌套路由（Nested Routes）</font>**<font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">就是用来定义这种层级关系的。</font>

***

### <font style="color:rgb(15, 17, 21);">2. 两种场景对比：使用嵌套路由 vs 不使用</font>

<font style="color:rgb(15, 17, 21);">我们用一个经典例子说明：一个商品列表页 (</font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">/products</font></code><font style="color:rgb(15, 17, 21);">) -> 点击进入商品详情页 (</font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">/product/1</font></code><font style="color:rgb(15, 17, 21);">)。</font>

#### <font style="color:rgb(15, 17, 21);">场景一：不使用嵌套路由（平级路由）</font>

**<font style="color:rgb(15, 17, 21);">配置方式：</font>**<font style="color:rgb(15, 17, 21);">\ </font><font style="color:rgb(15, 17, 21);">你在</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">GoRouter</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">的</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">routes</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">里平行地定义两个路由：</font>

<font style="color:rgb(15, 17, 21);">dart</font>

```dart
GoRouter(
  routes: [
    GoRoute(
      path: '/products',
      builder: (context, state) => const ProductListScreen(),
    ),
    GoRoute( // 这是一个与 /products 平级的路由
      path: '/product/:id', // 注意路径，它没有反映出从属关系
      builder: (context, state) => const ProductDetailScreen(),
    ),
  ],
);
```

**<font style="color:rgb(15, 17, 21);">导航方式：</font>**<font style="color:rgb(15, 17, 21);">\ </font><font style="color:rgb(15, 17, 21);">在列表页点击商品后，你使用</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">context.go('/product/1')</font></code><font style="color:rgb(15, 17, 21);">。</font>

**<font style="color:rgb(15, 17, 21);">会发生什么？</font>**

1. <font style="color:rgb(15, 17, 21);">GoRouter 看到 URL 从</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">/products</font></code><font style="color:rgb(15, 17, 21);"> </font>**<font style="color:rgb(15, 17, 21);">彻底改变</font>**<font style="color:rgb(15, 17, 21);">为</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">/product/1</font></code><font style="color:rgb(15, 17, 21);">。</font>
2. <font style="color:rgb(15, 17, 21);">它认为你要完全跳到一个新的、不相关的页面。</font>
3. **<font style="color:rgb(15, 17, 21);">整个页面被替换</font>**<font style="color:rgb(15, 17, 21);">！</font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">ProductListScreen</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">组件会被销毁，然后全新构建</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">ProductDetailScreen</font></code><font style="color:rgb(15, 17, 21);">。</font>
4. **<font style="color:rgb(15, 17, 21);">浏览器历史记录</font>**<font style="color:rgb(15, 17, 21);">：记录为从</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">/products</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">跳转到了</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">/product/1</font></code><font style="color:rgb(15, 17, 21);">。</font>

**<font style="color:rgb(15, 17, 21);">问题：</font>**

* **<font style="color:rgb(15, 17, 21);">用户体验差</font>**<font style="color:rgb(15, 17, 21);">：没有页面转场动画，感觉是“硬切”而不是“导航”。</font>
* **<font style="color:rgb(15, 17, 21);">丢失状态</font>**<font style="color:rgb(15, 17, 21);">：列表页被销毁，你再从详情页返回时，列表页会重新加载，之前的滚动位置、搜索状态等全部丢失。</font>
* **<font style="color:rgb(15, 17, 21);">不符合预期</font>**<font style="color:rgb(15, 17, 21);">：从逻辑上讲，详情页是依赖于列表页的，但 URL 结构没有体现这一点。</font>

#### <font style="color:rgb(15, 17, 21);">场景二：使用嵌套路由</font>

**<font style="color:rgb(15, 17, 21);">配置方式：</font>**<font style="color:rgb(15, 17, 21);">\ </font><font style="color:rgb(15, 17, 21);">你将详情页定义为列表页的</font>**<font style="color:rgb(15, 17, 21);">子路由</font>**<font style="color:rgb(15, 17, 21);">。</font>

<font style="color:rgb(15, 17, 21);">dart</font>

```dart
GoRouter(
  routes: [
    GoRoute(
      path: '/products',
      builder: (context, state) => const ProductListScreen(),
      routes: [ // 在父路由下定义子路由
        GoRoute(
          path: 'product/:id', // 注意：子路径是相对路径，不以 `/` 开头
          builder: (context, state) => const ProductDetailScreen(),
        ),
      ],
    ),
  ],
);
```

**<font style="color:rgb(15, 17, 21);">导航方式：</font>**<font style="color:rgb(15, 17, 21);">\ </font><font style="color:rgb(15, 17, 21);">在列表页点击商品后，你依然使用</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">context.go('/product/1')</font></code><font style="color:rgb(15, 17, 21);">。</font>

**<font style="color:rgb(15, 17, 21);">会发生什么？</font>**

1. <font style="color:rgb(15, 17, 21);">GoRouter 看到 URL 从</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">/products</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">变为</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">/products/product/1</font></code><font style="color:rgb(15, 17, 21);">。</font>
   * **<font style="color:rgb(15, 17, 21);">注意</font>**<font style="color:rgb(15, 17, 21);">：虽然你写的是</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">go('/product/1')</font></code><font style="color:rgb(15, 17, 21);">，但 GoRouter 会根据嵌套关系</font>**<font style="color:rgb(15, 17, 21);">自动构建出完整路径</font>**<font style="color:rgb(15, 17, 21);">。你也可以显式地写</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">go('/products/product/1')</font></code><font style="color:rgb(15, 17, 21);">。</font>
2. <font style="color:rgb(15, 17, 21);">它理解这个导航过程：</font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">/products</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">是父级，需要被保留；</font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">/product/1</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">是新的子级，需要被推送到父级的页面栈上。</font>
3. **<font style="color:rgb(15, 17, 21);">页面层级得以保持</font>**<font style="color:rgb(15, 17, 21);">：</font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">ProductListScreen</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">不会被销毁，它作为底层页面存在，</font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">ProductDetailScreen</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">会以动画形式（例如从右滑入）覆盖在其之上。</font>
4. **<font style="color:rgb(15, 17, 21);">浏览器历史记录</font>**<font style="color:rgb(15, 17, 21);">：记录为从</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">/products</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">跳转到了</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">/products/product/1</font></code><font style="color:rgb(15, 17, 21);">。</font>

**<font style="color:rgb(15, 17, 21);">优点：</font>**

* **<font style="color:rgb(15, 17, 21);">完美的用户体验</font>**<font style="color:rgb(15, 17, 21);">：有了自然的导航转场动画。</font>
* **<font style="color:rgb(15, 17, 21);">状态保持</font>**<font style="color:rgb(15, 17, 21);">：父页面（列表页）的状态得以保留，返回时无缝衔接。</font>
* **<font style="color:rgb(15, 17, 21);">URL 反映结构</font>**<font style="color:rgb(15, 17, 21);">：URL</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">/products/product/1</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">清晰地表明了信息的层级关系，既对人类可读，也对搜索引擎友好。</font>
* **<font style="color:rgb(15, 17, 21);">与浏览器历史完美集成</font>**<font style="color:rgb(15, 17, 21);">：点击浏览器后退按钮，URL 从</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">/products/product/1</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">变回</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">/products</font></code><font style="color:rgb(15, 17, 21);">，GoRouter 会正确地</font>**<font style="color:rgb(15, 17, 21);">弹出（Pop）</font>**<font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">详情页，显示出之前保留下来的列表页。</font>

***

### <font style="color:rgb(15, 17, 21);">结论与最佳实践</font>

1. **<font style="color:rgb(15, 17, 21);">不是技术上的必须，而是逻辑上的必须</font>**<font style="color:rgb(15, 17, 21);">：从技术上讲，你用平级路由</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">go()</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">也能跳转，但会得到糟糕的体验。为了获得</font>**<font style="color:rgb(15, 17, 21);">正确的导航行为和良好的用户体验</font>**<font style="color:rgb(15, 17, 21);">，你必须让你的路由配置真实地反映页面的层级逻辑关系，而这只能通过嵌套路由来实现。</font>
2. <code>**<font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">go()</font>**</code>**<font style="color:rgb(15, 17, 21);"> </font>\*\*\*\*<font style="color:rgb(15, 17, 21);">方法需要嵌套路由来发挥威力</font>**<font style="color:rgb(15, 17, 21);">：</font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">go()</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">的智能之处在于它能根据嵌套配置，决定是“替换整个导航栈”还是“推送新页面”。如果没有嵌套路由，</font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">go()</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">就无法做出智能判断，只能进行整体替换。</font>

**<font style="color:rgb(15, 17, 21);">孙子级同理</font>**<font style="color:rgb(15, 17, 21);">：如果你的应用结构是</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">Home -> List -> Detail</font></code><font style="color:rgb(15, 17, 21);">，那么你就应该进行多层嵌套：</font>

3. <font style="color:rgb(15, 17, 21);">dart</font>

```dart
GoRoute(
  path: '/',
  builder: ...,
  routes: [
    GoRoute(
      path: 'products',
      builder: ...,
      routes: [
        GoRoute(
          path: 'product/:id',
          builder: ...,
        ),
      ],
    ),
  ],
);
```

<font style="color:rgb(15, 17, 21);">这样，</font><code><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">go('/products/product/1')</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">就能在保持 Home 和 List 页面的基础上，动画地推出 Detail 页。</font>

**<font style="color:rgb(15, 17, 21);">所以，你的问题的最终答案是：</font>**

**<font style="color:rgb(15, 17, 21);">是的。为了在 Web 端使用 </font>**<code>**<font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">go()</font>**</code>**<font style="color:rgb(15, 17, 21);"> 方法时能获得与浏览器历史同步的正确导航行为（保留父级状态、拥有转场动画），子级、孙子级页面必须通过嵌套路由（Nested Routes）来配置。</font>**<font style="color:rgb(15, 17, 21);"> 这是 GoRouter 设计的精髓所在。</font>
