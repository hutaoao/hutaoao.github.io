### 组件数据必要
<font style="color:rgb(39, 56, 73);">组件的 </font><font style="color:rgb(214, 50, 0);background-color:rgb(248, 248, 248);">data</font><font style="color:rgb(39, 56, 73);"> 必须是一个函数。</font>

```javascript
Vue.component('some-comp', {
  data: function () {
    return {
      foo: 'bar'
    }
  }
})

// In a .vue file
export default {
  data () {
    return {
      foo: 'bar'
    }
  }
}

// 在一个 Vue 的根实例上直接使用对象是可以的，
// 因为只存在一个这样的实例。
new Vue({
  data: {
    foo: 'bar'
  }
})
```



### Prop 定义
<font style="color:rgb(39, 56, 73);">Prop 定义应该尽量详细。</font>

```javascript
props: {
  status: String
}

// 更好的做法！
props: {
  status: {
    type: String,
    required: true,
    validator: function (value) {
      return [
        'syncing',
        'synced',
        'version-conflict',
        'error'
      ].indexOf(value) !== -1
    }
  }
}
```



### 为v-for设置键值
<font style="color:rgb(48, 68, 85);">在组件上</font>_<font style="color:rgb(79, 89, 89);">总是</font>_<font style="color:rgb(48, 68, 85);">必须用 </font><font style="color:rgb(214, 50, 0);background-color:rgb(248, 248, 248);">key</font><font style="color:rgb(48, 68, 85);"> 配合 </font><font style="color:rgb(214, 50, 0);background-color:rgb(248, 248, 248);">v-for</font><font style="color:rgb(48, 68, 85);">，以便维护内部组件及其子树的状态。</font>

```vue
<ul>
  <li
    v-for="todo in todos"
    :key="todo.id"
  >
{% raw %}
    {{ todo.text }}
{% endraw %}
  </li>
</ul>
```



### 避免v-if和v-for用在一起
**<font style="color:rgb(39, 56, 73);">永远不要把</font>****<font style="color:rgb(39, 56, 73);"> </font>****<font style="color:rgb(214, 50, 0);background-color:rgb(248, 248, 248);">v-if</font>****<font style="color:rgb(39, 56, 73);"> </font>****<font style="color:rgb(39, 56, 73);">和</font>****<font style="color:rgb(39, 56, 73);"> </font>****<font style="color:rgb(214, 50, 0);background-color:rgb(248, 248, 248);">v-for</font>****<font style="color:rgb(39, 56, 73);"> </font>****<font style="color:rgb(39, 56, 73);">同时用在同一个元素上。</font>**

<font style="color:rgb(48, 68, 85);">一般我们在两种常见的情况下会倾向于这样做：</font>

+ <font style="color:rgb(48, 68, 85);">为了过滤一个列表中的项目 (比如</font><font style="color:rgb(48, 68, 85);"> </font><font style="color:rgb(214, 50, 0);background-color:rgb(248, 248, 248);">v-for="user in users" v-if="user.isActive"</font><font style="color:rgb(48, 68, 85);">)。在这种情形下，请将</font><font style="color:rgb(48, 68, 85);"> </font><font style="color:rgb(214, 50, 0);background-color:rgb(248, 248, 248);">users</font><font style="color:rgb(48, 68, 85);"> </font><font style="color:rgb(48, 68, 85);">替换为一个计算属性 (比如</font><font style="color:rgb(48, 68, 85);"> </font><font style="color:rgb(214, 50, 0);background-color:rgb(248, 248, 248);">activeUsers</font><font style="color:rgb(48, 68, 85);">)，让其返回过滤后的列表。</font>
+ <font style="color:rgb(48, 68, 85);">为了避免渲染本应该被隐藏的列表 (比如 </font><font style="color:rgb(214, 50, 0);background-color:rgb(248, 248, 248);">v-for="user in users" v-if="shouldShowUsers"</font><font style="color:rgb(48, 68, 85);">)。这种情形下，请将 </font><font style="color:rgb(214, 50, 0);background-color:rgb(248, 248, 248);">v-if</font><font style="color:rgb(48, 68, 85);"> 移动至容器元素上 (比如 </font><font style="color:rgb(214, 50, 0);background-color:rgb(248, 248, 248);">ul</font><font style="color:rgb(48, 68, 85);">、</font><font style="color:rgb(214, 50, 0);background-color:rgb(248, 248, 248);">ol</font><font style="color:rgb(48, 68, 85);">)。</font>

```vue
<ul>
  <li
    v-for="user in activeUsers"
    :key="user.id"
  >
{% raw %}
    {{ user.name }}
{% endraw %}
  </li>
</ul>

<ul v-if="shouldShowUsers">
  <li
    v-for="user in users"
    :key="user.id"
  >
{% raw %}
    {{ user.name }}
{% endraw %}
  </li>
</ul>
```



### 为组件样式设置作用域
**<font style="color:rgb(39, 56, 73);">对于应用来说，顶级</font>****<font style="color:rgb(39, 56, 73);"> </font>****<font style="color:rgb(214, 50, 0);background-color:rgb(248, 248, 248);">App</font>****<font style="color:rgb(39, 56, 73);"> </font>****<font style="color:rgb(39, 56, 73);">组件和布局组件中的样式可以是全局的，但是其它所有组件都应该是有作用域的。</font>**

<font style="color:rgb(48, 68, 85);">这条规则只和</font>[单文件组件](https://cn.vuejs.org/v2/guide/single-file-components.html)<font style="color:rgb(48, 68, 85);">有关。你</font>_<font style="color:rgb(79, 89, 89);">不一定</font>_<font style="color:rgb(48, 68, 85);">要使用</font><font style="color:rgb(48, 68, 85);"> </font>[scopedattribute](https://vue-loader.vuejs.org/zh-cn/features/scoped-css.html)<font style="color:rgb(48, 68, 85);">。设置作用域也可以通过</font><font style="color:rgb(48, 68, 85);"> </font>[CSS Modules](https://vue-loader.vuejs.org/zh-cn/features/css-modules.html)<font style="color:rgb(48, 68, 85);">，那是一个基于 class 的类似</font><font style="color:rgb(48, 68, 85);"> </font>[BEM](http://getbem.com/)<font style="color:rgb(48, 68, 85);"> </font><font style="color:rgb(48, 68, 85);">的策略，当然你也可以使用其它的库或约定。</font>

**<font style="color:rgb(39, 56, 73);">不管怎样，对于组件库，我们应该更倾向于选用基于 class 的策略而不是</font>****<font style="color:rgb(39, 56, 73);"> </font>****<font style="color:rgb(214, 50, 0);background-color:rgb(248, 248, 248);">scoped</font>****<font style="color:rgb(39, 56, 73);"> </font>****<font style="color:rgb(39, 56, 73);">attribute。</font>**

<font style="color:rgb(48, 68, 85);">这让覆写内部样式更容易：使用了常人可理解的 class 名称且没有太高的选择器优先级，而且不太会导致冲突。</font>

```html
<template>
  <button class="button button-close">X</button>
</template>

<!-- 使用 `scoped` attribute -->
<style scoped>
.button {
  border: none;
  border-radius: 2px;
}

.button-close {
  background-color: red;
}
</style>
```

```html
<template>
  <button :class="[$style.button, $style.buttonClose]">X</button>
</template>

<!-- 使用 CSS Modules -->
<style module>
.button {
  border: none;
  border-radius: 2px;
}

.buttonClose {
  background-color: red;
}
</style>
```

```html
<template>
  <button class="c-Button c-Button--close">X</button>
</template>

<!-- 使用 BEM 约定 -->
<style>
.c-Button {
  border: none;
  border-radius: 2px;
}

.c-Button--close {
  background-color: red;
}
</style>
```



### 私有 property 名
<font style="color:rgb(39, 56, 73);">使用模块作用域保持不允许外部访问的函数的私有性。如果无法做到这一点，就始终为插件、混入等不考虑作为对外公共 API 的自定义私有 property 使用 </font><font style="color:rgb(214, 50, 0);background-color:rgb(248, 248, 248);">$_</font><font style="color:rgb(39, 56, 73);"> 前缀。并附带一个命名空间以回避和其它作者的冲突 (比如 </font><font style="color:rgb(214, 50, 0);background-color:rgb(248, 248, 248);">$_yourPluginName_</font><font style="color:rgb(39, 56, 73);">)。</font>

```javascript
var myGreatMixin = {
  // ...
  methods: {
    $_myGreatMixin_update: function () {
      // ...
    }
  }
}
```

```javascript
// 甚至更好！
var myGreatMixin = {
  // ...
  methods: {
    publicMethod() {
      // ...
      myPrivateFunction()
    }
  }
}

function myPrivateFunction() {
  // ...
}

export default myGreatMixin
```



### 组件文件
<font style="color:rgb(39, 56, 73);">只要有能够拼接文件的构建系统，就把每个组件单独分成文件。</font>

<font style="color:rgb(48, 68, 85);">当你需要编辑一个组件或查阅一个组件的用法时，可以更快速的找到它。</font>

```javascript
components/
|- TodoList.js
|- TodoItem.js
components/
|- TodoList.vue
|- TodoItem.vue
```



### 单文件组件文件的大小写
[单文件组件](https://cn.vuejs.org/v2/guide/single-file-components.html)<font style="color:rgb(39, 56, 73);">的文件名应该要么始终是单词大写开头 (PascalCase)，要么始终是横线连接 (kebab-case)。</font>

```javascript
components/
|- MyComponent.vue
components/
|- my-component.vue
```



### 基础组件名
<font style="color:rgb(39, 56, 73);">应用特定样式和约定的基础组件 (也就是展示类的、无逻辑的或无状态的组件) 应该全部以一个特定的前缀开头，比如 </font><font style="color:rgb(214, 50, 0);background-color:rgb(248, 248, 248);">Base</font><font style="color:rgb(39, 56, 73);">、</font><font style="color:rgb(214, 50, 0);background-color:rgb(248, 248, 248);">App</font><font style="color:rgb(39, 56, 73);"> 或 </font><font style="color:rgb(214, 50, 0);background-color:rgb(248, 248, 248);">V</font><font style="color:rgb(39, 56, 73);">。</font>

```javascript
components/
|- BaseButton.vue
|- BaseTable.vue
|- BaseIcon.vue

components/
|- AppButton.vue
|- AppTable.vue
|- AppIcon.vue

components/
|- VButton.vue
|- VTable.vue
|- VIcon.vue
```



### 单例组件名
<font style="color:rgb(39, 56, 73);">只应该拥有单个活跃实例的组件应该以 </font><font style="color:rgb(214, 50, 0);background-color:rgb(248, 248, 248);">The</font><font style="color:rgb(39, 56, 73);"> 前缀命名，以示其唯一性。</font>

```javascript
components/
|- TheHeading.vue
|- TheSidebar.vue
```



### 紧密耦合的组件名
<font style="color:rgb(39, 56, 73);">和父组件紧密耦合的子组件应该以父组件名作为前缀命名。</font>

```javascript
components/
|- TodoList.vue
|- TodoListItem.vue
|- TodoListItemButton.vue

components/
|- SearchSidebar.vue
|- SearchSidebarNavigation.vue
```



### 组件名中的单词顺序
<font style="color:rgb(39, 56, 73);">组件名应该以高级别的 (通常是一般化描述的) 单词开头，以描述性的修饰词结尾。</font>

```javascript
components/
|- SearchButtonClear.vue
|- SearchButtonRun.vue
|- SearchInputQuery.vue
|- SearchInputExcludeGlob.vue
|- SettingsCheckboxTerms.vue
|- SettingsCheckboxLaunchOnStartup.vue
```



### 自闭合组件
<font style="color:rgb(39, 56, 73);">在</font>[单文件组件](https://cn.vuejs.org/v2/guide/single-file-components.html)<font style="color:rgb(39, 56, 73);">、字符串模板和 </font>[JSX](https://cn.vuejs.org/v2/guide/render-function.html#JSX)<font style="color:rgb(39, 56, 73);"> 中没有内容的组件应该是自闭合的——但在 DOM 模板里永远不要这样做。</font>

```html
<!-- 在单文件组件、字符串模板和 JSX 中 -->
<MyComponent/>

<!-- 在 DOM 模板中 -->
<my-component></my-component>
```



### 模板中的组件名大小写
<font style="color:rgb(39, 56, 73);">对于绝大多数项目来说，在</font>[单文件组件](https://cn.vuejs.org/v2/guide/single-file-components.html)<font style="color:rgb(39, 56, 73);">和字符串模板中组件名应该总是 PascalCase 的——但是在 DOM 模板中总是 kebab-case 的。</font>

```html
<!-- 在单文件组件和字符串模板中 -->
<MyComponent/>

<!-- 在 DOM 模板中 -->
<my-component></my-component>

或者
<!-- 在所有地方 -->
<my-component></my-component>
```



### JS/JSX 中的组件名大小写
<font style="color:rgb(39, 56, 73);">JS/</font>[JSX](https://cn.vuejs.org/v2/guide/render-function.html#JSX)<font style="color:rgb(39, 56, 73);"> 中的组件名应该始终是 PascalCase 的，尽管在较为简单的应用中只使用 </font><font style="color:rgb(214, 50, 0);background-color:rgb(248, 248, 248);">Vue.component</font><font style="color:rgb(39, 56, 73);"> 进行全局组件注册时，可以使用 kebab-case 字符串。</font>

```javascript
Vue.component('MyComponent', {
  // ...
})

Vue.component('my-component', {
  // ...
})

import MyComponent from './MyComponent.vue'

export default {
  name: 'MyComponent',
  // ...
}
```



### 完整单词的组件名
<font style="color:rgb(39, 56, 73);">组件名应该倾向于完整单词而不是缩写。</font>

```javascript
components/
|- StudentDashboardSettings.vue
|- UserProfileOptions.vue
```



### Prop 名大小写
<font style="color:rgb(39, 56, 73);">在声明 prop 的时候，其命名应该始终使用 camelCase，而在模板和 </font>[JSX](https://cn.vuejs.org/v2/guide/render-function.html#JSX)<font style="color:rgb(39, 56, 73);"> 中应该始终使用 kebab-case。</font>

```javascript
props: {
  greetingText: String
}
```

```html
<WelcomeMessage greeting-text="hi"/>
```



### 多个 attribute 的元素
<font style="color:rgb(39, 56, 73);">多个 attribute 的元素应该分多行撰写，每个 attribute 一行。</font>

```html
<img
  src="https://vuejs.org/images/logo.png"
  alt="Vue Logo"
>

<MyComponent
  foo="a"
  bar="b"
  baz="c"
/>
```



### 带引号的 attribute 值
<font style="color:rgb(39, 56, 73);">非空 HTML attribute 值应该始终带引号 (单引号或双引号，选你 JS 里不用的那个)。</font>

```html
<input type="text">

<AppSidebar :style="{ width: sidebarWidth + 'px' }">
```



### 指令缩写
<font style="color:rgb(39, 56, 73);">指令缩写 (用 </font><font style="color:rgb(214, 50, 0);background-color:rgb(248, 248, 248);">:</font><font style="color:rgb(39, 56, 73);"> 表示 </font><font style="color:rgb(214, 50, 0);background-color:rgb(248, 248, 248);">v-bind:</font><font style="color:rgb(39, 56, 73);">、用 </font><font style="color:rgb(214, 50, 0);background-color:rgb(248, 248, 248);">@</font><font style="color:rgb(39, 56, 73);"> 表示 </font><font style="color:rgb(214, 50, 0);background-color:rgb(248, 248, 248);">v-on:</font><font style="color:rgb(39, 56, 73);"> 和用 </font><font style="color:rgb(214, 50, 0);background-color:rgb(248, 248, 248);">#</font><font style="color:rgb(39, 56, 73);"> 表示 </font><font style="color:rgb(214, 50, 0);background-color:rgb(248, 248, 248);">v-slot:</font><font style="color:rgb(39, 56, 73);">) 应该要么都用要么都不用。</font>

```html
<input
  :value="newTodoText"
  :placeholder="newTodoInstructions"
>
<input
  v-bind:value="newTodoText"
  v-bind:placeholder="newTodoInstructions"
>

<input
  @input="onInput"
  @focus="onFocus"
>
<input
  v-on:input="onInput"
  v-on:focus="onFocus"
>

<template v-slot:header>
  <h1>Here might be a page title</h1>
</template>
<template v-slot:footer>
  <p>Here's some contact info</p>
</template>

<template #header>
  <h1>Here might be a page title</h1>
</template>
<template #footer>
  <p>Here's some contact info</p>
</template>
```
