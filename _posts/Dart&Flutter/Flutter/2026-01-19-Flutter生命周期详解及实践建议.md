# Flutter生命周期详解及实践建议

<font style="color:rgb(15, 17, 21);">好的，Flutter 的生命周期是一个核心概念，理解它对于构建健壮、高效的应用程序至关重要。Flutter 的生命周期可以分为两个层面：</font>

1. **<font style="color:rgb(15, 17, 21);">Widget 生命周期</font>**<font style="color:rgb(15, 17, 21);">：指单个 Widget 从创建到销毁的整个过程，主要由</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);">State</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">类的方法管理。</font>
2. **<font style="color:rgb(15, 17, 21);">App 生命周期</font>**<font style="color:rgb(15, 17, 21);">：指整个应用程序从启动到退出的状态变化，由</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);">WidgetsBindingObserver</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">监听。</font>

<font style="color:rgb(15, 17, 21);">下面我将为你详细讲解这两个层面。</font>

***

### <font style="color:rgb(15, 17, 21);">1. Widget 生命周期 (基于 StatefulWidget)</font>

<code><font style="color:rgb(15, 17, 21);">StatefulWidget</font></code><font style="color:rgb(15, 17, 21);"> 的生命周期围绕 </font><code><font style="color:rgb(15, 17, 21);">State</font></code><font style="color:rgb(15, 17, 21);"> 类展开。下图清晰地展示了其完整流程：</font>

#### <font style="color:rgb(15, 17, 21);">核心方法详解：</font>

1. <code>**<font style="color:rgb(15, 17, 21);">createState()</font>**</code>
   * **<font style="color:rgb(15, 17, 21);">时机</font>**<font style="color:rgb(15, 17, 21);">：当</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);">StatefulWidget</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">被插入到 Widget 树中时，Flutter 框架会立即调用其</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);">createState()</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">方法来创建对应的</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);">State</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">对象。</font>
   * **<font style="color:rgb(15, 17, 21);">用法</font>**<font style="color:rgb(15, 17, 21);">：你必须重写此方法并返回你的</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);">State</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">子类实例。通常你不会在此方法中做太多事情。</font>
2. <code>**<font style="color:rgb(15, 17, 21);">initState()</font>**</code>
   * **<font style="color:rgb(15, 17, 21);">时机</font>**<font style="color:rgb(15, 17, 21);">：在</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);">State</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">对象被创建后，</font>**<font style="color:rgb(15, 17, 21);">且只会被调用一次</font>**<font style="color:rgb(15, 17, 21);">。它是生命周期的第一个“有用”的方法。</font>
   * **<font style="color:rgb(15, 17, 21);">用法</font>**<font style="color:rgb(15, 17, 21);">：这是执行一次性初始化操作的理想位置。</font>
     * <font style="color:rgb(15, 17, 21);">初始化依赖于该</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);">State</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">对象的数据（如</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);">AnimationController</font></code><font style="color:rgb(15, 17, 21);">）。</font>
     * <font style="color:rgb(15, 17, 21);">订阅（Subscribe）到 Streams 或</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);">ChangeNotifier</font></code><font style="color:rgb(15, 17, 21);">。</font>
     * <font style="color:rgb(15, 17, 21);">发起网络请求（但需要注意，</font><code><font style="color:rgb(15, 17, 21);">build</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">方法可能会在数据返回前被调用）。</font>
   * **<font style="color:rgb(15, 17, 21);">重要</font>**<font style="color:rgb(15, 17, 21);">：你必须首先调用</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);">super.initState()</font></code><font style="color:rgb(15, 17, 21);">。</font>
3. <code>**<font style="color:rgb(15, 17, 21);">didChangeDependencies()</font>**</code>
   * **<font style="color:rgb(15, 17, 21);">时机</font>**<font style="color:rgb(15, 17, 21);">：</font>
     * <font style="color:rgb(15, 17, 21);">在</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);">initState()</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">之后立即被调用。</font>
     * <font style="color:rgb(15, 17, 21);">当此</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);">State</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">对象所依赖的\*\* InheritedWidget\*\* 发生变化时（例如，主题、Locale 等），该方法会被</font>**<font style="color:rgb(15, 17, 21);">重新调用</font>**<font style="color:rgb(15, 17, 21);">。</font>
   * **<font style="color:rgb(15, 17, 21);">用法</font>**<font style="color:rgb(15, 17, 21);">：如果你需要执行与 InheritedWidget 相关的初始化操作（例如从</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);">Provider</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">获取数据），可以在这里进行。你也可以在这里发起网络请求，但需要小心处理以免重复请求。通常更推荐在</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);">initState()</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">中初始化，然后使用</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);">Provider</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">的监听机制来更新。</font>
4. <code>**<font style="color:rgb(15, 17, 21);">build()</font>**</code>
   * **<font style="color:rgb(15, 17, 21);">时机</font>**<font style="color:rgb(15, 17, 21);">：</font>
     * <font style="color:rgb(15, 17, 21);">在</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);">didChangeDependencies()</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">之后立即调用。</font>
     * <font style="color:rgb(15, 17, 21);">每当你需要重新绘制 UI 时调用，例如：</font>
       * <font style="color:rgb(15, 17, 21);">调用</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);">setState(() {})</font></code><font style="color:rgb(15, 17, 21);">。</font>
       * <font style="color:rgb(15, 17, 21);">父 Widget 重建并传入了新的配置（</font><code><font style="color:rgb(15, 17, 21);">widget.someValue</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">改变）。</font>
       * <font style="color:rgb(15, 17, 21);">所依赖的 InheritedWidget 发生变化。</font>
   * **<font style="color:rgb(15, 17, 21);">用法</font>**<font style="color:rgb(15, 17, 21);">：这是一个</font>**<font style="color:rgb(15, 17, 21);">必须</font>**<font style="color:rgb(15, 17, 21);">重写的方法。它根据当前的状态（State）和配置（Widget）来描述用户界面。这个方法应该是一个</font>**<font style="color:rgb(15, 17, 21);">纯函数</font>**<font style="color:rgb(15, 17, 21);">，没有副作用（例如，不要在这里修改状态或发起网络请求）。</font>
5. <code>**<font style="color:rgb(15, 17, 21);">didUpdateWidget(oldWidget)</font>**</code>
   * **<font style="color:rgb(15, 17, 21);">时机</font>**<font style="color:rgb(15, 17, 21);">：当父 Widget 重建并重新传入一个</font>**<font style="color:rgb(15, 17, 21);">相同运行时类型</font>**<font style="color:rgb(15, 17, 21);">的</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);">Widget</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">时，Flutter 会对比新旧 Widget 的配置。如果配置发生了变化，它会更新当前的</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);">widget</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">属性，然后调用此方法。</font>
   * **<font style="color:rgb(15, 17, 21);">用法</font>**<font style="color:rgb(15, 17, 21);">：用于响应 Widget 配置的更改。例如，如果旧的 Widget 有一个</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);">AnimationController</font></code><font style="color:rgb(15, 17, 21);">，而新的 Widget 传入了不同的动画参数，你可以在这里停止旧的并初始化新的。通常，你不需要重写此方法，因为</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);">State</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">对象会自动根据新的</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);">widget.*</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">属性在</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);">build()</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">方法中重建。但在处理复杂状态（如动画）时非常有用。</font>
   * **<font style="color:rgb(15, 17, 21);">对比</font>**<font style="color:rgb(15, 17, 21);">：</font><code><font style="color:rgb(15, 17, 21);">setState()</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">是内部状态变化，</font><code><font style="color:rgb(15, 17, 21);">didUpdateWidget</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">是外部配置变化。</font>
6. <code>**<font style="color:rgb(15, 17, 21);">setState()</font>**</code>
   * **<font style="color:rgb(15, 17, 21);">时机</font>**<font style="color:rgb(15, 17, 21);">：当你需要改变</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);">State</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">对象内部的数据（状态）并希望 UI 随之更新时，你需要在修改数据的回调函数中调用它。</font>
   * **<font style="color:rgb(15, 17, 21);">用法</font>**<font style="color:rgb(15, 17, 21);">：它通知框架：“我的状态已经改变了，请重新调用</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);">build()</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">方法来更新 UI”。你只能在当前 State 对象是</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);">mounted</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">的时候调用它。</font>
7. <code>**<font style="color:rgb(15, 17, 21);">deactivate()</font>**</code>
   * **<font style="color:rgb(15, 17, 21);">时机</font>**<font style="color:rgb(15, 17, 21);">：当</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);">State</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">对象从 Widget 树中</font>**<font style="color:rgb(15, 17, 21);">暂时移除</font>**<font style="color:rgb(15, 17, 21);">时调用。最常见的情况是它在树中的位置被另一个 Widget 替换了。</font>**<font style="color:rgb(15, 17, 21);">注意</font>**<font style="color:rgb(15, 17, 21);">：同一个 Widget 可以在树的不同位置被移除和重新插入。</font>
   * **<font style="color:rgb(15, 17, 21);">用法</font>**<font style="color:rgb(15, 17, 21);">：很少使用。可用于在 Widget 暂时消失前保存一些状态。</font>
8. <code>**<font style="color:rgb(15, 17, 21);">dispose()</font>**</code>
   * **<font style="color:rgb(15, 17, 21);">时机</font>**<font style="color:rgb(15, 17, 21);">：当</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);">State</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">对象</font>**<font style="color:rgb(15, 17, 21);">被永久地从 Widget 树中移除</font>**<font style="color:rgb(15, 17, 21);">时调用。这是生命周期的终点，该方法</font>**<font style="color:rgb(15, 17, 21);">只会被调用一次</font>**<font style="color:rgb(15, 17, 21);">。</font>
   * **<font style="color:rgb(15, 17, 21);">用法</font>**<font style="color:rgb(15, 17, 21);">：</font>**<font style="color:rgb(15, 17, 21);">这是进行清理工作的关键位置</font>**<font style="color:rgb(15, 17, 21);">，必须在此处释放所有资源以避免内存泄漏。</font>
     * <font style="color:rgb(15, 17, 21);">取消计时器（</font><code><font style="color:rgb(15, 17, 21);">Timer</font></code><font style="color:rgb(15, 17, 21);">）。</font>
     * <font style="color:rgb(15, 17, 21);">取消动画（</font><code><font style="color:rgb(15, 17, 21);">AnimationController.dispose()</font></code><font style="color:rgb(15, 17, 21);">）。</font>
     * <font style="color:rgb(15, 17, 21);">取消对 Streams 或</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);">ChangeNotifier</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">的订阅。</font>
   * **<font style="color:rgb(15, 17, 21);">重要</font>**<font style="color:rgb(15, 17, 21);">：你必须调用</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);">super.dispose()</font></code><font style="color:rgb(15, 17, 21);">。</font>

#### <code><font style="color:rgb(15, 17, 21);">mounted</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">属性</font>

* <font style="color:rgb(15, 17, 21);">这是</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);">State</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">类的一个</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);">bool</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">属性。当</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);">createState()</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">被调用后，</font><code><font style="color:rgb(15, 17, 21);">mounted</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">被设为</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);">true</font></code><font style="color:rgb(15, 17, 21);">。当</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);">dispose()</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">被调用后，它被设为</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);">false</font></code><font style="color:rgb(15, 17, 21);">。</font>
* **<font style="color:rgb(15, 17, 21);">重要规则</font>**<font style="color:rgb(15, 17, 21);">：在调用</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);">setState()</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">之前，</font>**<font style="color:rgb(15, 17, 21);">必须</font>**<font style="color:rgb(15, 17, 21);">检查</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);">if (mounted) { ... }</font></code><font style="color:rgb(15, 17, 21);">，尤其是在异步回调中（如网络请求、</font><code><font style="color:rgb(15, 17, 21);">Future.delayed</font></code><font style="color:rgb(15, 17, 21);">）。因为在回调执行时，Widget 可能已经被销毁（</font><code><font style="color:rgb(15, 17, 21);">dispose</font></code><font style="color:rgb(15, 17, 21);">），此时调用</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);">setState()</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">会抛出异常。</font>

***

### <font style="color:rgb(15, 17, 21);">2. App 生命周期</font>

<font style="color:rgb(15, 17, 21);">App 生命周期指的是整个应用程序所处的状态（如前台运行、退到后台、挂起等）。你可以通过混入</font><font style="color:rgb(15, 17, 21);"> </font><code><font style="color:rgb(15, 17, 21);">WidgetsBindingObserver</font></code><font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">来监听这些状态变化。</font>

#### <font style="color:rgb(15, 17, 21);">使用方法：</font>

**<font style="color:rgb(15, 17, 21);">State 类混入</font>\*\*\*\*<font style="color:rgb(15, 17, 21);"> </font>**<code>**<font style="color:rgb(15, 17, 21);">WidgetsBindingObserver</font>**</code><font style="color:rgb(15, 17, 21);">：</font>

1. dart

```dart
class _MyHomePageState extends State<MyHomePage> with WidgetsBindingObserver {
  // ...
}
```

**<font style="color:rgb(15, 17, 21);">在</font>\*\*\*\*<font style="color:rgb(15, 17, 21);"> </font>**<code>**<font style="color:rgb(15, 17, 21);">initState</font>**</code>**<font style="color:rgb(15, 17, 21);"> </font>\*\*\*\*<font style="color:rgb(15, 17, 21);">中注册监听器</font>**<font style="color:rgb(15, 17, 21);">：</font>

2. dart

```dart
@override
void initState() {
  super.initState();
  WidgetsBinding.instance.addObserver(this); // 注册监听
}
```

**<font style="color:rgb(15, 17, 21);">重写</font>\*\*\*\*<font style="color:rgb(15, 17, 21);"> </font>**<code>**<font style="color:rgb(15, 17, 21);">didChangeAppLifecycleState</font>**</code>**<font style="color:rgb(15, 17, 21);"> </font>\*\*\*\*<font style="color:rgb(15, 17, 21);">方法</font>**<font style="color:rgb(15, 17, 21);">：</font>

3. dart

```dart
@override
void didChangeAppLifecycleState(AppLifecycleState state) {
  super.didChangeAppLifecycleState(state);
  print('App state changed to: $state');
  // 根据不同的状态执行操作
  switch (state) {
    case AppLifecycleState.inactive:
      // 应用处于非活动状态，无法接收用户输入（如接电话、分屏）。
      break;
    case AppLifecycleState.paused:
      // 应用对用户不可见，进入后台。
      // 暂停动画、视频、释放不必要的资源。
      break;
    case AppLifecycleState.resumed:
      // 应用重新可见，回到前台。
      // 重新开始动画、恢复逻辑。
      break;
    case AppLifecycleState.detached:
      // 应用仍托管在Flutter引擎上但已从宿主View中分离（很少见）。
      break;
    case AppLifecycleState.hidden:
      // 应用完全隐藏（如切换到其他应用，但未完全进入后台栈，某些系统）。
      break;
  }
}
```

**<font style="color:rgb(15, 17, 21);">在</font>\*\*\*\*<font style="color:rgb(15, 17, 21);"> </font>**<code>**<font style="color:rgb(15, 17, 21);">dispose</font>**</code>**<font style="color:rgb(15, 17, 21);"> </font>\*\*\*\*<font style="color:rgb(15, 17, 21);">中取消监听</font>**<font style="color:rgb(15, 17, 21);">：</font>

4. dart

```dart
@override
void dispose() {
  WidgetsBinding.instance.removeObserver(this); // 取消监听
  super.dispose();
}
```

### <font style="color:rgb(15, 17, 21);">总结与实践建议</font>

| <font style="color:rgb(15, 17, 21);">方法</font> | <font style="color:rgb(15, 17, 21);">调用次数</font> | <font style="color:rgb(15, 17, 21);">常用用途</font> |

| <code>**<font style="color:rgb(15, 17, 21);">didChangeDependencies</font>**</code> | <font style="color:rgb(15, 17, 21);">≥1次</font> | <font style="color:rgb(15, 17, 21);">处理与 InheritedWidget 相关的初始化或变化。</font> |
| <code>**<font style="color:rgb(15, 17, 21);">build</font>**</code> | <font style="color:rgb(15, 17, 21);">≥1次</font> | <font style="color:rgb(15, 17, 21);">构建UI。必须是纯函数。</font> |
| <code>**<font style="color:rgb(15, 17, 21);">didUpdateWidget</font>**</code> | <font style="color:rgb(15, 17, 21);">≥0次</font> | <font style="color:rgb(15, 17, 21);">响应外部配置（Widget）的变化，更新内部状态。</font> |
| <code>**<font style="color:rgb(15, 17, 21);">setState</font>**</code> | <font style="color:rgb(15, 17, 21);">≥0次</font> | <font style="color:rgb(15, 17, 21);">通知框架内部状态已变，需要重建UI。</font> |
| <code>**<font style="color:rgb(15, 17, 21);">deactivate</font>**</code> | <font style="color:rgb(15, 17, 21);">≥0次</font> | <font style="color:rgb(15, 17, 21);">Widget 被暂时移除时保存临时状态。</font> |

| <code>**<font style="color:rgb(15, 17, 21);">didChangeAppLifecycleState</font>**</code> | <font style="color:rgb(15, 17, 21);">≥0次</font> | <font style="color:rgb(15, 17, 21);">监听App全局状态，管理全局资源（如摄像头、音频）。</font> |
