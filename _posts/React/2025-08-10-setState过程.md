# setState 过程

<font style="color:rgb(30, 31, 36);">在 React 中，setState 方法用于更新组件的状态。当你调用 setState 时，React 会开始一个更新组件的过程。以下是 setState 过程的大致步骤：</font>

1. **<font style="color:rgb(30, 31, 36);">合并新状态</font>**<font style="color:rgb(30, 31, 36);">：</font>
    - <font style="color:rgb(30, 31, 36);">React 首先会将你传递给</font><font style="color:rgb(30, 31, 36);"> </font><font style="color:rgb(30, 31, 36);">setState</font><font style="color:rgb(30, 31, 36);"> </font><font style="color:rgb(30, 31, 36);">的新状态（对象或函数）与当前的状态进行合并。</font>
    - <font style="color:rgb(30, 31, 36);">如果你传递的是一个函数，这个函数会接收当前的状态和属性作为参数，并返回一个新的状态对象。</font>
2. **<font style="color:rgb(30, 31, 36);">调度更新</font>**<font style="color:rgb(30, 31, 36);">：</font>
    - <font style="color:rgb(30, 31, 36);">React 并不会立即更新组件，而是将更新放入一个队列中。这是为了优化性能，因为 React 可能会批量处理多个</font><font style="color:rgb(30, 31, 36);"> </font><font style="color:rgb(30, 31, 36);">setState</font><font style="color:rgb(30, 31, 36);"> </font><font style="color:rgb(30, 31, 36);">调用，从而减少不必要的重新渲染。</font>
    - <font style="color:rgb(30, 31, 36);">实际上，React 使用了调度器（Scheduler）和合成事件系统（SyntheticEvent System）来确保更新在适当的时候被处理。</font>
3. **<font style="color:rgb(30, 31, 36);">开始 React 更新流程</font>**<font style="color:rgb(30, 31, 36);">：</font>
    - <font style="color:rgb(30, 31, 36);">当 React 准备处理队列中的更新时，它会开始 React 的更新流程。</font>
    - <font style="color:rgb(30, 31, 36);">这个流程包括确定哪些组件需要重新渲染（基于状态变化和 props 的变化）。</font>
4. **<font style="color:rgb(30, 31, 36);">调用生命周期方法（类组件）</font>**<font style="color:rgb(30, 31, 36);">：</font>
    - <font style="color:rgb(30, 31, 36);">对于类组件，React 会在更新过程中调用相应的生命周期方法。</font>
    - <font style="color:rgb(30, 31, 36);">在 React 16.3 之前，这些方法可能包括</font><font style="color:rgb(30, 31, 36);"> </font><font style="color:rgb(30, 31, 36);">componentWillUpdate</font><font style="color:rgb(30, 31, 36);">（已被弃用）和</font><font style="color:rgb(30, 31, 36);"> </font><font style="color:rgb(30, 31, 36);">componentDidUpdate</font><font style="color:rgb(30, 31, 36);">。</font>
    - <font style="color:rgb(30, 31, 36);">在 React 16.3 及更高版本中，</font><font style="color:rgb(30, 31, 36);">componentWillUpdate</font><font style="color:rgb(30, 31, 36);"> </font><font style="color:rgb(30, 31, 36);">已被弃用，并引入了新的生命周期方法，如</font><font style="color:rgb(30, 31, 36);"> </font><font style="color:rgb(30, 31, 36);">getDerivedStateFromProps</font><font style="color:rgb(30, 31, 36);"> </font><font style="color:rgb(30, 31, 36);">和</font><font style="color:rgb(30, 31, 36);"> </font><font style="color:rgb(30, 31, 36);">getSnapshotBeforeUpdate</font><font style="color:rgb(30, 31, 36);">。</font>
5. **<font style="color:rgb(30, 31, 36);">重新渲染</font>**<font style="color:rgb(30, 31, 36);">：</font>
    - <font style="color:rgb(30, 31, 36);">React 会基于新的状态和 props 来重新渲染受影响的组件。</font>
    - <font style="color:rgb(30, 31, 36);">这通常涉及调用组件的</font><font style="color:rgb(30, 31, 36);"> </font><font style="color:rgb(30, 31, 36);">render</font><font style="color:rgb(30, 31, 36);"> </font><font style="color:rgb(30, 31, 36);">方法来生成新的 React 元素树。</font>
6. **<font style="color:rgb(30, 31, 36);">DOM 更新</font>**<font style="color:rgb(30, 31, 36);">：</font>
    - <font style="color:rgb(30, 31, 36);">React 会使用 React 的 Diffing 算法（Reconciliation）来比较新的 React 元素树和旧的 DOM 树，以找出需要更改的部分。</font>
    - <font style="color:rgb(30, 31, 36);">然后，React 会更新这些部分的 DOM，而不是重新渲染整个页面。</font>
7. **<font style="color:rgb(30, 31, 36);">调用生命周期方法（类组件）</font>**<font style="color:rgb(30, 31, 36);">：</font>
    - <font style="color:rgb(30, 31, 36);">在 DOM 更新后，React 会调用类组件的</font><font style="color:rgb(30, 31, 36);"> </font><font style="color:rgb(30, 31, 36);">componentDidUpdate</font><font style="color:rgb(30, 31, 36);"> </font><font style="color:rgb(30, 31, 36);">生命周期方法（如果定义了的话）。</font>
8. **<font style="color:rgb(30, 31, 36);">回调</font>**<font style="color:rgb(30, 31, 36);">：</font>
    - <font style="color:rgb(30, 31, 36);">如果你在调用</font><font style="color:rgb(30, 31, 36);"> </font><font style="color:rgb(30, 31, 36);">setState</font><font style="color:rgb(30, 31, 36);"> </font><font style="color:rgb(30, 31, 36);">时传递了一个回调函数作为第二个参数，这个回调将在更新被应用到 DOM 并且屏幕被更新后执行。</font>

<font style="color:rgb(30, 31, 36);"></font>
