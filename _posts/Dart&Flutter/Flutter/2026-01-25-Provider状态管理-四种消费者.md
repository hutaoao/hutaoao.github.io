# Provider状态管理-四种消费者

## Provider.of

Provider.of<T>(context)<font style="color:rgb(52, 52, 60);">是</font>Provider<font style="color:rgb(52, 52, 60);">为我们提供的静态方法，当我们使用该方法去获取值的时候会返回查找到的最近的</font>T<font style="color:rgb(52, 52, 60);">类型的</font>provider<font style="color:rgb(52, 52, 60);">给我们，而且也不会遍历整个组件树，下面我们看下代码：</font>

### 第一步：定义模型

<font style="color:rgb(52, 52, 60);">我们定义了一个</font>CountNotifier1<font style="color:rgb(52, 52, 60);">的模型，后面所有的示例代码将围绕该模型来演示</font>

```dart
import 'package:flutter/material.dart';

class CountNotifier1 with ChangeNotifier {

  int count = 0;

  void increment() {
    count++;
    notifyListeners();
  }
}
```

### 第二步：应用程序入口设置

```dart
return ChangeNotifierProvider(
  create: (_) => CountNotifier1(),
  child: MaterialApp(
    debugShowCheckedModeBanner: false,
    home: ConsumerExample(),
  ),
);
```

### 第三步：使用Provider.of

<font style="color:rgb(52, 52, 60);">这里读取值和点击按钮+1时都是通过</font>Provider.of<T>()<font style="color:rgb(52, 52, 60);">来获取及使用。</font>

```dart
import 'package:flutter/material.dart';
import 'package:flutter_provider_example/consumer_example/count_notifier1.dart';
import 'package:provider/provider.dart';

class ConsumerExample extends StatelessWidget {

  @override
  Widget build(BuildContext context) {

    return Scaffold(
      appBar: AppBar(
        title: Text("ConsumerExample"),
      ),
      body: Center(
        child: Column(
          mainAxisAlignment: MainAxisAlignment.center,
          children: [
            Text(Provider.of<CountNotifier1>(context).count.toString(),
                 style: TextStyle(
                   color: Colors.red,
                   fontSize: 50
                 ),
                ),
            Padding(
              padding: EdgeInsets.only(
                top: 20
              ),
              child: ElevatedButton(
                onPressed: (){
                  Provider.of<CountNotifier1>(context).increment();
                },
                child: Text("点击加1"),
              ),
            )
          ],
        ),
      ),
    );
  }
}
```

<font style="color:rgb(52, 52, 60);">当我们运行代码的时候会提示一个报错，它提示说试图从</font>Widget<font style="color:rgb(52, 52, 60);">树外部监听提供者公开的值，如果要修复可以把</font>listen<font style="color:rgb(52, 52, 60);">改成</font>false<font style="color:rgb(52, 52, 60);">，这个问题其实是在</font>Provider 4.0.2<font style="color:rgb(52, 52, 60);">后会出现的，最主要的是它的默认行为就是</font>ture<font style="color:rgb(52, 52, 60);">，错误日志如下：</font>

## Consumer

Consumber<font style="color:rgb(52, 52, 60);">只是在</font>Widget<font style="color:rgb(52, 52, 60);">中调用了</font>Prvoider.of<font style="color:rgb(52, 52, 60);">，并将其构造实现委托给了构造器，比如我们常见的</font>Builder<font style="color:rgb(52, 52, 60);">，如果你的</font>Widget<font style="color:rgb(52, 52, 60);">依赖多个模型，那么它还提供了</font>Consumer23456<font style="color:rgb(52, 52, 60);">方便调用，我们接下来对上面的案例采用</font>Consumer<font style="color:rgb(52, 52, 60);">来修改</font>

### 用Consumer包裹组件

<font style="color:rgb(52, 52, 60);">里面有个</font>builder<font style="color:rgb(52, 52, 60);">构造器，当我们把</font>body<font style="color:rgb(52, 52, 60);">改成下面重新运行后可以发现和使用</font>Provider.of<font style="color:rgb(52, 52, 60);">的结果一样，但是这里不需要在像使用</font>Provider.of<font style="color:rgb(52, 52, 60);">那样每次使用都要写一大串的重复性代码。</font>

<font style="color:rgb(52, 52, 60);">里面有三个属性：</font>

* **<font style="color:rgb(52, 52, 60);">context:</font>**<font style="color:rgb(52, 52, 60);"> </font><font style="color:rgb(52, 52, 60);">当前的上下文</font>
* <font style="color:rgb(52, 52, 60);">**Provider.of</font><font style="color:rgb(52, 52, 60);">(context):** 模型对象</font>
* **<font style="color:rgb(52, 52, 60);">child:</font>**<font style="color:rgb(52, 52, 60);"> </font><font style="color:rgb(52, 52, 60);">子组件(不需要刷新的部分)</font>

```dart
body: Consumer(
  builder: (_, CountNotifier1 countNotifier1, child) {
    return Center(
      child: Column(
        mainAxisAlignment: MainAxisAlignment.center,
        children: [
          Text(countNotifier1.count.toString(),
               style: TextStyle(
                 color: Colors.red,
                 fontSize: 50
               ),
              ),
          Padding(
            padding: EdgeInsets.only(
              top: 20
            ),
            child: ElevatedButton(
              onPressed: (){
                countNotifier1.increment();
              },
              child: Text("点击加1"),
            ),
          )
        ],
      ),
    );
  },
),
```

### 优化Consumer

#### 优化方式一：尽可能调整Consumer的位置

<font style="color:rgb(52, 52, 60);">我们在上面的代码中发现</font>Center<font style="color:rgb(52, 52, 60);">以及</font>Column<font style="color:rgb(52, 52, 60);">组件也被</font>Consumer<font style="color:rgb(52, 52, 60);">包裹了进来，但是这两个组件是不需要更新状态的，而我们每次构建的</font>Widget<font style="color:rgb(52, 52, 60);">的时候，会重建整个</font>body<font style="color:rgb(52, 52, 60);">，所以我们优化一下代码结构，看起来就像下面这样：</font>

```dart
body: Center(
  child: Consumer(
    builder: (_, CountNotifier1 countNotifier1, child) {
      return Center(
        child: Column(

            ),
            Container(
              child: Column(
                children: [
                  Text("更多组件1"),

                  Text("更多组件4"),
                  Text("更多组件5"),
```

#### 优化方式二：不需要刷新但被Consumer包裹的组件用child

<font style="color:rgb(52, 52, 60);">比如上面我们有更多组件1-6甚至数百个组件无需刷新状态，但由于你用</font>Consumer<font style="color:rgb(52, 52, 60);">包裹会导致全部刷新，那么明显会导致性能的下降，你可能会想到单独用多个</font>Consumer<font style="color:rgb(52, 52, 60);">包裹需要刷新的组件就解决了，但这不就是本末倒置了吗，本身</font>Provider<font style="color:rgb(52, 52, 60);">是解决代码的</font>**<font style="color:rgb(52, 52, 60);">健壮</font>**<font style="color:rgb(52, 52, 60);">、</font>**<font style="color:rgb(52, 52, 60);">重复</font>**<font style="color:rgb(52, 52, 60);">的代码，所以这个时候我们可以采用</font>Consumer<font style="color:rgb(52, 52, 60);">为我们提供的</font>child<font style="color:rgb(52, 52, 60);">参数，如下：</font>

## Selector

Selector<font style="color:rgb(52, 52, 60);">类和</font>Consumer<font style="color:rgb(52, 52, 60);">类似，只是对</font>build<font style="color:rgb(52, 52, 60);">调用</font>Widget<font style="color:rgb(52, 52, 60);">方法时提供更精细的控制，简单点来说，</font>Selector<font style="color:rgb(52, 52, 60);">也是一个消费者，它允许你可以从模型中准备定义哪些属性。</font>

<font style="color:rgb(52, 52, 60);">我们来举个例子：</font>

<font style="color:rgb(52, 52, 60);">比如，用户模型中有50个属性，但是我只需要更新年龄，这样我希望不需要重建</font>**<font style="color:rgb(52, 52, 60);">用户名</font>**<font style="color:rgb(52, 52, 60);">、</font>**<font style="color:rgb(52, 52, 60);">电话号码</font>**<font style="color:rgb(52, 52, 60);">等组件，那么</font>Selector<font style="color:rgb(52, 52, 60);">就是用于解决这个问题，我们看一下示例：</font>

### 第一步：定义模型

```dart
import 'package:flutter/material.dart';

class UserModel6 with ChangeNotifier {

  String name = "Jimi";
  int age = 18;
  String phone = "18888888888";


  void increaseAge() {
    age++;
    notifyListeners();
  }
}
```

### 第二步：应用程序入口设置

```dart
return ChangeNotifierProvider(
  create: (_) => UserModel6(),
  child: MaterialApp(
    debugShowCheckedModeBanner: false,
    home: SelectorExample(),
  ),
);
```

### 第三步：使用Selector更精细的控制

```dart
import 'package:flutter/material.dart';
import 'package:flutter_provider_example/selector_example/user_model6.dart';
import 'package:provider/provider.dart';

class SelectorExample extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text("SelectorExample"),
      ),
      body: Center(
        child: Selector<UserModel6, int>(
          selector: (_, userModel6) => userModel6.age,
          builder: (_, age, child) {
            return Column(
              mainAxisAlignment: MainAxisAlignment.center,
              children: [
                Text(age.toString(),
                     style: TextStyle(
                       color: Colors.red,
                       fontSize: 30
                     )
                    ),
                child!
              ],
            );
          },
          child: Padding(
            padding: EdgeInsets.all(20),
            child: ElevatedButton(
              onPressed: (){
                Provider.of<UserModel6>(context, listen: false).increaseAge();
              },
              child: Text("改变年龄"),
            ),
          ),
        ),
      ),
    );
  }
}
```

## InheritedContext

InheritedContext<font style="color:rgb(52, 52, 60);">是</font>Provider<font style="color:rgb(52, 52, 60);">内置扩展了</font>BuildContext<font style="color:rgb(52, 52, 60);">，它不保存了组件在树中自己位置的引用，我们在上面的案例中见到</font>Provider.of<CountNotifier1>(context，listen: false)<font style="color:rgb(52, 52, 60);">，其实这个</font>of<font style="color:rgb(52, 52, 60);">方法就是使用</font>Flutter<font style="color:rgb(52, 52, 60);">查找树并找到</font>Provider<font style="color:rgb(52, 52, 60);">子类型为</font>CountNotifier1<font style="color:rgb(52, 52, 60);">而已。</font>

### 三大方式：

* **<font style="color:rgb(52, 52, 60);">BuildContext.read:</font>**<font style="color:rgb(52, 52, 60);"> </font>BuildContext.read<CountNotifier1>()<font style="color:rgb(52, 52, 60);">可以替换掉</font>Provider.of<CountNotifier1>(context，listen: false)<font style="color:rgb(52, 52, 60);">，它会找到</font>CountNotifier1<font style="color:rgb(52, 52, 60);">并返回它。</font>
* **<font style="color:rgb(52, 52, 60);">BuildContext.watch:</font>**<font style="color:rgb(52, 52, 60);"> </font>BuildContext.watch<CountNotifier1>()<font style="color:rgb(52, 52, 60);">可以替换掉</font>Provider.of<CountNotifier1>(context，listen: false)<font style="color:rgb(52, 52, 60);">，看起来和</font>read<font style="color:rgb(52, 52, 60);">没有什么不同，但是使用</font>watch<font style="color:rgb(52, 52, 60);">你就不需要在使用</font>Consumer<font style="color:rgb(52, 52, 60);">。</font>
* **<font style="color:rgb(52, 52, 60);">BuildContext.select:</font>**<font style="color:rgb(52, 52, 60);"> </font>BuildContext.select<CountNotifier1>()<font style="color:rgb(52, 52, 60);">可以替换掉</font>Provider.of<CountNotifier1>(context，listen: false)<font style="color:rgb(52, 52, 60);">，看起来和</font>watch<font style="color:rgb(52, 52, 60);">也没有什么不同，但是使用</font>select<font style="color:rgb(52, 52, 60);">你就不需要在使用</font>Selector<font style="color:rgb(52, 52, 60);">。</font>

> <code>**<font style="color:#DF2A3F;">context.watch<T>()</font>**</code>**<font style="color:#DF2A3F;"> - 监听T上的更改</font>**
> <code>**<font style="color:#DF2A3F;">context.read<T>()</font>**</code>**<font style="color:#DF2A3F;"> - 返回T而不侦听它</font>**

### BuildContext.read
