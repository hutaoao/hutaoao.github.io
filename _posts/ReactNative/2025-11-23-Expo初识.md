### 优势

* \*\*React Native集成：\*\*Expo基于Facebook的React Native框架，允许开发者用JavaScript和React语法编写跨平台的应用。
* \*\*统一的SDK：\*\*Expo有自己的SDK，其中包含了许多预封装的模块，如地图、推送通知、相机和音频等，使得添加这些常见功能变得简单易行。
* \*\*热更新（Hot Reloading）：\*\*Expo支持实时重载，极大地提高了开发效率，开发者可以迅速看到代码更改的效果。
* \*\*原生模块支持：\*\*虽然大部分功能可以通过SDK实现，但当需要自定义原生模块时，Expo的eject功能可以让应用脱离Expo环境，转为完全的React Native项目，从而可以直接进行原生代码的修改。
* \*\*即用型工具：\*\*包括二维码扫描、设备检测、远程控制等功能，使得调试和分享应用程序变得更加简单。
* **<font style="color:rgb(51, 51, 51);">基于文件系统的本机路由：</font>**`expo-router`

### <font style="color:rgb(42, 43, 46);">环境变量</font>

#### .env

<font style="color:rgb(17, 24, 28);">Expo CLI 将自动从</font>**.env**<code><font style="color:rgb(17, 24, 28);">EXPO_PUBLIC_</font></code><font style="color:rgb(17, 24, 28);">文件中加载带有前缀的环境变量以供在您的 JavaScript 代码中使用。</font>

<font style="color:rgb(17, 24, 28);">在项目目录的根目录中创建一个</font>**<font style="color:rgb(17, 24, 28);">.env</font>**<font style="color:rgb(17, 24, 28);">文件，并在新行中以以下形式添加特定于环境的变量</font><code><font style="color:rgb(17, 24, 28);">EXPO_PUBLIC_[NAME]=VALUE</font></code><font style="color:rgb(17, 24, 28);">：</font>

```plain
EXPO_PUBLIC_API_URL=https://staging.example.com
EXPO_PUBLIC_API_KEY=abc123
```

现在您可以在源代码中直接使用环境变量：

```javascript
import { Button } from 'react-native';

function Post() {
  const apiUrl = process.env.EXPO_PUBLIC_API_URL;

  async function onPress() {
    await fetch(apiUrl, { ... })
  }

  return <Button onPress={onPress} title="Post" />;
}
```

<font style="color:rgb(17, 24, 28);">当您运行 时</font><code><font style="color:rgb(17, 24, 28);">npx expo start</font></code><font style="color:rgb(17, 24, 28);">，应用程序包中的</font><code><font style="color:rgb(17, 24, 28);">process.env.EXPO_PUBLIC_API_URL</font></code><font style="color:rgb(17, 24, 28);">将被替换为</font><code><font style="color:rgb(17, 24, 28);">https://staging.example.com</font></code><font style="color:rgb(17, 24, 28);">。您可以在编辑代码时更新变量，而无需重新启动 Expo CLI 或清除缓存。您需要执行完全重新加载（例如，摇动手势，然后在 Expo Go 或您的开发版本中重新加载）才能看到更新的值。</font>

> <font style="color:rgb(17, 24, 28);background-color:rgb(255, 251, 209);">不要将敏感信息（例如私钥）存储在</font><code><font style="color:rgb(17, 24, 28);">EXPO_PUBLIC_</font></code><font style="color:rgb(17, 24, 28);background-color:rgb(255, 251, 209);">变量中。这些变量将以纯文本形式显示在已编译的应用程序中。</font>

#### 使用多个 .env 文件定义单独的环境

可以有单独的\*\*.env\*\*、**.env.development**、**.env.production**等文件，它们将根据标准优先级加载。

如果你创建了特定于环境的文件，例如\*\*.env.\*\***production**，则可以`NODE_ENV`在运行 Expo CLI 时通过设置来加载它：

<code>**<font style="color:#DF2A3F;">NODE_ENV=production npx expo start</font>**</code>

#### Expo Application Services 中的环境变量

如果您按照[环境变量指南](https://docs.expo.dev/guides/environment-variables)中的说明在\*\*.env**文件中为本地开发设置变量，则可以在**eas.json\*\*中的构建配置文件中设置相同的变量。例如，您可以在本地开发时将 API URL 变量设置为本地后端服务器，在测试时将 API URL 变量设置为测试服务器，在生产构建时将 API URL 变量设置为生产服务器。

在这种情况下，你的\*\*.env\*\*文件可能如下所示：

```javascript
EXPO_PUBLIC_API_URL=http://api.local
```

<font style="color:rgb(17, 24, 28);">将任何适用的</font>**<font style="color:rgb(17, 24, 28);">.env</font>**<font style="color:rgb(17, 24, 28);">文件添加到您的</font>**<font style="color:rgb(17, 24, 28);">.gitignore</font>**<font style="color:rgb(17, 24, 28);">（和</font>**<font style="color:rgb(17, 24, 28);">.easignore</font>**<font style="color:rgb(17, 24, 28);">，如果您的项目有的话）文件中，这样它们就不会随您的 EAS Build 作业一起上传：</font>

```javascript
# ignores all .env files
.env*
```

**然后，您可以在eas.json**中为每个构建配置文件设置相同的环境变量：

**eas.json**

```json
{
  "build": {
    "production": {
      "env": {
        "EXPO_PUBLIC_API_URL": "https://api.production.com"
      }
    },
    "test": {
      "env": {
        "EXPO_PUBLIC_API_URL": "https://api.test.com"
      }
    }
  }
}
```

任何引用都`process.env.EXPO_PUBLIC_API_URL`将根据环境替换为适用的值。**<font style="color:rgb(17, 24, 28);"></font>**

> ‼️本地的 **.env**、**.env.development**、**.env.production如果被.\*\*\*\*<font style="color:rgb(17, 24, 28);">gitignore忽略没有上传的话，则使用的是</font>****eas.json中的环境变量（需要重新**`eas build ...`**）；**
> ‼️**如果**本地的 **.env**、**.env.development**、**.env.production没有被.\*\*\*\*<font style="color:rgb(17, 24, 28);">gitignore忽略则优先使用对应的环境变量</font>**
