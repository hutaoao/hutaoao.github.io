# let 和 const

### <font style="color:rgb(237, 28, 36);">ES6 声明变量的六种方法</font>

> <code><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">var</font></code><font style="color:rgb(13, 20, 30);">命令和</font><code><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">function</font></code><font style="color:rgb(13, 20, 30);">命令。ES6 除了添加</font><code><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">let</font></code><font style="color:rgb(13, 20, 30);">和</font><code><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">const</font></code><font style="color:rgb(13, 20, 30);">命令，另外两种声明变量的方法：</font><code><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">import</font></code><font style="color:rgb(13, 20, 30);">命令和</font><code><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">class</font></code><font style="color:rgb(13, 20, 30);">命令。</font>

### 变量提升

<code><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">var</font></code><font style="color:rgb(13, 20, 30);">命令会发生“</font>**<font style="color:rgb(13, 20, 30);">变量提升</font>**<font style="color:rgb(13, 20, 30);">”现象，即变量可以在声明之前使用，值为</font><code><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">undefined</font></code><font style="color:rgb(13, 20, 30);">。</font>

<code><font style="color:rgb(13, 20, 30);"></font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">let</font></code><font style="color:rgb(13, 20, 30);">命令改变了语法行为，它所声明的变量一定要在声明后使用，否则报错。</font>

```javascript
// var 的情况
console.log(foo); // 输出undefined
var foo = 2;

// let 的情况
console.log(bar); // 报错ReferenceError
let bar = 2;
```

### 暂时性死区

> <font style="color:rgb(13, 20, 30);">ES6 明确规定，如果区块中存在</font><code><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">let</font></code><font style="color:rgb(13, 20, 30);">和</font><code><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">const</font></code><font style="color:rgb(13, 20, 30);">命令，这个区块对这些命令声明的变量，从一开始就形成了封闭作用域。凡是在声明之前就使用这些变量，就会报错。</font>
> <font style="color:rgb(13, 20, 30);">总之，在代码块内，使用</font><code><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">let</font></code><font style="color:rgb(13, 20, 30);">命令声明变量之前，该变量都是不可用的。这在语法上，称为“</font>**<font style="color:rgb(13, 20, 30);">暂时性死区</font>**<font style="color:rgb(13, 20, 30);">”（temporal dead zone，简称 TDZ）。</font>

<font style="color:rgb(13, 20, 30);"></font>

<font style="color:rgb(13, 20, 30);">只要块级作用域内存在</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">let</font><font style="color:rgb(13, 20, 30);">命令，它所声明的变量就“绑定”（binding）这个区域，不再受外部的影响。</font>

```javascript
var tmp = 123;

if (true) {
  tmp = 'abc'; // ReferenceError
  let tmp;
}
```

<font style="color:rgb(13, 20, 30);">上面代码中，存在全局变量</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">tmp</font><font style="color:rgb(13, 20, 30);">，但是块级作用域内</font><code><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">let</font></code><font style="color:rgb(13, 20, 30);">又声明了一个局部变量</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">tmp</font><font style="color:rgb(13, 20, 30);">，导致后者绑定这个块级作用域，所以在</font><code><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">let</font></code><font style="color:rgb(13, 20, 30);">声明变量前，对</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">tmp</font><font style="color:rgb(13, 20, 30);">赋值会报错。</font>

<font style="color:rgb(13, 20, 30);"></font>

```javascript
if (true) {
  // TDZ开始
  tmp = 'abc'; // ReferenceError
  console.log(tmp); // ReferenceError

  let tmp; // TDZ结束
  console.log(tmp); // undefined

  tmp = 123;
  console.log(tmp); // 123
}
```

<font style="color:rgb(13, 20, 30);">上面代码中，在</font><code><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">let</font></code><font style="color:rgb(13, 20, 30);">命令声明变量</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">tmp</font><font style="color:rgb(13, 20, 30);">之前，都属于变量</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">tmp</font><font style="color:rgb(13, 20, 30);">的“死区”。</font>

<font style="color:rgb(13, 20, 30);"></font>

<font style="color:rgb(13, 20, 30);">“暂时性死区”也意味着</font><code><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">typeof</font></code><font style="color:rgb(13, 20, 30);">不再是一个百分之百安全的操作。</font>

```javascript
typeof x; // ReferenceError
let x;
```

<font style="color:rgb(13, 20, 30);">上面代码中，变量</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">x</font><font style="color:rgb(13, 20, 30);">使用</font><code><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">let</font></code><font style="color:rgb(13, 20, 30);">命令声明，所以在声明之前，都属于</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">x</font><font style="color:rgb(13, 20, 30);">的“死区”，只要用到该变量就会报错。因此，</font><code><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">typeof</font></code><font style="color:rgb(13, 20, 30);">运行时就会抛出一个</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">ReferenceError</font><font style="color:rgb(13, 20, 30);">。</font>

<font style="color:rgb(13, 20, 30);"></font>

<font style="color:rgb(13, 20, 30);">作为比较，如果一个变量根本没有被声明，使用</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">typeof</font><font style="color:rgb(13, 20, 30);">反而不会报错。</font>

```javascript
typeof undeclared_variable // "undefined"
```

<font style="color:rgb(13, 20, 30);">上面代码中，</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">undeclared\_variable</font><font style="color:rgb(13, 20, 30);">是一个不存在的变量名，结果返回“undefined”。所以，在没有</font><code><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">let</font></code><font style="color:rgb(13, 20, 30);">之前，</font><code><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">typeof</font></code><font style="color:rgb(13, 20, 30);">运算符是百分之百安全的，永远不会报错。</font>

<font style="color:rgb(13, 20, 30);"></font>

> **<font style="color:rgb(13, 20, 30);">目的：主要是为了减少运行时错误，防止在变量声明前就使用这个变量，从而导致意料之外的行为</font>**

### const

<code><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">const</font></code><font style="color:rgb(13, 20, 30);">声明一个只读的常量。一旦声明，常量的值就不能改变。这意味着，</font><code><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">const</font></code><font style="color:rgb(13, 20, 30);">一旦声明变量，就必须</font>**<font style="color:rgb(13, 20, 30);">立即初始化</font>**<font style="color:rgb(13, 20, 30);">，不能留到以后赋值。</font>

> <font style="color:rgb(13, 20, 30);">是变量指向的那个</font>**<font style="color:rgb(13, 20, 30);">内存地址</font>**<font style="color:rgb(13, 20, 30);">所保存的数据不得改动。对于简单类型的数据（数值、字符串、布尔值），值就保存在变量指向的那个内存地址，因此等同于常量。但对于复合类型的数据（主要是对象和数组），变量指向的内存地址，保存的只是一个指向实际数据的指针，</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">const</font><font style="color:rgb(13, 20, 30);">只能保证这个指针是固定的（即总是指向另一个固定的地址），至于它指向的数据结构是不是可变的，就完全不能控制了。</font>

<font style="color:rgb(13, 20, 30);"></font>

<font style="color:rgb(13, 20, 30);">如果真的想将对象冻结，应该使用</font><code><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">Object.freeze</font></code><font style="color:rgb(13, 20, 30);">方法。</font>

```javascript
const foo = Object.freeze({});

// 常规模式时，下面一行不起作用；
// 严格模式时，该行会报错
foo.prop = 123;
```

### <font style="color:rgb(237, 28, 36);">顶层对象</font>

<font style="color:rgb(13, 20, 30);">顶层对象，</font>**<font style="color:rgb(13, 20, 30);">在浏览器环境指的是</font>**<code>**<font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">window</font>**</code>**<font style="color:rgb(13, 20, 30);">对象</font>**<font style="color:rgb(13, 20, 30);">，在 </font>**<font style="color:rgb(13, 20, 30);">Node 指的是</font>**<code>**<font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">global</font>**</code>**<font style="color:rgb(13, 20, 30);">对象</font>**<font style="color:rgb(13, 20, 30);">。ES5 之中，顶层对象的属性与全局变量是等价的。</font>

```javascript
window.a = 1;
a // 1

a = 2;
window.a // 2
```

```javascript
var a = 1;
// 如果在 Node 的 REPL 环境，可以写成 global.a
// 或者采用通用方法，写成 this.a
window.a // 1

let b = 1;
window.b // undefined
```

<font style="color:rgb(13, 20, 30);">上面代码中，全局变量</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">a</font><font style="color:rgb(13, 20, 30);">由</font><code><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">var</font></code><font style="color:rgb(13, 20, 30);">命令声明，所以它是顶层对象的属性；全局变量</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">b</font><font style="color:rgb(13, 20, 30);">由</font><code><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">let</font></code><font style="color:rgb(13, 20, 30);">命令声明，所以它不是顶层对象的属性，返回</font><code><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">undefined</font></code><font style="color:rgb(13, 20, 30);">。</font>

<font style="color:rgb(13, 20, 30);"></font>

> <font style="color:rgb(13, 20, 30);">ES6规定</font><code><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">var</font></code><font style="color:rgb(13, 20, 30);">命令和</font><code><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">function</font></code><font style="color:rgb(13, 20, 30);">命令声明的全局变量，依旧是顶层对象的属性；</font>
> <font style="color:rgb(13, 20, 30);">另一方面规定，</font><code><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">let</font></code><font style="color:rgb(13, 20, 30);">命令、</font><code><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">const</font></code><font style="color:rgb(13, 20, 30);">命令、</font><code><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">class</font></code><font style="color:rgb(13, 20, 30);">命令声明的全局变量，不属于顶层对象的属性。</font>

### <font style="color:rgb(237, 28, 36);">globalThis 对象</font>

<font style="color:rgb(13, 20, 30);">JavaScript 语言存在一个顶层对象，它提供全局环境（即</font>**<font style="color:rgb(13, 20, 30);">全局作用域</font>**<font style="color:rgb(13, 20, 30);">），所有代码都是在这个环境中运行。但是，顶层对象在各种实现里面是不统一的。（</font><code><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">this</font></code><font style="color:rgb(13, 20, 30);">关键字，但是有局限性。）</font>

* <font style="color:rgb(13, 20, 30);">浏览器里面，顶层对象是</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">window</font><font style="color:rgb(13, 20, 30);">，但 Node 和 Web Worker 没有</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">window</font><font style="color:rgb(13, 20, 30);">。</font>
* <font style="color:rgb(13, 20, 30);">浏览器和 Web Worker 里面，</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">self</font><font style="color:rgb(13, 20, 30);">也指向顶层对象，但是 Node 没有</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">self</font><font style="color:rgb(13, 20, 30);">。</font>
* <font style="color:rgb(13, 20, 30);">Node 里面，顶层对象是</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">global</font><font style="color:rgb(13, 20, 30);">，但其他环境都不支持。</font>

[ES2020](https://github.com/tc39/proposal-global)<font style="color:rgb(13, 20, 30);"> 在语言标准的层面，引入</font><code><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">globalThis</font></code><font style="color:rgb(13, 20, 30);">作为顶层对象。也就是说，任何环境下，</font><code><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">globalThis</font></code><font style="color:rgb(13, 20, 30);">都是存在的，都可以从它拿到顶层对象，指向全局环境下的</font><code><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">this</font></code><font style="color:rgb(13, 20, 30);">。</font>

<font style="color:rgb(13, 20, 30);">垫片库</font>[<font style="color:rgb(13, 20, 30);">global-this</font>](https://github.com/ungap/global-this)<font style="color:rgb(13, 20, 30);">模拟了这个提案，可以在所有环境拿到</font><code><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">globalThis</font></code><font style="color:rgb(13, 20, 30);">。</font>
