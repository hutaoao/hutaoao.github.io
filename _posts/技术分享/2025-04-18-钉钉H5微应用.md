* <font style="color:rgb(51, 51, 51);">账号授权（免登）</font>

<font style="color:rgb(51, 51, 51);">支持扫码登录和使用钉钉账号登录。</font>

<font style="color:rgb(51, 51, 51);"></font>

<font style="color:rgb(51, 51, 51);">开发者可以使用钉钉的分享SDK接入钉钉分享功能。用户在分享应用时可以选择分享到钉钉。</font>

<font style="color:rgb(51, 51, 51);"></font>

* <font style="color:rgb(51, 51, 51);">机器人</font>

<font style="color:rgb(51, 51, 51);">群机器人是钉钉群的高级扩展功能。群机器人可以将第三方服务的信息聚合到群聊中，实现自动化的信息同步。</font>

<font style="color:rgb(51, 51, 51);"></font>

* <font style="color:rgb(51, 51, 51);">H5微应用</font>

<font style="color:rgb(51, 51, 51);">微应用是指用H5方式开发的应用。</font>

## 基础概念

### CorpId

<code><font style="color:rgb(51, 51, 51);background-color:rgb(246, 246, 246);">CorpId</font></code> <font style="color:rgb(51, 51, 51);">是企业在钉钉中的标识，每个企业拥有唯一的 </font><code><font style="color:rgb(51, 51, 51);background-color:rgb(246, 246, 246);">CorpId</font></code><font style="color:rgb(51, 51, 51);">。</font>

<font style="color:rgb(51, 51, 51);"></font>

### UserId

<font style="color:rgb(51, 51, 51);">企业内每个员工都有唯一的 </font><code><font style="color:rgb(51, 51, 51);background-color:rgb(246, 246, 246);">UserId</font></code><font style="color:rgb(51, 51, 51);">，创建后不可修改。</font>

<font style="color:rgb(51, 51, 51);"></font>

### AppKey/AppSecret

<code><font style="color:rgb(51, 51, 51);">AppKey</font></code><font style="color:rgb(51, 51, 51);"> 是</font>**<font style="color:rgb(51, 51, 51);">企业内部应用</font>**<font style="color:rgb(51, 51, 51);">的唯一身份标识，</font><code><font style="color:rgb(51, 51, 51);">AppSecret</font></code><font style="color:rgb(51, 51, 51);"> 是对应的调用密钥。</font>

**<font style="color:rgb(51, 51, 51);">在</font>**钉钉开发者后台**<font style="color:rgb(51, 51, 51);">创建企业内部应用后，系统会自动生成一对 </font>**<code>**<font style="color:rgb(51, 51, 51);">AppKey</font>**</code>**<font style="color:rgb(51, 51, 51);"> 和 </font>**<code>**<font style="color:rgb(51, 51, 51);">AppSecret</font>**</code>**<font style="color:rgb(51, 51, 51);">。</font>**

**<font style="color:rgb(51, 51, 51);"></font>**

### access\_token

<code><font style="color:rgb(51, 51, 51);">access_token</font></code><font style="color:rgb(51, 51, 51);"> 是企业后台通过钉钉接口获取信息的重要凭据。在调用钉钉接口时必须携带 </font><code><font style="color:rgb(51, 51, 51);">access_token</font></code><font style="color:rgb(51, 51, 51);"> 用于验证接口的访问权限。</font>

> <font style="color:rgb(115, 119, 122);">默认access\_token的有效期为7200秒（两小时），有效期内重复获取，会返回相同的access\_token，并自动续期。</font>

## H5微应用开发

### 概述

<font style="color:rgb(51, 51, 51);">H5微应用JSAPI为应用提供了调用原生控件的能力，帮助开发者高效使用拍照、定位等手机系统的能力，同时可以直接使用扫一扫、分享、钉盘等钉钉特有的能力，带给微应用接近原生代码的体验。</font>

<font style="color:rgb(51, 51, 51);"></font>

### 准备工作

1. <font style="color:rgb(51, 51, 51);">使用npm安装。</font>

```javascript
npm install dingtalk-jsapi --save
import * as dd from 'dingtalk-jsapi'; 
```

2. <font style="color:rgb(51, 51, 51);">浏览器引入。在浏览器中使用 script 和 link 标签直接引入文件，并使用全局变量 dd。</font>**<font style="color:rgb(115, 119, 122);"></font>**

```javascript
<script src="https://g.alicdn.com/dingding/dingtalk-jsapi/2.10.3/dingtalk.open.js"></script>
```

### [JSAPI鉴权](https://developers.dingtalk.com/document/app/jsapi-overview)

<font style="color:rgb(51, 51, 51);">钉钉提供的JSAPI有很多是手机的基础能力，对这些JSAPI的调用不需要进行鉴权（</font>**<font style="color:rgb(51, 51, 51);">即不需要进行dd.config</font>**<font style="color:rgb(51, 51, 51);">），只需要保证在dd.ready里面调用即可。对于一些钉钉业务相关、安全相关的JSAPI的调用，需要开发者先进行鉴权然后再调用。\ </font>

### 调用JSAPI组件

> <font style="color:rgb(115, 119, 122);">所有JSAPI组件的调用，必须在 </font><code>**<font style="color:#E8323C;">dd.ready</font>**</code>**<font style="color:#E8323C;"> </font>**<font style="color:rgb(115, 119, 122);">里面执行。</font>

#### 免鉴权

<font style="color:rgb(51, 51, 51);">手机的基础能力（弹窗提示、设置导航栏等），对这些JSAPI的调用不需要进行鉴权，直接在</font><code><font style="color:rgb(51, 51, 51);">dd.ready</font></code><font style="color:rgb(51, 51, 51);"> 里调用</font>

```javascript
dd.ready(function() {
    // dd.ready参数为回调函数，在环境准备就绪时触发，jsapi的调用需要保证在该回调函数触发后调用，否则无效。
    dd.runtime.permission.requestAuthCode({
        corpId: "corpid",
        onSuccess: function(result) {
        /*{
            code: 'hYLK98jkf0m' //string authCode
        }*/
        },
        onFail : function(err) {}
  
    });
});
```

#### 需鉴权

<font style="color:rgb(51, 51, 51);">对于一些钉钉</font><font style="color:#E8323C;">业务</font><font style="color:rgb(51, 51, 51);">、</font><font style="color:#E8323C;">安全相关</font>（如：定位、支付）<font style="color:rgb(51, 51, 51);">的JSAPI的调用，需要先鉴权，然后再调用。</font>

> **<font style="color:#E8323C;">dd.config </font>**<font style="color:rgb(115, 119, 122);">中所有的参数必须直接来自服务端，不能直接在前端定义。</font>

```javascript
dd.config({
    agentId: '', // 必填，微应用ID
    corpId: '',//必填，企业ID
    timeStamp: '', // 必填，生成签名的时间戳
    nonceStr: '', // 必填，自定义固定字符串。
    signature: '', // 必填，签名
    type:0/1,   //选填。0表示微应用的jsapi,1表示服务窗的jsapi；不填默认为0。该参数从dingtalk.js的0.8.3版本开始支持
    jsApiList : [
        'runtime.info',
        'biz.contact.choose',
        'device.notification.confirm',
        'device.notification.alert',
        'device.notification.prompt',
        'biz.ding.post',
        'biz.util.openLink',
    ] // 必填，需要使用的jsapi列表，注意：不要带dd。
});

dd.error(function (err) {
    alert('dd error: ' + JSON.stringify(err));
})//该方法必须带上，用来捕获鉴权出现的异常信息，否则不方便排查出现的问题
```

鉴权后就<font style="color:rgb(51, 51, 51);">可以在dd.ready里调用JSAPI了。</font>

<font style="color:rgb(51, 51, 51);"></font>

## 接入免登获取用户信息（企业内部应用）

1. <font style="color:rgb(51, 51, 51);">获取免登授权码 - </font>[H5微应用免登](https://developers.dingtalk.com/document/app/logon-free-process#topic-2025319)。

```javascript
dd.ready(function() {
  dd.runtime.permission.requestAuthCode({
    corpId: "ding12345xxx", // 企业id
    onSuccess: function (info) {
      code = info.code // 通过该免登授权码可以获取用户身份
    }});
});
```

> <font style="color:rgb(51, 51, 51);">获取的免登授权码有效期5分钟，且只能使用一次。</font>

2. <font style="color:rgb(51, 51, 51);">获取access\_token。</font><font style="color:rgb(51, 51, 51);">调用接口获取access\_token，详情请参考</font>[获取企业内部应用的access\_token](https://developers.dingtalk.com/document/app/obtain-orgapp-token#topic-1936350)<font style="color:rgb(51, 51, 51);">。</font>
3. <font style="color:rgb(51, 51, 51);">获取用户userid。</font><font style="color:rgb(51, 51, 51);">调用接口获取用户的userid，详情请参考</font>[通过免登码获取用户信息](https://developers.dingtalk.com/document/app/get-user-userid-through-login-free-code#topic-1936806)<font style="color:rgb(51, 51, 51);">。</font>
4. <font style="color:rgb(51, 51, 51);">获取用户详情。调用接口获取用户详情信息，详情请参考</font>[获取用户详情](https://developers.dingtalk.com/document/app/query-user-details#topic-1960047)<font style="color:rgb(51, 51, 51);">。</font>

> <font style="color:rgb(51, 51, 51);">注意：</font><font style="color:rgb(51, 51, 51);">服务端API</font><font style="color:rgb(51, 51, 51);">在调用前，确保已经为应用添加了接口权限！</font>

## TIPS

* **rem 布局**

***

设计稿宽 750px，即 1rem=100px。

```javascript
html {
  font-size: calc(100vw / 750 * 100);
}
```

* **钉钉 JSAPI支持自定义头部导航栏（颜色、自定义按钮等）**

* **<font style="color:rgb(51, 51, 51);">iPhoneX等机型底部留白</font>**

**<font style="color:rgb(51, 51, 51);"></font>**

<font style="color:rgb(51, 51, 51);">H5页面高度设置 height:100vh，页面会产生滚动条。设置 height:100%，iPhoneX等机型下面会留白。</font>

meta标签新增扩展：<code>**<font style="color:#E8323C;">viewport-fit=cover</font>**</code>

```javascript
<meta name="viewport" content="width=device-width,initial-scale=1.0, minimum-scale=1.0, maximum-scale=1.0, user-scalable=no, viewport-fit=cover">
```

`viewport-fit` 设置可视视窗的大小，它有三个属性值：

* Auto：默认值，与Contain表现一致。
* Contain：可视窗口完全包含网页内容（左图）。
* Cover：网页内容完全覆盖可视窗口（右图）。


* **原生 fetch 图片上传（当时所有安卓无法上传是代码错误造成）**

***

<font style="color:#E8323C;">不要手动设置 </font><code><font style="color:#E8323C;">headers</font></code><font style="color:#E8323C;"> 的 </font><code><font style="color:#E8323C;">Content-Type</font></code>，<font style="color:rgb(0, 0, 0);">手动设置 </font><code><font style="color:rgb(18, 18, 18);">multipart/form-data</font></code><font style="color:rgb(0, 0, 0);"> , 但是少了</font><font style="color:#E8323C;">浏览器自动生成</font><font style="color:rgb(0, 0, 0);">的</font><code>**<font style="color:rgb(0, 0, 0);">boundary</font>**</code>**<font style="color:rgb(0, 0, 0);"> </font>**<font style="color:rgb(0, 0, 0);">参数，缺失也无法解析。</font>

<font style="color:rgb(0, 0, 0);"></font>

<code>**boundary**</code>\*\* 的重要性：\*\*

在rfc1867协议中规定，我们在指定content-type为multipart/form-data时，表明我们需要传递的是多媒体内容，我们需要指定boundary【分隔符】来分割当上传多个文件/图片。boundary是随机生成的数字和字母的组合。

按照上面的写法可以看出，Request Header里面的content-type中只是设置了multipart/form-data，而没有随机生成的boundary值。
