# memo

`memo` 是个高阶组件， 结合了 <code><font style="color:#DF2A3F;">PurComponent</font></code> 和 <code><font style="color:#DF2A3F;">shouldComponentUpdate</font></code> 功能，会对传入的 props 进行\*\*<font style="color:#DF2A3F;">浅比较</font>\*\*，来决定是否更新被包裹的组件

memo 接受两个参数：

* **WrapComponent**：你要优化的组件
* **(prev, next) => boolean**：通过对比 prev（旧 props），next（新 props）是否一致，返回 true（不更新）、false（更新）

> 注意：`memo` 只针对 **<font style="color:#DF2A3F;">props</font>** 来决定是否渲染，且是\*\*<font style="color:#DF2A3F;">浅比较</font>\*\*

**<font style="color:rgb(3, 106, 202);"></font>**

**<font style="color:rgb(3, 106, 202);">现在我们来看一个的例子：</font>**

```tsx
import { Button } from "antd";
import React, { useState } from "react";

const Child = () => <div>{console.log("子组件又渲染")}</div>;

const Parent = () => {
  const [number, setNumber] = useState(0);
  const [flag, setFlag] = useState(false);

  return (
    <>
      <Child />
      <Button type='primary' onClick={() => setFlag(!flag)}>
        {flag ? "显示" : "隐藏"}
      </Button>
    </>
  );
};

export default Parent;
```

**<font style="color:rgb(3, 106, 202);">运行结果如下：</font>**

![1697080059363-76bd4042-760e-499f-886e-97f5bf8133e0.png](/assets/img/posts/React/1697080059363-76bd4042-760e-499f-886e-97f5bf8133e0-193775.png)

在上面的例子中，父组件中的两个状态 **<font style="color:#DF2A3F;">number</font>**、**<font style="color:#DF2A3F;">flag</font>** 都和 Child 组件没有关系，当我点击按钮时，**<font style="color:#DF2A3F;">flag</font>**<font style="color:#DF2A3F;"> 发生改变，此时父组件重新渲染，按钮文案变为显示，控制台却打印出 "子组件又渲染" 的信息</font>，说明子组件也跟着重新渲染了。而这肯定是不合理的，我们不希望子组件做无关的刷新，此时我们可以给子组件加上 `React.memo`

```tsx
import { Button } from "antd";
import React, { useState } from "react";

const Child = React.memo(() => <div>{console.log("子组件又渲染")}</div>);

const Parent = () => {
  const [number, setNumber] = useState(0);
  const [flag, setFlag] = useState(false);

  return (
    <>
      <Child />
      <Button type='primary' onClick={() => setFlag(!flag)}>
        {flag ? "显示" : "隐藏"}
      </Button>
    </>
  );
};

export default Parent;
```

**<font style="color:rgb(3, 106, 202);">运行结果如下：</font>**

![1697080234521-4e9692d4-459c-4d15-aada-8ed9e813fc42.png](/assets/img/posts/React/1697080234521-4e9692d4-459c-4d15-aada-8ed9e813fc42-988981.png)

在给子组件加上 `memo` 包裹后，再次点击按钮，<font style="color:#DF2A3F;">flag 发生改变，此时控制台并没有打印 "子组件又渲染" 的信息，说明此时子组件不会做无关的刷新</font>，从而达到了性能优化的目的。

**<font style="color:#117CEE;">总而言之，如果组件被 </font>**<code>**<font style="color:#117CEE;">memo</font>**</code>**<font style="color:#117CEE;"> 包裹，那么组件的 </font>**<code>**<font style="color:#117CEE;">props</font>**</code>**<font style="color:#117CEE;"> 不发生改变时，组件不会重新渲染。这样，我们合理的使用 </font>**<code>**<font style="color:#117CEE;">memo</font>**</code>**<font style="color:#117CEE;"> 就可以为我们的项目带来很大的性能优化。</font>**

### memo 的注意事项

<font style="color:rgb(89, 89, 89);">虽然，memo 可以帮助我们避免组件无意义的重新渲染，达到性能优化的目的，但是你还是得注意一下 memo 的一些注意事项</font>

####

<font style="color:rgb(19, 92, 224);">memo 对 props 是浅比较</font> <font style="color:rgb(89, 89, 89);">上文我们也说了，</font><code><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">memo</font></code><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);"> 对于新旧 props 的比较是浅比较</font><font style="color:rgb(89, 89, 89);">，当一个引用类型的 props 改变时，只要它的</font><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">地址没有发生改变</font><font style="color:rgb(89, 89, 89);">，那么就算 props 中某一项数据发生了改变，那么被 </font><code><font style="color:rgb(89, 89, 89);">memo</font></code><font style="color:rgb(89, 89, 89);"> 包裹的组件是不会重新渲染的。</font>

<font style="color:rgb(89, 89, 89);"></font>

<font style="color:rgb(89, 89, 89);">比如下面的例子：</font>

```tsx
import { Button, Divider } from "antd";
import React, { useState } from "react";

const Child = React.memo((props) => (
  <div>
    {props.list.map((item) => (
{% raw %}
      <div style={{marginLeft: '20px'}}> {item} </div>
{% endraw %}
    ))}
  </div>
));

const Parent = () => {
  const [list, setList] = useState([1, 2, 3]);

  return (
    <>
      <Child list={list} />
      <Divider />
      <Button
        type='primary'
        onClick={() => {
          list.push(4);
          console.log('list', list)
        }}
      >
        点击改变 list
      </Button>
    </>
  );
};

export default Parent;
```

<font style="color:rgb(89, 89, 89);">上面的代码，当我们点击按钮后，</font>**<font style="color:rgb(3, 106, 202);">运行结果如下</font>**<font style="color:rgb(89, 89, 89);">：</font>

![1697080783307-8102cd0a-0e33-413f-b42f-eb4380466487.png](/assets/img/posts/React/1697080783307-8102cd0a-0e33-413f-b42f-eb4380466487-334074.png)

<font style="color:rgb(89, 89, 89);">我们可以看见，虽然点击了按钮改变了 list，但此时子组件渲染的只有初始的 </font><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">1 2 3</font><font style="color:rgb(89, 89, 89);">，并没有重新渲染，这是因为虽然 list 内容改变了，但是 list 是引用类型的数据，memo 对新旧 list 进行浅比较，</font><font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">发现地址没变</font><font style="color:rgb(89, 89, 89);">，就不重新渲染。</font>

<font style="color:rgb(89, 89, 89);"></font>

<font style="color:rgb(89, 89, 89);">那这种情况怎么办呢？很简单，</font>**<font style="color:#DF2A3F;">返回一个新的数组即可</font>**

```tsx
onClick={() => {
 setList([...list, 4])
}}
```

![1697080863445-3081fab7-0d6b-422d-9bad-37429b6e264e.png](/assets/img/posts/React/1697080863445-3081fab7-0d6b-422d-9bad-37429b6e264e-759155.png)

<font style="color:rgb(89, 89, 89);">setList，子组件重新渲染</font>

#### <font style="color:rgb(19, 92, 224);">memo 是否用的越多越好</font>

<font style="color:rgb(89, 89, 89);">既然 </font><code><font style="color:rgb(89, 89, 89);">memo</font></code><font style="color:rgb(89, 89, 89);"> 可以对组件进行性能优化，那能不能所有组件都用 </font><code><font style="color:rgb(89, 89, 89);">memo</font></code><font style="color:rgb(89, 89, 89);"> 包裹呢？</font>

<font style="color:rgb(89, 89, 89);">答案肯定是否定的。</font>

<font style="color:rgb(255, 80, 44);background-color:rgb(255, 245, 245);">因为缓存本身也是需要开销的</font><font style="color:rgb(89, 89, 89);">。如果每一个组件都用 </font><code><font style="color:rgb(89, 89, 89);">memo</font></code><font style="color:rgb(89, 89, 89);"> 去包裹一下，那么对浏览器的开销就会很大，本末倒置了。</font>

<font style="color:rgb(89, 89, 89);">所以我们应该选择性的用 </font><code><font style="color:rgb(89, 89, 89);">memo</font></code><font style="color:rgb(89, 89, 89);"> 包裹组件，而不是滥用。</font>

**<font style="color:rgb(3, 106, 202);">在项目中，一般如果一个子组件经常被重新渲染，那需要根据具体情况具体分析，有目的性的去缓存它。</font>**
