# @Builder装饰器

<font style="color:rgba(0, 0, 0, 0.9);">ArkUI提供了一种轻量的UI元素复用机制@Builder，该自定义组件内部UI结构固定，仅与使用方进行数据传递，开发者可以将重复使用的UI元素抽象成一个方法，在build方法里调用。</font>

<font style="color:rgba(0, 0, 0, 0.9);">为了简化语言，我们将@Builder装饰的函数也称为“自定义构建函数”。</font>

<font style="color:rgba(0, 0, 0, 0.9);"></font>

## <font style="color:#DF2A3F;">限制条件</font>
+ <font style="color:rgb(36, 39, 40);">@Builder通过</font>**<font style="color:#DF2A3F;">按引用传递的方式传入参数，才会触发动态渲染UI，并且参数只能是一个</font>**<font style="color:rgb(36, 39, 40);">。</font>
+ <font style="color:rgb(36, 39, 40);">@Builder如果传入的参数是两个或两个以上，不会触发动态渲染UI。</font>
+ <font style="color:rgb(36, 39, 40);">@Builder传入的参数中同时包含按值传递和按引用传递两种方式，不会触发动态渲染UI。</font>
+ <font style="color:rgb(36, 39, 40);">@Builder的参数必须按照对象字面量的形式，把所需要的属性一一传入，才会触发动态渲染UI。</font>
