# 变量的结构赋值

### <font style="color:rgb(237, 28, 36);">数组的解构赋值</font>

#### 最简单的：（<font style="color:rgb(13, 20, 30);">模式匹配</font>）

```javascript
let [a, b, c] = [1, 2, 3];
a // 1
b // 2
c // 3
```

```javascript
let [foo, [[bar], baz]] = [1, [[2], 3]];
foo // 1
bar // 2
baz // 3

let [head, ...tail] = [1, 2, 3, 4];
head // 1
tail // [2, 3, 4]

let [x, y, ...z] = ['a'];
x // "a"
y // undefined
z // []
```

结构不成功，变量的值就等于 <code><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">undefined</font></code>

```javascript
let [foo] = [];
let [bar, foo] = [1];
foo // undefined
```

#### <font style="color:rgb(13, 20, 30);">不完全解构</font>

```javascript
let [x, y] = [1, 2, 3];
x // 1
y // 2

let [a, [b], d] = [1, [2, 3], 4];
a // 1
b // 2
d // 4
```

<font style="color:rgb(13, 20, 30);">如果等号的右边不是数组，那么将会报错。</font>

```javascript
// 报错
let [foo] = 1;
let [foo] = false;
let [foo] = NaN;
let [foo] = undefined;
let [foo] = null;
let [foo] = {};
```

#### 默认值

```javascript
let [foo = true] = [];
foo // true

let [x, y = 'b'] = ['a']; // x='a', y='b'
let [x, y = 'b'] = ['a', undefined]; // x='a', y='b'
```

> <font style="color:rgb(13, 20, 30);">注意，ES6 内部使用严格相等运算符（</font><code><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">===</font></code><font style="color:rgb(13, 20, 30);">），判断一个位置是否有值。所以，只有当一个数组成员严格等于</font><code><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">undefined</font></code><font style="color:rgb(13, 20, 30);">，默认值才会生效。</font>

```javascript
let [x = 1] = [undefined];
x // 1

let [x = 1] = [null];
x // null
```

<font style="color:rgb(13, 20, 30);">上面代码中，如果一个数组成员是</font><code><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">null</font></code><font style="color:rgb(13, 20, 30);">，默认值就不会生效，因为</font><code><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">null</font></code><font style="color:rgb(13, 20, 30);">不严格等于</font><code><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">undefined</font></code><font style="color:rgb(13, 20, 30);">。</font>

<font style="color:rgb(13, 20, 30);"></font>

<font style="color:rgb(13, 20, 30);">如果默认值是一个表达式，那么这个表达式是</font>**<font style="color:#DF2A3F;">惰性求值</font>**<font style="color:rgb(13, 20, 30);">的，</font>**<font style="color:#DF2A3F;">即只有在用到的时候，才会求值</font>**<font style="color:rgb(13, 20, 30);">。</font>

```javascript
function f() {
  console.log('aaa');
}

let [x = f()] = [1];
```

<font style="color:rgb(13, 20, 30);">上面代码中，因为</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">x</font><font style="color:rgb(13, 20, 30);">能取到值，所以函数</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">f</font><font style="color:rgb(13, 20, 30);">根本不会执行。</font>

<font style="color:rgb(13, 20, 30);"></font>

<font style="color:rgb(13, 20, 30);">默认值可以引用解构赋值的其他变量，但该变量必须</font>**<font style="color:rgb(13, 20, 30);">已经声明</font>**<font style="color:rgb(13, 20, 30);">。</font>

```javascript
let [x = 1, y = x] = [];     // x=1; y=1
let [x = 1, y = x] = [2];    // x=2; y=2
let [x = 1, y = x] = [1, 2]; // x=1; y=2
let [x = y, y = 1] = [];     // ReferenceError: y is not defined
```

<font style="color:rgb(13, 20, 30);">上面最后一个表达式之所以会报错，是因为</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">x</font><font style="color:rgb(13, 20, 30);">用</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">y</font><font style="color:rgb(13, 20, 30);">做默认值时，</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">y</font><font style="color:rgb(13, 20, 30);">还没有声明。</font>

### <font style="color:rgb(237, 28, 36);">对象的解构赋值</font>

开胃菜

```javascript
let { foo, bar } = { foo: 'aaa', bar: 'bbb' };
foo // "aaa"
bar // "bbb"
```

<font style="color:rgb(13, 20, 30);">对象的解构与数组有一个重要的不同。</font>**<font style="color:rgb(13, 20, 30);">数组的元素是按次序排列的</font>**<font style="color:rgb(13, 20, 30);">，变量的取值由它的位置决定；而</font>**<font style="color:rgb(13, 20, 30);">对象的属性没有次序，变量必须与属性同名</font>**<font style="color:rgb(13, 20, 30);">，才能取到正确的值。</font>

```javascript
let { bar, foo } = { foo: 'aaa', bar: 'bbb' };
foo // "aaa"
bar // "bbb"

let { baz } = { foo: 'aaa', bar: 'bbb' };
baz // undefined
```

<font style="color:rgb(13, 20, 30);">如果解构失败，变量的值等于</font><code><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">undefined</font></code><font style="color:rgb(13, 20, 30);">。</font>

```javascript
let {foo} = {bar: 'baz'};
foo // undefined
```

<font style="color:rgb(13, 20, 30);">对象的解构赋值，可以很方便地将现有对象的方法，赋值到某个变量。</font>

```javascript
// 例一
let { log, sin, cos } = Math;

// 例二
const { log } = console;
log('hello') // hello
```

<font style="color:rgb(13, 20, 30);">如果变量名与属性名不一致，必须写成下面这样。</font>

```javascript
let { foo: baz } = { foo: 'aaa', bar: 'bbb' };
baz // "aaa"

let obj = { first: 'hello', last: 'world' };
let { first: f, last: l } = obj;
f // 'hello'
l // 'world'
```

<font style="color:rgb(13, 20, 30);">也就是说，对象的解构赋值的内部机制，是先找到同名属性，然后再赋给对应的变量。</font><font style="color:#DF2A3F;">真正被赋值的是后者，而不是前者。</font>

```javascript
let { foo: baz } = { foo: 'aaa', bar: 'bbb' };
baz // "aaa"
foo // error: foo is not defined
```

<font style="color:rgb(13, 20, 30);">上面代码中，</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">foo</font><font style="color:rgb(13, 20, 30);">是匹配的模式，</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">baz</font><font style="color:rgb(13, 20, 30);">才是变量。真正被赋值的是变量</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">baz</font><font style="color:rgb(13, 20, 30);">，而不是模式</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">foo</font><font style="color:rgb(13, 20, 30);">。</font>

默认值

```javascript
var {x = 3} = {};
x // 3

var {x, y = 5} = {x: 1};
x // 1
y // 5

var {x: y = 3} = {};
y // 3

var {x: y = 3} = {x: 5};
y // 5

var { message: msg = 'Something went wrong' } = {};
msg // "Something went wrong"
```

<font style="color:rgb(13, 20, 30);">默认值生效的条件是，对象的属性值严格等于</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">undefined</font><font style="color:rgb(13, 20, 30);">。</font>

```javascript
var {x = 3} = {x: undefined};
x // 3

var {x = 3} = {x: null};
x // null
```

<font style="color:rgb(13, 20, 30);">上面代码中，属性</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">x</font><font style="color:rgb(13, 20, 30);">等于</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">null</font><font style="color:rgb(13, 20, 30);">，因为</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">null</font><font style="color:rgb(13, 20, 30);">与</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">undefined</font><font style="color:rgb(13, 20, 30);">不严格相等，所以是个有效的赋值，导致默认值</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">3</font><font style="color:rgb(13, 20, 30);">不会生效。</font>

### <font style="color:rgb(237, 28, 36);">字符串的解构赋值</font>

<font style="color:rgb(13, 20, 30);">字符串也可以解构赋值。这是因为此时，字符串被转换成了一个类似数组的对象。</font>

```javascript
const [a, b, c, d, e] = 'hello';
a // "h"
b // "e"
c // "l"
d // "l"
e // "o"
```

<font style="color:rgb(13, 20, 30);">类似数组的对象都有一个</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">length</font><font style="color:rgb(13, 20, 30);">属性，因此还可以对这个属性解构赋值。</font>

```javascript
let {length : len} = 'hello';
len // 5
```

### 用途

#### <font style="color:rgb(13, 20, 30);">（1）交换变量的值</font>

```javascript
let x = 1;
let y = 2;

[x, y] = [y, x];
```

<font style="color:rgb(13, 20, 30);">上面代码交换变量</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">x</font><font style="color:rgb(13, 20, 30);">和</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">y</font><font style="color:rgb(13, 20, 30);">的值，这样的写法不仅简洁，而且易读，语义非常清晰。</font>

####

#### <font style="color:rgb(13, 20, 30);">（2）从函数返回多个值</font>

<font style="color:rgb(13, 20, 30);">函数只能返回一个值，如果要返回多个值，只能将它们放在数组或对象里返回。有了解构赋值，取出这些值就非常方便。</font>

```javascript
// 返回一个数组

function example() {
  return [1, 2, 3];
}
let [a, b, c] = example();

// 返回一个对象

function example() {
  return {
    foo: 1,
    bar: 2
  };
}
let { foo, bar } = example();
```

**<font style="color:rgb(13, 20, 30);"></font>**

#### <font style="color:rgb(13, 20, 30);">（3）函数参数的定义</font>

<font style="color:rgb(13, 20, 30);">解构赋值可以方便地将一组参数与变量名对应起来。</font>

```javascript
// 参数是一组有次序的值
function f([x, y, z]) { ... }
f([1, 2, 3]);

// 参数是一组无次序的值
function f({x, y, z}) { ... }
f({z: 3, y: 2, x: 1});
```

#### <font style="color:rgb(13, 20, 30);">（4）提取 JSON 数据</font>

<font style="color:rgb(13, 20, 30);">解构赋值对提取 JSON 对象中的数据，尤其有用。</font>

```javascript
let jsonData = {
  id: 42,
  status: "OK",
  data: [867, 5309]
};

let { id, status, data: number } = jsonData;

console.log(id, status, number);
// 42, "OK", [867, 5309]
```

#### <font style="color:rgb(13, 20, 30);">（5）函数参数的默认值</font>

```javascript
jQuery.ajax = function (url, {
  async = true,
  beforeSend = function () {},
  cache = true,
  complete = function () {},
  crossDomain = false,
  global = true,
  // ... more config
} = {}) {
  // ... do stuff
};
```

<font style="color:rgb(13, 20, 30);">指定参数的默认值，就避免了在函数体内部再写</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">var foo = config.foo || 'default foo';</font><font style="color:rgb(13, 20, 30);">这样的语句。</font>

<font style="color:rgb(13, 20, 30);"></font>

#### <font style="color:rgb(13, 20, 30);">（6）遍历 Map 结构</font>

<font style="color:rgb(13, 20, 30);">任何部署了 Iterator 接口的对象，都可以用</font><font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">for...of</font><font style="color:rgb(13, 20, 30);">循环遍历。Map 结构原生支持 Iterator 接口，配合变量的解构赋值，获取键名和键值就非常方便。</font>

```javascript
const map = new Map();
map.set('first', 'hello');
map.set('second', 'world');

for (let [key, value] of map) {
  console.log(key + " is " + value);
}
// first is hello
// second is world
```

<font style="color:rgb(13, 20, 30);">如果只想获取键名，或者只想获取键值，可以写成下面这样。</font>

```javascript
// 获取键名
for (let [key] of map) {
  // ...
}

// 获取键值
for (let [,value] of map) {
  // ...
}
```

#### <font style="color:rgb(13, 20, 30);">（7）输入模块的指定方法</font>

<font style="color:rgb(13, 20, 30);">加载模块时，往往需要指定输入哪些方法。解构赋值使得输入语句非常清晰。</font>

```javascript
const { SourceMapConsumer, SourceNode } = require("source-map");
```
