# Flutter生命周期详解及实践建议

好的，Flutter 的生命周期是一个核心概念，理解它对于构建健壮、高效的应用程序至关重要。Flutter 的生命周期可以分为两个层面：

1. **Widget 生命周期**：指单个 Widget 从创建到销毁的整个过程，主要由 <code>State</code> 类的方法管理。
2. **App 生命周期**：指整个应用程序从启动到退出的状态变化，由 <code>WidgetsBindingObserver</code> 监听。

下面我将为你详细讲解这两个层面。

***

### 1. Widget 生命周期 (基于 StatefulWidget)

<code>StatefulWidget</code> 的生命周期围绕 <code>State</code> 类展开。下图清晰地展示了其完整流程：



![1758522239179-0eee397e-2a0a-4dc1-b715-838909910aca.png](/assets/img/posts/移动开发/Flutter/1758522239179-0eee397e-2a0a-4dc1-b715-838909910aca-940069.png)



#### 核心方法详解：

1. <code>**createState()**</code>
   * **时机**：当 <code>StatefulWidget</code> 被插入到 Widget 树中时，Flutter 框架会立即调用其 <code>createState()</code> 方法来创建对应的 <code>State</code> 对象。
   * **用法**：你必须重写此方法并返回你的 <code>State</code> 子类实例。通常你不会在此方法中做太多事情。
2. <code>**initState()**</code>
   * **时机**：在 <code>State</code> 对象被创建后，**且只会被调用一次**。它是生命周期的第一个“有用”的方法。
   * **用法**：这是执行一次性初始化操作的理想位置。
     * 初始化依赖于该 <code>State</code> 对象的数据（如 <code>AnimationController</code>）。
     * 订阅（Subscribe）到 Streams 或 <code>ChangeNotifier</code>。
     * 发起网络请求（但需要注意，<code>build</code> 方法可能会在数据返回前被调用）。
   * **重要**：你必须首先调用 <code>super.initState()</code>。
3. <code>**didChangeDependencies()**</code>
   * **时机**：
     * 在 <code>initState()</code> 之后立即被调用。
     * 当此 <code>State</code> 对象所依赖的\*\* InheritedWidget\*\* 发生变化时（例如，主题、Locale 等），该方法会被**重新调用**。
   * **用法**：如果你需要执行与 InheritedWidget 相关的初始化操作（例如从 <code>Provider</code> 获取数据），可以在这里进行。你也可以在这里发起网络请求，但需要小心处理以免重复请求。通常更推荐在 <code>initState()</code> 中初始化，然后使用 <code>Provider</code> 的监听机制来更新。
4. <code>**build()**</code>
   * **时机**：
     * 在 <code>didChangeDependencies()</code> 之后立即调用。
     * 每当你需要重新绘制 UI 时调用，例如：
       * 调用 <code>setState(() {})</code>。
       * 父 Widget 重建并传入了新的配置（<code>widget.someValue</code> 改变）。
       * 所依赖的 InheritedWidget 发生变化。
   * **用法**：这是一个**必须**重写的方法。它根据当前的状态（State）和配置（Widget）来描述用户界面。这个方法应该是一个**纯函数**，没有副作用（例如，不要在这里修改状态或发起网络请求）。
5. <code>**didUpdateWidget(oldWidget)**</code>
   * **时机**：当父 Widget 重建并重新传入一个**相同运行时类型**的 <code>Widget</code> 时，Flutter 会对比新旧 Widget 的配置。如果配置发生了变化，它会更新当前的 <code>widget</code> 属性，然后调用此方法。
   * **用法**：用于响应 Widget 配置的更改。例如，如果旧的 Widget 有一个 <code>AnimationController</code>，而新的 Widget 传入了不同的动画参数，你可以在这里停止旧的并初始化新的。通常，你不需要重写此方法，因为 <code>State</code> 对象会自动根据新的 <code>widget.*</code> 属性在 <code>build()</code> 方法中重建。但在处理复杂状态（如动画）时非常有用。
   * **对比**：<code>setState()</code> 是内部状态变化，<code>didUpdateWidget</code> 是外部配置变化。
6. <code>**setState()**</code>
   * **时机**：当你需要改变 <code>State</code> 对象内部的数据（状态）并希望 UI 随之更新时，你需要在修改数据的回调函数中调用它。
   * **用法**：它通知框架：“我的状态已经改变了，请重新调用 <code>build()</code> 方法来更新 UI”。你只能在当前 State 对象是 <code>mounted</code> 的时候调用它。
7. <code>**deactivate()**</code>
   * **时机**：当 <code>State</code> 对象从 Widget 树中**暂时移除**时调用。最常见的情况是它在树中的位置被另一个 Widget 替换了。**注意**：同一个 Widget 可以在树的不同位置被移除和重新插入。
   * **用法**：很少使用。可用于在 Widget 暂时消失前保存一些状态。
8. <code>**dispose()**</code>
   * **时机**：当 <code>State</code> 对象**被永久地从 Widget 树中移除**时调用。这是生命周期的终点，该方法**只会被调用一次**。
   * **用法**：**这是进行清理工作的关键位置**，必须在此处释放所有资源以避免内存泄漏。
     * 取消计时器（<code>Timer</code>）。
     * 取消动画（<code>AnimationController.dispose()</code>）。
     * 取消对 Streams 或 <code>ChangeNotifier</code> 的订阅。
   * **重要**：你必须调用 <code>super.dispose()</code>。

#### <code>mounted</code> 属性

* 这是 <code>State</code> 类的一个 <code>bool</code> 属性。当 <code>createState()</code> 被调用后，<code>mounted</code> 被设为 <code>true</code>。当 <code>dispose()</code> 被调用后，它被设为 <code>false</code>。
* **重要规则**：在调用 <code>setState()</code> 之前，**必须**检查 <code>if (mounted) { ... }</code>，尤其是在异步回调中（如网络请求、<code>Future.delayed</code>）。因为在回调执行时，Widget 可能已经被销毁（<code>dispose</code>），此时调用 <code>setState()</code> 会抛出异常。

### 2. App 生命周期

App 生命周期指的是整个应用程序所处的状态（如前台运行、退到后台、挂起等）。你可以通过混入 <code>WidgetsBindingObserver</code> 来监听这些状态变化。

#### 使用方法：

**State 类混入\*\*\*\* **<code>**WidgetsBindingObserver**</code>：

1. dart

```dart
class _MyHomePageState extends State<MyHomePage> with WidgetsBindingObserver {
  // ...
}
```

**在\*\*\*\* **<code>**initState**</code>** \*\*\*\*中注册监听器**：

2. dart

```dart
@override
void initState() {
  super.initState();
  WidgetsBinding.instance.addObserver(this); // 注册监听
}
```

**重写\*\*\*\* **<code>**didChangeAppLifecycleState**</code>** \*\*\*\*方法**：

3. dart

```dart
@override
void didChangeAppLifecycleState(AppLifecycleState state) {
  super.didChangeAppLifecycleState(state);
  print('App state changed to: $state');
  // 根据不同的状态执行操作
  switch (state) {
    case AppLifecycleState.inactive:
      // 应用处于非活动状态，无法接收用户输入（如接电话、分屏）。
      break;
    case AppLifecycleState.paused:
      // 应用对用户不可见，进入后台。
      // 暂停动画、视频、释放不必要的资源。
      break;
    case AppLifecycleState.resumed:
      // 应用重新可见，回到前台。
      // 重新开始动画、恢复逻辑。
      break;
    case AppLifecycleState.detached:
      // 应用仍托管在Flutter引擎上但已从宿主View中分离（很少见）。
      break;
    case AppLifecycleState.hidden:
      // 应用完全隐藏（如切换到其他应用，但未完全进入后台栈，某些系统）。
      break;
  }
}
```

**在\*\*\*\* **<code>**dispose**</code>** \*\*\*\*中取消监听**：

4. dart

```dart
@override
void dispose() {
  WidgetsBinding.instance.removeObserver(this); // 取消监听
  super.dispose();
}
```

### 总结与实践建议

| 方法 | 调用次数 | 常用用途 |
| --- | --- | --- |
| <code>**initState**</code> | 1次 | 初始化依赖、订阅、动画控制器、一次性请求。 |
| <code>**didChangeDependencies**</code> | ≥1次 | 处理与 InheritedWidget 相关的初始化或变化。 |
| <code>**build**</code> | ≥1次 | 构建UI。必须是纯函数。 |
| <code>**didUpdateWidget**</code> | ≥0次 | 响应外部配置（Widget）的变化，更新内部状态。 |
| <code>**setState**</code> | ≥0次 | 通知框架内部状态已变，需要重建UI。 |
| <code>**deactivate**</code> | ≥0次 | Widget 被暂时移除时保存临时状态。 |
| <code>**dispose**</code> | 1次 | **关键**：清理资源，防止内存泄漏。 |
| <code>**didChangeAppLifecycleState**</code> | ≥0次 | 监听App全局状态，管理全局资源（如摄像头、音频）。 |

**黄金法则**：

* **初始化在\*\*\*\* **<code>**initState**</code>，**清理在\*\*\*\* **<code>**dispose**</code>。
* 在异步回调中调用 <code>setState</code> 前，**务必检查\*\*\*\* **<code>**mounted**</code>。
* 保持 <code>build</code> 方法纯净，不要在其中修改状态或执行耗时操作。
